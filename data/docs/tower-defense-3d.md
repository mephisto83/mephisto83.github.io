# tower-defense-3d

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

Tower Defense 3D is a browser-based 3D tower defense game built with React Three Fiber and TypeScript. The game features a unique resource management system where players deploy autonomous miners to collect energy from floating asteroids to power defensive towers against incoming enemy waves. Unlike traditional tower defense games, this implementation includes real-time 3D graphics with WebGL rendering, dynamic pathfinding for enemies across multiple routes, and a sophisticated energy economy that requires strategic resource allocation between mining operations and defensive capabilities.

## Architecture

The application follows a component-entity-system (CES) architecture with React Three Fiber handling 3D rendering and Zustand managing global game state. The core architecture separates concerns into distinct systems: EnemySystem handles pathfinding and AI behavior, TowerSystem manages tower targeting and combat, MiningSystem controls autonomous miner behavior and resource collection, ProjectileSystem handles ballistics and damage calculation, WaveSystem spawns enemies in configurable waves, and GameLogicSystem monitors win/lose conditions. Game state is centralized in src/state/gameState.ts using Zustand, while game configuration is defined in src/game/config/gameConfig.ts. The 3D scene is rendered through src/game/GameCanvas.tsx with individual entity components in src/game/components/ handling visual representation and basic interactions.

## Key Features

- 3D tower defense gameplay with WebGL rendering using React Three Fiber
- Autonomous miner units that collect energy from dynamic asteroid fields
- Multiple tower types (Basic, Sniper, Cannon, Miner Station) with distinct combat capabilities
- Enemy AI with pathfinding along multiple predefined routes using CatmullRomCurve3
- Real-time energy management system requiring strategic resource allocation
- Dynamic asteroid spawning and physics simulation with velocity-based movement
- 3D audio system with positional sound effects for combat and explosions
- Modular game systems architecture with frame-based update loops
- Configurable wave spawning with difficulty scaling based on wave progression
- Health-based combat system for towers, miners, and enemies with collision detection

## Getting Started

Clone the repository and ensure Node.js 16+ is installed. Run 'npm install' to install dependencies including React Three Fiber, Three.js, and Zustand. The project uses Vite as the build system, so run 'npm run dev' to start the development server on localhost. The game requires WebGL support and will load 3D models from public/models/ and audio assets from public/audio/. For production builds, run 'npm run build' followed by 'npm run preview'. The game state persists in browser memory during play sessions, and all asset loading is handled by the AssetPreloader component in src/game/utils/AssetPreloader.tsx.
