# exr-py-read

![Language](https://img.shields.io/badge/language-Python-blue)

## Overview

exr-py-read is a Python utility library for reading, processing, and manipulating OpenEXR image files. The project provides a comprehensive framework for performing mathematical operations on EXR images including addition, subtraction, normalization, and conversion to height maps. It's designed to work with high dynamic range (HDR) images commonly used in computer graphics and VFX workflows, offering both low-level channel manipulation and high-level image processing operations.

## Architecture

The architecture follows a modular design with core functionality split across src/imageprocess.py and src/illustration.py. The ImageProcess class serves as the main processing engine, providing static methods for file I/O, mathematical operations, and format conversions. The Illustration class acts as a data container, encapsulating EXR file metadata (data window, size) along with raw RGB channel data. Test scripts (test.py, test2.py, test3.py) demonstrate various usage patterns, while add-images.py showcases the operation-based processing system using ExrUtil.executeOperations().

## Tech Stack

- Python 3.7
- OpenEXR
- Imath
- NumPy-style array processing
- JSON for configuration and output

## Key Features

- EXR file reading and writing with full metadata preservation
- RGB channel extraction and manipulation as 32-bit float arrays
- Image arithmetic operations (addition, subtraction) between multiple EXR files
- Brightness normalization algorithms for HDR tone mapping
- Height map generation with configurable scaling factors (sx, sy, sz)
- JSON-based operation configuration system for batch processing
- Cross-platform support with Windows-specific Python paths in package.json

## Getting Started

1. Install Python 3.7-32bit as indicated by the hardcoded paths in package.json. 2. Install the provided OpenEXR wheel files: 'pip install OpenEXR-1.3.2-cp37-cp37m-win32.whl' for Python 3.7 or 'OpenEXR-1.2.0-cp27-none-win_amd64.whl' for Python 2.7. 3. Ensure sample EXR files (something.exr, thinlines.exr) are in the root directory. 4. Run tests using npm-style commands: 'npm run test', 'npm run test2', 'npm run test3' or directly with Python. 5. For image processing operations, use 'npm run add-images' which demonstrates the ExrUtil.executeOperations() workflow. 6. Output files (output.exr, output2.exr, thinlines_out.exr, thinline_hm.json) will be generated in the root directory.
