## Overview

Bridge is a React-Redux communication application that serves as a contact management and messaging platform. The application appears to be designed for managing connections between users, facilitating conversations, and handling business card-style contact information with emails and phone numbers. Based on the codebase structure and naming conventions, it seems to be a social networking or professional networking application that allows users to create connections, exchange contact cards, and engage in messaging conversations through a unified interface.

## Architecture

The application follows a classic Redux architecture with React frontend components. The codebase is organized into distinct layers: accessors (src/accessors/) that provide read-only access to state data, actions (src/actions/) that define state mutations, reducers (src/reducers/) that handle state updates, and services (src/service/) for API communications. The accessor layer includes specialized modules like contactAccessor.js, cardAccessor.js, messengerAccessor.js, and connectionsAccessor.js that provide structured access to different domain objects. The application uses Redux for state management with redux-thunk for async actions, react-router for navigation, and Material-UI for UI components. The server-side is handled through Express.js (referenced in src/server.js), and the build process uses Webpack with Babel for ES6+ transpilation.

## Getting Started

1. Clone the repository and navigate to the project directory. 2. Install dependencies using 'npm install' which will install React, Redux, Material-UI and other dependencies listed in package.json. 3. Ensure you have Node.js installed as the application uses Express.js server (src/server.js). 4. Run 'npm start' to launch the development environment which will execute watch-html, watch-js, and serve scripts concurrently. 5. The application will start the Express server via 'node src/server.js' and set up file watchers for HTML and JavaScript changes. 6. For production builds, use 'npm run build-prod' which uses Webpack to bundle the application. 7. Run tests using 'npm test' which executes Mocha with Babel compilation for ES6 support. 8. For development with hot reloading, the gulpfile.js is configured to watch the src directory and copy changes to '../BridgeNative/src', suggesting this may be part of a larger multi-platform application setup.
