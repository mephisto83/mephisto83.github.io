# bridge.serve

![Language](https://img.shields.io/badge/language-C%23-blue)

## Overview

Bridge.serve is a C#-based authentication and user management service built on ASP.NET Web API with OAuth 2.0 support. The project implements a comprehensive authentication system that supports multiple external providers (Facebook, Google, Twitter) while maintaining local user accounts through a custom Identity framework implementation. The system appears to be designed as a backend service for a contact/connection management application called 'Bridge', providing secure authentication endpoints and user data management capabilities.

## Architecture

The project follows a layered architecture with three main components: Bridge.Auth (authentication layer), Bridge.Business (business logic layer), and Bridge.Data (data access layer). The authentication layer implements OAuth 2.0 flows through ApplicationOAuthProvider.cs and supports external authentication providers via dedicated provider classes (FacebookAuthProvider.cs, GoogleAuthProvider.cs, TwitterAuthProvider.cs). The business layer contains repositories for managing entities like Cards, Connections, Contacts, and Conversations, while utilizing NHibernate for ORM through HibernateSessionFactory.cs. The data layer defines entity models including BridgeUser for authentication and various business entities. The system uses dependency injection through ProjectStrapper and implements custom Identity stores (UserStore.cs, RoleStore.cs) for user management.

## Key Features

- OAuth 2.0 authentication server implementation with bearer token support
- Multi-provider external authentication (Facebook, Google, Twitter) with unified user management
- Custom Identity framework implementation with BridgeUser entity and specialized stores
- RESTful API endpoints for login, external authentication, and user management
- NHibernate-based data persistence with repository pattern implementation
- Contact and connection management system with card-based user profiles
- Image storage integration with Azure Blob Storage through ImageBlobStorageUploadProvider.cs
- Real-time communication capabilities via SignalR integration
- Cross-origin resource sharing (CORS) support for web client integration

## Getting Started

1. Clone the repository and open the solution in Visual Studio. 2. Restore NuGet packages for all projects (Bridge.Auth, Bridge.Business, Bridge.Data). 3. Configure database connection strings in App.config files for NHibernate session factory. 4. Set up external authentication provider credentials (Facebook, Google, Twitter) in the Startup.cs configuration. 5. Configure Azure Blob Storage connection strings in Bridge.Business for image storage functionality. 6. Build the solution to ensure all dependencies are resolved. 7. Run database migrations or schema creation scripts for NHibernate entities. 8. Configure IIS or IIS Express to host the Bridge.Auth Web API project. 9. Update CORS settings and allowed origins in WebApiConfig.cs for client applications. 10. Test the authentication endpoints using /api/sanity for basic connectivity and /api/login/ExternalLogin for OAuth flows.
