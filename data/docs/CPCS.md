# CPCS

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

CPCS (Client Profile Case Study) is a comprehensive educational web application designed for counseling and psychology programs. It enables facilitators to create multidimensional client profile assignments where students answer structured questions to build realistic case studies. The system addresses specific pain points in counseling education by providing role-based access, drag-and-drop question management, email-link authentication, and professional PDF generation. The platform supports three distinct user roles: administrators who manage the system and question banks, facilitators who create and manage assignments, and students who complete client profiles through secure, accessible interfaces.

## Architecture

The application follows a modern serverless architecture built on Firebase. The frontend is a React 18 SPA using TypeScript and Vite, structured with feature-based organization under src/components/ and src/features/. The backend leverages Firebase services: Firestore for data persistence with role-based security rules, Firebase Functions for serverless business logic (functions/src/), Firebase Auth for authentication with custom claims, and Firebase Storage for file management. Key backend functions include admin.ts for user management, assignments.ts for assignment CRUD operations, submissions.ts for student data handling, and email.ts for notification services. The system implements comprehensive security through firestore.rules and uses Firebase Functions for server-side validation and PDF generation via Puppeteer.

## Key Features

- Role-based access control with admin, facilitator, and student roles managed through Firebase custom claims
- Drag-and-drop question reordering interface for facilitators after assignment creation
- Email-link authentication system eliminating password requirements for students
- Magic link re-entry capability allowing students to return to their profiles
- Professional PDF generation with accessible design using Puppeteer in Cloud Functions
- Comprehensive admin user management including email reset, account deletion, and role changes
- Question categorization and course-specific question lists with bulk import/export
- Sample profile auto-generation for classroom demonstrations
- Assignment kit downloads including questions and all student responses
- Real-time submission tracking and optional feedback system with visibility controls
- Accessible, WCAG 2.1 AA compliant interface with screen reader support
- AI integration with Groq and OpenAI for enhanced content generation
- Firebase emulator support for local development and testing

## Getting Started

1. Install prerequisites: Node.js 18+, Firebase CLI (`npm install -g firebase-tools`), and create a Firebase project. 2. Clone the repository and run `npm install` in the root directory, then `cd functions && npm install` for backend dependencies. 3. Initialize Firebase with `firebase login` and `firebase init`, selecting Hosting, Firestore, Functions, and Storage services. 4. Configure environment variables by copying the Firebase config to .env.local with keys: VITE_FIREBASE_API_KEY, VITE_FIREBASE_AUTH_DOMAIN, VITE_FIREBASE_PROJECT_ID, VITE_FIREBASE_STORAGE_BUCKET, VITE_FIREBASE_MESSAGING_SENDER_ID, VITE_FIREBASE_APP_ID. 5. Start development with `npm run firebase:emulators` in one terminal and `npm run dev` in another. 6. Create initial admin user by running the bootstrap scripts (setup-admin-quick.js) or manually through Firebase Auth emulator. 7. Deploy to production with `npm run firebase:deploy` which builds the frontend and deploys all Firebase services. 8. Access the Firebase Console to set up additional configuration like SendGrid API keys for email functionality using `firebase functions:config:set sendgrid.key='your-key'`.
