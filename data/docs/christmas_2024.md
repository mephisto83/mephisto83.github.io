# christmas_2024

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

This is a TypeScript-based image processing and batch generation system designed for creating personalized Christmas-themed images and collages. The project appears to be a family photo processing tool that takes individual portrait images (head shots and full-body photos) and applies various AI-generated scenarios and artistic styles to create customized holiday images for family members. The system processes a large collection of family photos (86 files, 8215 lines of code) and generates styled variations using different AI models and artistic filters.

## Architecture

The application follows a pipeline architecture with three main processing stages: pre-processing (index.ts), cloud image processing (icanfeel.ts), and collage generation (collage.ts). The system uses JSON configuration files (pre_source.json, postsource.json, cloud_postsource.json) to define subjects, prompts, and rendering targets. Core interfaces defined in interface.ts structure the data flow, while util.ts provides common functionality for directory management and item interleaving. The Sharp library handles image manipulation, resizing, and composite operations. The architecture supports multiple AI models and style transfers, with organized output directory structures for different subjects, scenarios, and styles.

## Key Features

- Batch processing of family portrait images with AI-generated scenarios and prompts
- Support for multiple AI models including sd3_5, juggernautXL_v8Rundiffusion, and other safetensors models
- Style transfer capabilities using reference style images from configurable directories
- Automated collage generation with shuffled image layouts and thumbnail resizing
- Hierarchical output organization by model, subject, prompt, and scenario
- Character description templating system for personalized image prompts
- Cloud image processing pipeline for atmospheric and background elements
- Directory structure management with automatic folder creation
- Interleaved processing to distribute workload across different subjects evenly

## Getting Started

1. Clone the repository and navigate to the project directory. 2. Install dependencies with 'npm install' (requires Sharp library and ts-node). 3. Create a pre_source.json file with subjects array containing name, character-description, and image paths for head-shot and full-body photos. 4. Place family photos in the images/ directory following the naming convention seen in the file tree. 5. Configure style images directory path in the styleImagesSource variable (currently set to D:\images\styles\kg). 6. Run 'npm run build' to compile TypeScript to JavaScript. 7. Execute 'npm run quick' to run the main processing pipeline, or 'npm run collage' to generate photo collages. 8. Use 'npm run dev' for development mode with ts-node. Generated images will be saved in the output/ directory organized by model and subject name.
