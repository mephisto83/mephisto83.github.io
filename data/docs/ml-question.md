# ml-question

![Language](https://img.shields.io/badge/language-Python-blue)

## Overview

ml-question is a Python-based mathematical equation parsing and analysis system that processes LaTeX mathematical expressions and converts them into structured data representations suitable for machine learning applications. The system implements a sophisticated lexical analyzer and parser that can break down complex mathematical equations into abstract syntax trees (ASTs) and generate numerical feature vectors from the parsed mathematical structures. The primary goal appears to be creating machine-readable representations of mathematical equations that can be used for training ML models or performing automated mathematical reasoning.

## Architecture

The system follows a multi-layered parsing architecture with distinct separation of concerns. The core workflow starts with EquationData class which handles LaTeX input preprocessing and delegates to LexicalBuilder for tokenization. The LexicalBuilder performs lexical analysis using pylatexenc to parse LaTeX syntax, while LexicalBuilderRule and LexicalBuilderRules manage parsing rules and operator precedence. Node objects represent individual mathematical elements in an AST structure, with TokenType enums defining the vocabulary of mathematical operations and symbols. The Graph class provides functionality to convert the parsed AST into numerical feature vectors and adjacency matrices suitable for ML consumption. The system uses a stack-based parsing approach with look-ahead and look-behind capabilities for handling complex mathematical expressions with proper operator precedence and associativity.

## Key Features

- LaTeX mathematical expression parsing with support for integrals, trigonometric functions, variables, and operators
- Abstract Syntax Tree generation from parsed mathematical expressions with unique node identification via UUID
- Rule-based lexical analysis system supporting operator precedence and associativity through LexicalBuilderRule classes
- Mathematical expression preprocessing with automatic parentheses handling and equation massaging
- Graph-based representation conversion that transforms ASTs into numerical feature vectors for ML applications
- Comprehensive tokenization supporting numbers, variables, operators, functions (cos, sin, tan), integrals, and special symbols
- Stack-based parsing engine with contextual analysis for handling complex nested mathematical structures
- Node relationship mapping with parent-child hierarchies and graph connectivity analysis

## Getting Started

1. Install Python 3.x and ensure pip is available
2. Install required dependencies: pip install pylatexenc numpy
3. Clone the repository and navigate to the project directory
4. The main entry point appears to be run.py for execution
5. To use the system, import EquationData class: from equationdata import EquationData
6. Create an instance with a LaTeX equation: eq_data = EquationData(your_latex_equation)
7. Parse and build the equation tree: math_node = eq_data.findMathNode(); tree = eq_data.buildTree(math_node)
8. Generate ML-ready features using Graph class: from graph import Graph; features = Graph.buildEquationImage(eq_data)
9. Run tests using the provided test files: python test_equationdata.py, python test_lexicalbuilder.py, etc.
