## Overview

Slicer-Mind-the-Gap is a web-based 3D audio visualization game built with TypeScript and WebGL technologies. The project appears to be a rhythm-based or music-synchronized game where players navigate through dynamically generated levels while audio plays. Based on the extensive collection of Blender files (.blend), 3D models (.fbx), texture assets, and audio files (.mp3), this is a complete interactive 3D experience that combines real-time audio processing with visual effects and gameplay mechanics.

## Architecture

The project follows a modular architecture with separate concerns for level generation, audio processing, and 3D rendering. The core logic is implemented in TypeScript with key files including generate_level.js for procedural level creation, levels.js for level management, and build-composite.js for asset compilation. The 3D assets are organized in a structured hierarchy under Models/ directory with cubes/ subfolder containing various cube variants and their associated PBR textures (BaseColor, Normal, OcclusionRoughnessMetallic). Audio visualization components are managed through multiple Blender files (audio_visualizer.blend variations) suggesting different visual states or configurations. The composite.json and levels.json files serve as configuration data for the game's content pipeline.

## Key Features

- Real-time audio visualization with multiple visual configurations through audio_visualizer.blend files
- Procedural level generation system via generate_level.js
- 3D cube-based gameplay mechanics with various cube types (thin, wide, fat_cube_storage, center, left, right)
- PBR material system with BaseColor, Normal, and OcclusionRoughnessMetallic texture maps
- Multi-difficulty level system with UI screens for main menu, difficulty selection, and error handling
- Audio synchronization with coolsaber.mp3 and 4 clash 2.mp3 files
- Modular 3D asset pipeline with organized Models/ structure and FBX export workflow
- Game state management through levels.json configuration

## Getting Started

1. Clone the repository and ensure you have a web server capable of serving static files. 2. Install Node.js if not already present to run the JavaScript build tools. 3. Run build-composite.js to compile the asset pipeline and generate the composite.json configuration. 4. Execute generate_level.js to create the initial level data and populate levels.json. 5. Serve the project through a local web server (the HTML files require WebGL context which needs HTTP/HTTPS protocol). 6. Ensure audio files (coolsaber.mp3, 4 clash 2.mp3) are accessible and test audio playback functionality. 7. If modifying 3D assets, use Blender to edit .blend files and export to .fbx format, maintaining the texture organization in respective subdirectories. 8. Test the game flow through the UI screens: mainmenu.jpg → difficultylevel.jpg → gameplay → oopsscreen.jpg error handling.
