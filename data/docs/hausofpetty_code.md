# hausofpetty_code

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

hausofpetty_code is a comprehensive Firebase-based backend system for a fashion e-commerce platform that manages social media content, product updates, and sales operations. The project implements a sophisticated multi-layered architecture with Firebase Functions as the serverless backend, Firestore as the database, and includes both server-side controllers and client-side Redux state management. The system appears to be designed for managing 'Haus Agents' (likely fashion agents or influencers) who can create and manage social posts, track product updates, and handle sales items through a robust permission-based system.

## Architecture

The architecture follows a clean separation of concerns with three main layers: controllers (funcs/functions/src/controllers/auto/), services (funcs/functions/src/functions/), and models (funcs/functions/src/models/). The system implements a dual-deployment strategy with both Firebase Functions (funcs/) and local development code (localcode/). Each entity (SocialPost, ProductUpdates, SalesItem, etc.) follows a consistent pattern with property definitions, service layers, Redux slices, and comprehensive test suites. Authentication and authorization are handled through Firebase Auth with custom claims validation, ensuring users can only access their own hausAgent data. The frontend uses Redux Toolkit for state management with async thunks for API communication.

## Getting Started

1. Install dependencies by running npm install in both funcs/functions/src/ and localcode/ directories. 2. Set up Firebase project and configure Firebase Admin SDK credentials. 3. Deploy Firebase security rules from localcode/features/features/*/rules files. 4. Configure Firestore database with proper indexes for query operations. 5. Set up environment variables for Firebase configuration in functions/config.ts. 6. Run tests using npm test in localcode/maestro-tests/ to verify setup. 7. Deploy Firebase Functions using the build-mac script or equivalent for your platform. 8. Configure hausAgent claims in Firebase Auth for user authorization. 9. Set up local development environment by configuring firestore.ts with proper connection settings. 10. Initialize Redux store with all feature slices from localcode/features/features/*/slice.ts files.
