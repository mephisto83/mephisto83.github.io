# red-quick-groups

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

red-quick-groups is a TypeScript library that provides a generic, abstract framework for managing group membership and permissions systems. The library implements a comprehensive group management system with support for member invitation workflows, role-based access control, and configurable membership limits. It serves as a foundation for building applications that require user group functionality with features like member management, invitation systems, read/write permissions, and administrative roles.

## Architecture

The library follows an object-oriented architecture with a generic abstract base class `Group<T, M>` in `src/group.ts` that defines the contract for group management operations. The architecture uses TypeScript generics where T represents the group data structure and M represents member objects. The base class defines abstract methods that must be implemented by concrete classes (like the `Test` class in `tests/test-group.ts`). The system manages five key collections: members, invitees, admins, writers, readers, and membership requests, along with a configurable settings system for limits and permissions. The library uses dependency injection patterns where the concrete group implementation is passed to the constructor.

## Key Features

- Generic group management with TypeScript generics for type safety
- Member lifecycle management including adding, removing, and checking membership status
- Invitation system with invite/accept/reject workflows and configurable invite limits
- Role-based access control with admin, writer, and reader permission levels
- Membership request system for user-initiated join requests
- Configurable membership and invitation limits with get/set/has configuration methods
- Idempotent operations ensuring duplicate actions don't cause side effects
- Comprehensive test coverage with 20+ test scenarios in tests/tests.test.ts

## Getting Started

1. Clone the repository and navigate to the project directory. 2. Install dependencies using `npm install` which will install TypeScript, Mocha, Chai, NYC, and other dev dependencies from package.json. 3. Build the TypeScript code using the TypeScript compiler (tsc command, configured via tsconfig.json). 4. Run tests using `npm run test` which executes Mocha with watch mode, or use `npm run coverage` to generate test coverage reports with NYC. 5. To use the library, extend the abstract `Group<T, M>` class from `src/group.ts` and implement the required abstract methods: members(), invitees(), getMemberId(), writers(), admins(), readers(), membershipRequests(), has(), get(), and set() methods as demonstrated in the `Test` class in `tests/test-group.ts`.
