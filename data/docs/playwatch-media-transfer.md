# playwatch-media-transfer

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

Playwatch Media Transfer is a comprehensive Firebase-based backend service designed for managing media content, user interactions, and social media integrations within what appears to be a content distribution platform. The system features an auto-generated controller architecture with extensive CRUD operations for various entities including media customizations, social media accounts, curated content lists, and user management. Built as a Firebase Functions project, it provides a scalable serverless backend for a media platform that handles content creation, approval workflows, release scheduling, and multi-tenant content management with role-based access control.

## Architecture

The application follows a layered architecture with auto-generated controllers in `src/controllers/auto/` that handle HTTP requests, service functions in `src/functions/` for business logic and Firestore operations, and model definitions in `src/models/` with TypeScript interfaces and property enums. The system uses Firebase Firestore for data persistence with collection-based organization (e.g., `socialMediaAccounts/{owner}/socialMediaAccount`, `processedSocialMediaAccounts`). Each entity follows a consistent pattern with service files providing CRUD operations, model files defining interfaces and update handlers, and controllers implementing role-based access control through player permissions. The architecture supports both hierarchical (owner-based) and flat collection structures, with automatic denormalization and snapshot-based change tracking.

## Key Features

- Auto-generated CRUD controllers with role-based access control using PlayerMarketingPermissions and PlayerContentCreatorPermissions
- Social media account management with hierarchical storage under owner collections
- Processed social media account tracking with denormalized data structures
- Content curation system supporting curated media lists and series lists
- Release schedule management with automated release/unrelease actions
- Media processing capabilities with FFmpeg and Sharp integration
- Comprehensive permission system with player roles and content creator invitations
- Snapshot-based change tracking and model synchronization
- Multi-tenant architecture with owner-based data isolation
- Coherent screen management for content display coordination

## Getting Started

1. Install Node.js 16 and npm dependencies using `npm install`. 2. Configure Firebase project by setting up the Firebase Admin SDK keys (referenced as `playwatch-dev-test-firebase-adminsdk-so4h6-3f7d229d77.json` and `playwatch-f1376-firebase-adminsdk-a5ocm-5ca1202bab.json`). 3. Build the TypeScript code using `npm run build` which compiles to the `lib/` directory. 4. For development, run `npm run dev` to build and execute `lib/job_index.js` with staging flag. 5. For local testing, use `npm run serve-all` to start Firebase emulators. 6. Deploy to Firebase using `npm run deploy` for functions or `npm run publish` for hosting. 7. Set up Firestore rules using the `firestore.rules` file. 8. Configure environment variables for Firebase project IDs and ensure proper service account permissions for Firestore access.
