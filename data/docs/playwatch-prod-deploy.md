![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

PlayWatch is a comprehensive media streaming and content management platform built on Firebase with TypeScript. The system appears to be a production deployment for a video streaming service that manages content, users, metrics, and social media integrations. It features a sophisticated backend architecture with auto-generated controllers, models, and services for handling various aspects of media delivery, user management, content curation, and analytics tracking.

## Architecture

The application follows a layered Firebase Functions architecture with auto-generated CRUD operations. The core structure includes: (1) Firebase Functions backend with Express.js routing in functions/src/controllers/auto/ containing domain-specific controllers for accounts, media, players, devices, and metrics; (2) Data models with TypeScript interfaces and property definitions in functions/src/models/ providing type safety and schema definitions; (3) Service layer with Firestore operations in functions/src/functions/ handling database interactions, queries, and business logic; (4) Firebase hosting for frontend deployment with automatic CI/CD through npm scripts. The system uses a denormalized data approach with versioned entities and meta-properties for relationship tracking.

## Key Features

- Auto-generated CRUD controllers for 40+ domain entities including accounts, media, players, devices, and content houses
- Comprehensive metrics system with MetricDeployment, MetricDeploymentConfig, and MetricInitialization for analytics tracking
- Social media account management with SocialMediaAccount and ProcessedSocialMediaAccount models for integration tracking
- Content management system with curated media lists, series management, and content creator invitation workflows
- Player and device management with support for multiple content carriers and release zones
- Stripe payment integration for sales items and customer billing
- Versioned entity system with soft deletes and audit trails
- Firebase emulator support for local development and testing
- Datadog integration for monitoring and observability

## Getting Started

1. Clone the repository and navigate to the functions/ directory. 2. Install dependencies with 'npm install' ensuring Node.js 16 is installed. 3. Set up Firebase project and obtain service account credentials (playwatch-dev-test-793123b161be.json referenced in package.json). 4. Configure Firebase CLI and authenticate with your project. 5. Set GOOGLE_APPLICATION_CREDENTIALS environment variable pointing to your service account file. 6. Run 'npm run build' to compile TypeScript to JavaScript. 7. For local development, use 'npm run serve-all' to start Firebase emulators with all services. 8. Deploy functions with 'npm run deploy' or hosting with 'npm run publish'. 9. For full deployment use 'npm run all' which deploys all Firebase services with the message 'deploy all'.
