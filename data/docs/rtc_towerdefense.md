# rtc_towerdefense

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

RTC Tower Defense is a real-time multiplayer tower defense game built with React, TypeScript, and WebRTC technology. The game enables peer-to-peer multiplayer gameplay where players can join game lobbies, place defensive towers on a battlefield, and compete against each other in real-time without requiring a dedicated game server. The project leverages Firebase Firestore for WebRTC signaling and connection coordination, while the Excalibur.js game engine handles rendering and game mechanics.

## Architecture

The application follows a client-server architecture pattern where one player acts as the host (HostRTC) running the authoritative game simulation, while other players connect as clients (ClientRTC) via WebRTC data channels. The host manages the complete game state using gameLogic.ts functions like stepGame() and applyClientAction(), broadcasting state updates to all connected clients at 60fps. Firebase Firestore serves as the signaling server for WebRTC connection establishment through the rtc/ module, handling peer discovery, offer/answer exchange, and ICE candidate relay. The game state is centralized in shared/gameTypes.ts, defining interfaces for GameState, Player, Tower, and Unit entities. The React frontend (App.tsx, Game.tsx) provides the UI layer, while battlefield/Battlefield.tsx likely handles the game rendering via Excalibur.js.

## Key Features

- Peer-to-peer real-time multiplayer using WebRTC data channels
- Host-client architecture with authoritative server simulation
- Firebase Firestore-based WebRTC signaling for connection establishment
- Tower placement system with gold-based economy (barracks towers cost gold from player resources)
- Real-time unit spawning and battlefield simulation via stepGame() function
- Lobby system with player ready states and match start coordination
- Comprehensive test coverage for RTC connections (ClientRTC.test.ts, HostRTC.test.ts)
- Game state synchronization with tick-based simulation updates

## Getting Started

1. Clone the repository and run 'npm install' to install dependencies including React, Firebase, and Excalibur.js. 2. Create a Firebase project and obtain configuration credentials. 3. Create a .env.local file with Firebase environment variables: VITE_FIREBASE_API_KEY, VITE_FIREBASE_AUTH_DOMAIN, VITE_FIREBASE_PROJECT_ID as referenced in src/firebase.ts. 4. Run 'npm run dev' to start the Vite development server. 5. Open the application in multiple browser windows/tabs to test multiplayer functionality. 6. One player creates a game session (becomes host), others join using the game ID. 7. Players set ready status in lobby, host starts the match, and real-time gameplay begins with WebRTC peer-to-peer communication. 8. Run 'npm test' to execute the Vitest test suite covering RTC functionality and game logic.
