## Overview

This is a minimal TypeScript demonstration project showcasing dynamic distribution patterns through a host API interface. The project implements a simple command-line application that processes input arguments and communicates with an external host system through a defined API contract. The codebase serves as a proof-of-concept for creating distributable modules that can interact with host environments through standardized interfaces, making it suitable for plugin architectures or embedded execution contexts.

## Architecture

The architecture follows a clean separation of concerns with two main components: src/index.ts serves as the entry point containing the main execution logic wrapped in an async IIFE (Immediately Invoked Function Expression), while src/api.ts defines the contract through the HostAPI interface. The main module expects a globally declared 'api' object conforming to the HostAPI interface, which provides the output method for communicating results back to the host environment. This pattern enables loose coupling between the executable code and the host system, allowing the module to be distributed and executed in different contexts without modification.

## Key Features

- Host API abstraction through HostAPI interface for decoupled communication
- Command-line argument processing using process.argv for input handling
- Asynchronous execution pattern with async/await for potential future extensibility
- TypeScript compilation pipeline with npx tsc build script
- Minimal dependency footprint with only development-time TypeScript tooling

## Getting Started

1. Clone the repository and navigate to the project directory. 2. Install dependencies using 'npm install' to get @types/node and typescript packages. 3. Build the TypeScript code using 'npm run build' which executes 'npx tsc' to compile src/index.ts and src/api.ts into the dist/ directory. 4. The compiled JavaScript will be output to dist/index.js as specified in package.json main field. 5. To run the application, execute the compiled JavaScript with Node.js, providing command-line arguments as needed. Note that a host environment must provide the global 'api' object implementing the HostAPI interface before execution.
