## Overview

FleetGames is a comprehensive 3D ship generation and visualization platform that combines procedural content generation with immersive technologies. The project centers around creating randomized spaceship models using Blender's Python API and displaying them through multiple interfaces including augmented reality (AR) applications, web-based 3D viewers, and real-time multiplayer environments. The system generates unique ship designs by procedurally combining modular parts from predefined asset libraries, with each ship having configurable parameters for complexity, size constraints, and aesthetic properties.

## Architecture

The project follows a modular architecture with distinct components for content generation, visualization, and networking. The core generation pipeline uses `mind/blender/fleetmaker.py` and `generateship.py` to create procedural ships by selecting random parts from 'hulls' and 'parts' groups, applying constraints based on surface area and connection points. The visualization layer includes multiple frontends: an Expo-based AR application (`mind/expo/fleetar/`) using Three.js and ExpoTHREE for mobile AR experiences, a web-based Three.js environment (`mind/threeenv/`) with custom effects and shaders, and various utility modules for asset loading and special effects. A Node.js backend (`mind/nodejs/socket/`) provides real-time multiplayer capabilities via Socket.IO. The Java component (`mind/java/hashgraphtest/`) appears to integrate with Swirlds distributed ledger technology, suggesting blockchain or consensus mechanisms for fleet management.

## Key Features

- Procedural ship generation with configurable seed values, part limits, and surface area constraints using Blender's ShipWright class
- Augmented Reality ship visualization through Expo-based mobile application with touch gesture controls and real-time rendering
- Advanced 3D effects including lightning systems, laser beams, and ASCII art rendering effects for enhanced visual appeal
- Real-time multiplayer capabilities via Socket.IO for fleet-based gaming experiences
- Automated batch ship generation with support for OBJ/MTL export and JSON metadata output
- Distributed ledger integration through Swirlds hashgraph technology for decentralized fleet management
- Cross-platform asset pipeline supporting multiple 3D formats and texture management

## Getting Started

1. Install Blender 2.5.6+ and ensure Python API access for ship generation. 2. Set up Node.js environment and run `npm install` in `mind/nodejs/socket/`, `mind/expo/fleetar/`, and `mind/expo/fleetv1/` directories to install dependencies including Three.js, Expo SDK, and Socket.IO. 3. Configure Blender asset groups by creating 'hulls' and 'parts' groups with ship component models in the Blender scene. 4. For AR functionality, install Expo CLI globally and ensure mobile device has Expo client app installed. 5. Set up Java environment with Maven for hashgraph integration by running `npm run build:exampledemo` in `mind/java/hashgraphtest/`. 6. Configure Azure Storage credentials in `mind/fleetcreateagent/` for cloud asset management. 7. Run ship generation with `blender --python mind/blender/generateship.py -- [seed] [limit] [percentage] [filename]` to create procedural ships. 8. Start the Socket.IO server with `node index.js` in the socket directory for multiplayer functionality. 9. Launch AR application with `expo start` in the fleetar directory and scan QR code with mobile device.
