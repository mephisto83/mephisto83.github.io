## Overview

RedQuickBuilder is a comprehensive visual application builder and code generation platform built as an Electron desktop application. It enables rapid development of full-stack applications by providing a visual interface for designing data models, relationships, user interfaces, and automatically generating corresponding backend APIs, frontend components, and database schemas. The platform supports multiple output frameworks including React, React Native, .NET Core, and various web technologies, making it a versatile RAD (Rapid Application Development) tool for enterprise and web applications.

## Architecture

The application follows a modular Electron architecture with a React-based renderer process and Node.js main process. The core consists of visual node-based editors for model design (builder/app/actions/), template-driven code generation engine (builder/app/templates/), and multi-target output systems. Key architectural components include: the UI state management system (redutils.ts, uiactions.ts), template processing engine with support for multiple frameworks (templates/standard/, templates/reactweb/), code analysis and generation modules (builder/app/analysis/), and worker threads for background processing (reqthread/, visi_blend/). The system uses a fetch service architecture for data management and includes specialized modules for handling different output types including web applications, mobile apps, and backend services.

## Key Features

- Visual model designer with drag-drop interface for creating data models and relationships
- Multi-framework code generation supporting React, React Native, .NET Core, Java Spring Boot, and Python Flask
- Template-driven architecture with customizable code templates (accidentviolations.tpl, get_model_agent_*.tpl)
- Agent-based permission system with role-based access control generation
- Automatic CRUD API generation with authentication and authorization
- Visual workflow designer for data chains and business logic
- Cross-platform desktop application built with Electron
- Real-time code preview and analysis capabilities
- Built-in fetch service with caching and background processing
- Support for multiple database backends and ORM generation
- Integrated testing framework with Azure Pipelines CI/CD
- Customizable UI component library with progress indicators and form controls

## Getting Started

1. Clone the repository and navigate to the builder/ directory. 2. Install dependencies using 'yarn install' (requires Node.js 13.x as specified in azure-pipelines.yml). 3. Build the development DLL using 'yarn build-dll'. 4. Start the development environment with 'yarn dev' which launches both the main Electron process and renderer development server. 5. For production builds, run 'yarn build' followed by 'yarn package-win' (Windows), 'yarn package-mac' (macOS), or 'yarn package-linux' (Linux). 6. Optional: Set up worker threads by running 'yarn jobrunnbuild && yarn jobrunner' for background processing capabilities. 7. The application will launch as a desktop application where you can begin creating models, relationships, and generating code using the visual interface.
