# test_calling_tams

![Language](https://img.shields.io/badge/language-JavaScript-blue)

## Overview

test_calling_tams is a Node.js utility project designed for interacting with AI image generation services, specifically the Stable Diffusion API and CivitAI platform. The project serves as a testing and integration toolkit for various AI model operations including image-to-image generation, text-to-3D conversion, and model metadata retrieval. It provides a collection of standalone JavaScript modules that demonstrate different API endpoints and workflows, with a focus on processing and managing AI model information for creative applications.

## Architecture

The project follows a modular architecture with separate JavaScript files handling distinct API operations. The core components include civitai.js and fetch_model.js for CivitAI API interactions, get_models.js for retrieving model lists from Stable Diffusion API, index.js as the main image generation handler, process_models.js for model data processing and trained words extraction, and text_3d.js for 3D model generation. Data flows through JSON files (models.json, processed_models.json) that serve as intermediate storage for model information, with each module operating independently while sharing common API credentials and data structures.

## Key Features

- CivitAI model metadata retrieval with trained words extraction via civitai.js and fetch_model.js
- Stable Diffusion API integration for img2img generation with customizable parameters in index.js
- Automated model list fetching and JSON storage through get_models.js
- Intelligent model processing pipeline in process_models.js that extracts URLs from descriptions and enriches model data with trained words
- Text-to-3D generation capability via text_3d.js with configurable guidance scale and output formats
- Webhook integration support for asynchronous processing workflows
- Model categorization filtering for stable_diffusion_xl, stable_diffusion, and lora model types

## Getting Started

1. Clone the repository and navigate to the project directory. 2. Install dependencies using 'npm install' which will install form-data and request packages. 3. Update the API key variable in get_models.js, index.js, and text_3d.js with your actual Stable Diffusion API credentials (currently contains placeholder key). 4. Run individual modules based on your needs: 'node get_models.js' to fetch and save model list to models.json, 'node civitai.js' to retrieve CivitAI model data, 'node fetch_model.js' to get specific model information by ID, 'node process_models.js' to process models.json and generate processed_models.json with enriched data, 'node index.js' for image generation testing, or 'node text_3d.js' for 3D model generation. 5. Ensure you have valid image URLs when testing img2img functionality in index.js, and modify the webhook URLs to point to your actual endpoints if using asynchronous processing.
