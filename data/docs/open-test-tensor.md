# open-test-tensor

![Language](https://img.shields.io/badge/language-JavaScript-blue)

## Overview

Open Test Tensor is a comprehensive testing framework designed for automated UI testing, with a focus on React Native applications and image-based testing. The project combines visual regression testing, automated screenshot capture, and test recording capabilities to create a robust testing suite for mobile and desktop applications. It provides tools for capturing UI elements, comparing images, recording user interactions, and generating test suites through automated mechanisms.

## Architecture

The project is structured into two main components: image-tools for automated testing utilities and ui-examples for sample React Native applications. The image-tools module contains helper utilities for screen capture (activateAndPosition.js), image comparison (compareImages.js), test recording (testRecorder.js), keyboard state management (keyboardstate.js), and test suite creation (createtestsuite.js). The architecture follows a modular approach where AppleScript integration handles macOS window management and screenshot capture, while Node.js modules manage test execution, file I/O operations, and user interaction recording. The ui-examples directory provides a React Native NativeBase implementation as a test target.

## Key Features

- Automated screenshot capture and comparison using node-resemble-v2 for visual regression testing
- macOS integration through AppleScript for simulator window management and positioning via activateAndPosition.js
- Real-time user interaction recording with keyboard and mouse event capture using iohook library
- QR code generation and reading capabilities for test data injection and validation
- Bulk image processing with automatic resizing and optimization using sips command-line tool
- Test suite generation and execution framework with JSON-based test definitions
- React Native UI element extraction and testing with NativeBase component library integration
- Cross-platform mobile testing support for both iOS and Android platforms
- Interactive test creation wizard with directory management and test organization

## Getting Started

1. Install dependencies by running 'npm install' in the image-tools directory to install core testing packages including canvas, robotjs, iohook, and selenium-webdriver. 2. Set up the React Native environment by navigating to ui-examples/react-native/nativebase/OpenTestTensorNativeBaseUIElements and running 'npm install' followed by 'react-native run-ios' or 'react-native run-android'. 3. Configure macOS permissions for accessibility features required by robotjs and iohook for automated mouse/keyboard control. 4. Install Cairo graphics library as a prerequisite for the canvas module following the installation guide referenced in compareImages.js. 5. Start the test suite by running 'npm run testsuite' in the image-tools directory, which will launch the interactive test creation wizard. 6. Configure directory paths for test files, images, and shrunk images when prompted by the createtestsuite.js interface. 7. Use the various npm scripts like 'npm run qr' for QR code testing, 'npm run compare' for image comparison, or 'npm run hook' for keyboard event monitoring to validate individual components.
