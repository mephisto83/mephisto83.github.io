# hausofpetty_firebase

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

hausofpetty_firebase is a comprehensive Firebase-based backend service for a fashion e-commerce platform called 'Haus of Petty'. The application provides a complete serverless architecture for managing social media posts, product updates, sales items, and user management with integrated Stripe payment processing. The system is built around Firebase Functions with TypeScript, featuring auto-generated controllers, comprehensive data models, and role-based access control for managing fashion merchandise and social media content.

## Architecture

The application follows a layered architecture with clear separation of concerns. The main structure includes: controllers (auto-generated endpoints in controllers/auto/), services (business logic in functions/ directory), models (data definitions with properties and type definitions), and Firebase integration through firestore.ts. The system uses a request/response pattern with authentication middleware, permission checking, and comprehensive error handling. Key architectural components include the auto-generated controller system (agent.ts, salesitem.ts, social.ts), service layer functions for CRUD operations, and model definitions with property enums and interfaces. The application integrates Firebase Auth for user management, Firestore for data persistence, and Stripe for payment processing.

## Getting Started

1. Install dependencies: Run 'npm install' in root directory and 'cd functions && npm install' for function dependencies. 2. Configure Firebase: Ensure firebase.json, firestore.rules, and firestore.indexes.json are properly configured for your Firebase project. 3. Set up authentication: Place the Firebase admin SDK key (hausofpetty-200b1-firebase-adminsdk-2mrt1-df134bb279.json) in the functions directory. 4. Configure Stripe: Set up Stripe secrets in functions/src/functions/stripe-secrets.ts. 5. Build the project: Run 'npm run build' to compile TypeScript. 6. Deploy functions: Use 'npm run deploy' for functions only, 'npm run publish' for hosting, or 'npm run all' for complete deployment. 7. Local development: Use 'npm run serve-all' to start Firebase emulators with proper environment variables. 8. Configure Firestore rules and indexes using the provided firestore.rules and firestore.indexes.json files.
