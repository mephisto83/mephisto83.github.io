# distributo

> Library to distribute known tasks to nodes on a network

![Language](https://img.shields.io/badge/language-TypeScript-blue) ![Stars](https://img.shields.io/github/stars/mephisto83/distributo)

## Overview

Distributo is a TypeScript-based distributed task execution framework that enables seamless distribution of computational tasks across multiple worker nodes on a network. The library implements a master-worker architecture where a central coordinator (TaskDistributor/CoordinatorTaskDistributor) manages and distributes tasks to connected worker nodes (WorkerClient/EnhancedWorkerClient) via Socket.IO communication. The framework supports both simple batch task processing and complex dynamic task execution with dependency installation, compilation, and sandboxed code execution using VM2.

## Architecture

The system follows a hub-and-spoke architecture with two main components: Master nodes (task-distributor.ts and coordinator-tast-distributor.ts) that serve as central coordinators, and Worker nodes (worker-client.ts and enhanced-worker-client.ts) that execute distributed tasks. The CoordinatorTaskDistributor extends the base TaskDistributor with HTTP endpoints for dynamic task submission, Bonjour service discovery for automatic worker discovery, and enhanced task lifecycle management. EnhancedWorkerClient provides sophisticated task execution capabilities including dynamic TypeScript compilation, dependency installation via npm, and sandboxed code execution using NodeVM. Communication between master and workers occurs through Socket.IO with automatic reconnection and task status tracking.

## Key Features

- Master-worker distributed task execution with automatic load balancing
- Dynamic task generation from TypeScript projects with automatic dependency resolution
- Sandboxed code execution using VM2 with security isolation
- Automatic service discovery via Bonjour protocol for seamless network setup
- HTTP REST API endpoints (/addTask, /addTasks) for runtime task submission
- Task lifecycle management with status tracking (pending, installed, failed, completed)
- Built-in logging and monitoring using Winston with file and console outputs
- TypeScript compilation and npm dependency installation for dynamic tasks
- Reconnection handling and fault tolerance for worker disconnections
- Binary file support with Base64 encoding for complex project assets

## Getting Started

1. Install dependencies: `npm install` 2. Build the project: `npm run build` (automatically increments patch version) 3. For basic usage, create a TaskDistributor instance in your master node with getTasks and handleResults functions, then call start() 4. For advanced usage, instantiate CoordinatorTaskDistributor with taskFolderPath pointing to your projects directory and optional taskGenerator function 5. Create WorkerClient instances specifying serviceType and processTasks function, or use EnhancedWorkerClient with taskTypes array for dynamic task handling 6. Workers automatically discover masters via Bonjour, or specify masterUrl directly 7. Submit tasks via HTTP POST to /addTask or /addTasks endpoints on the coordinator's HTTP port (default 5000) 8. Monitor execution through Winston logs in console and .log files
