# graph-spring

> Blender node graph layout

![Language](https://img.shields.io/badge/language-Python-blue)

## Overview

graph-spring is a Python implementation of a force-directed graph layout algorithm designed for use with Blender node graphs. The project provides a physics-based simulation system that uses spring forces to automatically arrange graph nodes in an aesthetically pleasing and functionally optimal layout. The implementation follows a classical spring-embedder model where nodes are treated as masses connected by springs, allowing the system to reach equilibrium through iterative force calculations.

## Architecture

The architecture consists of several core classes organized in a hierarchical structure within GraphSpring.py. At the foundation level, the Vector class provides 2D mathematical operations, while the Point class represents nodes as physical objects with mass, position, velocity, and acceleration. The Spring class models connections between points using Hooke's law physics. Higher-level abstractions include the Data class for node metadata, Node and Edge classes for graph structure representation, and the Graph class that orchestrates the entire layout system with event listeners and simulation management. The test.py file provides comprehensive unit testing coverage for all components.

## Getting Started

1. Ensure Python is installed on your system. 2. Clone the repository and navigate to the project directory. 3. Import the GraphSpring module in your Python environment or Blender script using 'from GraphSpring import Vector, Point, Spring, Edge, Data, Node, Graph'. 4. Run the test suite by executing 'python test.py' to verify the installation. 5. Create a Graph instance and add Node objects with appropriate Data labels for your use case. 6. Connect nodes using Edge objects to define the graph structure. 7. The spring physics system will automatically calculate optimal node positions based on the defined connections and spring parameters.
