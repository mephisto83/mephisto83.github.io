## Overview

room-banking-processing is a TypeScript-based educational simulation system that processes and manages virtual banking operations for classroom environments. The system appears to be designed for educational institutions to teach financial literacy through simulated banking activities, featuring auctions, business plans, transactions, and virtual economies. Built on Firebase infrastructure, it provides a comprehensive backend for managing virtual banking operations including user accounts, transactions, auctions, schools, and business activities with automated processing jobs and email notifications.

## Architecture

The system follows a modular MVC architecture with auto-generated controllers in src/controllers/auto/ for each entity type (auctions, banks, classrooms, etc.). Core business logic is centralized in src/functions/ with specialized processing jobs in src/features/functions/ that handle automated tasks like auction processing, transaction scheduling, and business evaluations. The Firebase ecosystem provides authentication, database (Firestore), and cloud functions, while the system uses a job-based processing model with src/features/jobIndex.ts orchestrating scheduled tasks. Email processing capabilities are implemented using nodemailer and googleapis for communication workflows.

## Getting Started

1. Install dependencies with 'npm install' in both root and src/ directories. 2. Configure Firebase project and obtain service account credentials. 3. Set up Firestore database with appropriate security rules. 4. Configure environment variables for Firebase configuration in src/fireadminapp.ts and src/firestore.ts. 5. Set up Google APIs credentials for email processing capabilities. 6. Configure nodemailer settings for email functionality. 7. Build the project using 'npm run build' which compiles TypeScript to lib/ directory. 8. Deploy Firebase Functions or run locally using 'npm start' which executes the job processing system. 9. Set up scheduled jobs by configuring the functions in src/features/functions/ directory. 10. Initialize the database schema using the model definitions and ensure proper indexes are created for query performance.
