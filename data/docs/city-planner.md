# city-planner

> build a city based on the redquickbuilder graphs

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

City Planner is a TypeScript-based computational geometry library that generates procedural city layouts using Voronoi diagrams and bin packing algorithms. The project processes hierarchical section data from redquickbuilder graph outputs to create realistic urban planning layouts with proper spatial distribution and collision detection. It combines multiple packing strategies including shelf packing, potpack algorithms, and Voronoi tessellation to organize city sections into efficient 2D layouts that can be exported for visualization or further processing in tools like Blender.

## Architecture

The architecture centers around the CityPlanner class (src/city-planner.ts) which orchestrates the entire layout generation process. It loads section data from JSON files, calculates spatial distributions using mathematical algorithms, and generates city layouts through a multi-level approach. The CityLayout class (src/citylayout.ts) handles rectangle placement and collision detection, while ShelfPack (src/shelfpack.ts) and PotPack (src/potpack.ts) provide different bin packing strategies. The system uses Voronoi diagrams for organic space division and includes Python integration (python/ directory) for Blender visualization. The main entry point (src/index.ts) creates a 200,000x200,000 unit city and exports the results to JSON files for downstream processing.

## Key Features

- Hierarchical section-based city planning with percentage-based area distribution
- Multi-level bin packing using shelf-pack and potpack algorithms for optimal space utilization
- Voronoi diagram generation for organic city district boundaries and realistic spatial division
- Collision detection and automatic rectangle repositioning in CityLayout class
- Progress tracking with visual CLI progress bars during city generation
- Python/Blender integration for 3D visualization with building extrusion and scaling tools
- Comprehensive test suite covering layout algorithms, collision detection, and city planning logic
- JSON-based input/output pipeline supporting large-scale city data (927K+ lines of JSON)
- Auto-resizing sprite allocation system for dynamic layout expansion

## Getting Started

1. Clone the repository and install dependencies with 'npm install' to get @mephistowa/point-in-poly, @mephistowa/voronoi, cli-progress, and colors packages. 2. Ensure you have a vis_out.json file in the root directory containing redquickbuilder graph output data. 3. Build the project using 'npm run build' which runs TypeScript compilation and pack-up.js version incrementing. 4. Run the city generation with 'npm run run' or 'node ./dist/index.js' to create a 200,000x200,000 unit city layout. 5. The system will output voronoi.json and nodes.json files containing the generated city data. 6. For testing, run 'npm test' to execute the Mocha test suite, or 'npm run coverage' for coverage reports with nyc. 7. For Python/Blender integration, use the scripts in the python/ directory with Blender 2.80+ to visualize and manipulate the generated city layouts in 3D.
