# local-image-server

![Language](https://img.shields.io/badge/language-JavaScript-blue)

## Overview

Local Image Server is a Node.js-based web server designed to manage and serve images locally, with specialized functionality for handling AI-generated images from Midjourney. The application provides image storage, retrieval, search capabilities, and screen recording functionality through a RESTful API and WebSocket connections. It serves as a local image management system that can filter images by prompts, store job information, and handle image uploads with automatic file processing using the Sharp library for image metadata extraction.

## Architecture

The application follows a modular Express.js server architecture with the main server logic in app.js, complemented by specialized modules for screen recording (screenrecord.js), WebSocket communication (websocket.js), and record management (record.js). The server uses file-based storage with .lmfr (custom format) files for job status tracking and configuration records. The system implements a job queue mechanism for processing image metadata asynchronously and maintains a local directory structure for image storage with platform-specific paths for macOS and Windows.

## Tech Stack

- Node.js
- Express.js
- WebSocket (ws)
- Sharp (image processing)
- FFmpeg (screen recording)
- File system operations
- HTTP/HTTPS client requests

## Key Features

- RESTful API for image management with endpoints for /images, /bigimages, /store-images, and /image-search
- Image filtering and search functionality based on prompt keywords and metadata
- Automatic image downloading and storage from URLs with filename sanitization
- Screen recording capabilities using FFmpeg with configurable resolution, FPS, and codec options
- WebSocket server for real-time communication and status updates
- Job queue system for processing image metadata and status tracking
- File upload handling with express-fileupload middleware
- Cross-platform directory management for macOS and Windows
- Image metadata extraction using Sharp library to identify large images (>=2000x2000)
- Configuration management for clothing/avatar customization with persistent storage

## Getting Started

1. Install Node.js and npm on your system. 2. Clone the repository and navigate to the project directory. 3. Run 'npm install' to install dependencies including express, sharp, ws, and other required packages. 4. Ensure FFmpeg is installed on your system for screen recording functionality. 5. Create the target image directory: '/Users/andrewporter/Downloads/midjourneyimages' on macOS or 'F:/midjourneyimages' on Windows, or modify the directory path in app.js. 6. Create an 'uploads' directory in the project root for file uploads. 7. Start the server with 'node app.js' - the server will run on the default Express port. 8. The WebSocket server and HTTP server will start automatically, and the application will create .lmfr files for job status and record tracking as needed.
