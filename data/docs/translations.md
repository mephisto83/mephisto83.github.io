# translations

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

This is a TypeScript-based translation automation tool designed to process text translations using Google Translate's unofficial API. The project reads new text entries from a JSON file, automatically translates them into multiple target languages, and updates an existing translations database while maintaining translation memory for performance optimization. The tool is specifically configured to work with an 'expressivefeeling' project, handling categorized text translations with built-in rate limiting and caching mechanisms.

## Architecture

The application follows a simple script-based architecture centered around the main script.ts file. It implements a translation memory system using an in-memory cache to avoid redundant API calls, a TranslationService function that interfaces with Google Translate's API, and a processNewTexts function that orchestrates the translation workflow. The system reads from two JSON files: titlesnew.json containing new texts to translate, and titles.json serving as the persistent translation database. The architecture employs sequential processing with artificial delays to respect API rate limits.

## Tech Stack

- TypeScript
- Node.js
- node-fetch
- Google Translate API (unofficial)
- JSON file storage
- File System operations

## Key Features

- Translation memory caching to avoid duplicate API calls and improve performance
- Multi-language support with automatic detection of target languages from existing translations
- Rate limiting with 100ms delays between API calls to prevent service throttling
- Categorized text organization allowing structured translation management
- Automatic fallback to default language set (en, fr, es, de, no, ko) when no existing translations exist
- Persistent storage using JSON files for both input texts and translation database
- Error handling with graceful degradation returning empty strings on API failures

## Getting Started

1. Clone the repository and navigate to the project directory. 2. Install dependencies using 'npm install' to get node-fetch, TypeScript, and type definitions. 3. Ensure TypeScript is compiled by running 'tsc script.ts' or configure your IDE for TypeScript compilation. 4. Create or verify the existence of input files: 'C:\dev\expressivefeeling\titlesnew.json' containing an array of NewText objects with 'text' and 'category' properties, and 'C:\dev\expressivefeeling\titles.json' as the translations database (can be empty initially). 5. Update the file paths in script.ts (translations_json and new_texts variables) to match your local directory structure. 6. Run the script using 'node script.js' after compilation. 7. Monitor the console output for translation progress and any API errors. The script will automatically update the translations database with new translations.
