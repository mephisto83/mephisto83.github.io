# redquickdistribution

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

RedQuickDistribution is a distributed computing framework built in TypeScript that enables automatic job distribution across multiple networked agents. The system creates a network of worker nodes that can coordinate task execution through a combination of Git-based state synchronization and TCP-based file transfer. It appears to be designed for scenarios where computational work needs to be distributed across multiple machines in a decentralized manner, with each agent automatically discovering and claiming work units while maintaining state consistency through version control.

## Architecture

The system follows a distributed agent-based architecture where each node runs independently and coordinates through Git repositories. The core Distribution class (src/distribution.ts) manages local agent lifecycle, while FileMirror (src/fileMirror.ts) handles network communication and file transfers between agents. Each agent spawns multiple child processes using the ChildProcess class (src/childprocess.ts) to handle parallel work execution. The system uses a hierarchical directory structure under /agents/{hostname_platform}/{projectname}/{job_id}/ to organize work units, with JSON configuration files tracking job states. Network communication occurs through HTTP servers for coordination and raw TCP sockets for efficient file transfers, with dynamic port allocation managed by the FileMirror component.

## Key Features

- Multi-agent distributed computing with automatic agent discovery and coordination
- Git-based state synchronization for maintaining consistency across distributed nodes
- Dynamic child process spawning with configurable thread counts based on CPU cores
- TCP-based efficient file transfer system with dynamic port allocation
- Hierarchical job organization with unique agent identification using hostname and platform
- Automatic work distribution and load balancing across available agents
- Cross-platform support with platform-specific agent naming (linux/win32)
- Fault tolerance through periodic state updates and error handling in child processes

## Getting Started

1. Install dependencies: npm install (requires node-fetch and typescript). 2. Build the TypeScript code: npm run build. 3. Configure the entry point by updating the entryPath in index.js to point to your worker thread implementation. 4. Start the distribution system: node index.js (this will initialize agents, create the Git repository structure, and begin work coordination). 5. For file mirroring capabilities, run: node server.js to start the HTTP/TCP servers for inter-agent communication. 6. The system will automatically create agent directories under ./agents/{hostname}_{platform}/ and synchronize state through Git commits and pulls. 7. Ensure Git is configured in the working directory for distributed state management to function properly.
