## Overview

MEPH is a comprehensive JavaScript framework for building mobile-first web applications with a focus on modular component architecture and data binding. The framework provides a complete MVC/MVVM pattern implementation with custom controls, template system, dependency injection (IOC), and a sophisticated binding mechanism. Built around the concept of reusable custom controls and activities, MEPH enables developers to create enterprise-level applications with clean separation of concerns through its presenter-view-controller-model architecture.

## Architecture

MEPH follows a modular component-based architecture where everything is treated as a custom control. The core architecture includes: a class system with inheritance and mixins (MEPH.define), a template engine with HTML binding and injection capabilities, an IOC container for dependency management (MobileServices), activity containers that manage view lifecycles, providers for data and view configuration (MobileViewProvider, MenuProvider), and a binding system supporting multiple contexts (c$, m$, v$, ct$, p$). The framework uses a path-based module loading system where classes are organized by namespace and loaded on-demand.

## Key Features

- Custom control system with alias-based HTML tags and template injection
- Advanced data binding with pipe operations and multiple context references (model, view, controller, presenter)
- Comprehensive IOC container with static and dynamic instance management
- Template inheritance and extension system with instruction-based injection points
- Mobile-optimized application framework with activity-based navigation
- Session management with configurable authentication flows
- Provider pattern for views, menus, and data sources
- Mixin system for shared functionality across classes
- Path-based module loading and namespace management
- Built-in observable pattern for property change notifications

## Getting Started

1. Include the core framework files: meph.js, Promise.js polyfill, and meph-debug.css in your HTML. 2. Create an app.js file and initialize the framework with u4mFrameWork('MEPH', path). 3. Set up application paths using MEPH.setPath() for your modules and resources. 4. Wait for framework ready with MEPH.ready().then() and create a MobileApplication instance. 5. Configure the IOC container with providers for viewProvider (MobileViewProvider), menuProvider (MenuProvider), and sessionManager. 6. Set up your JSON configuration files (Views.json, Menu.json) in the data directory. 7. Create your view components following the namespace/module/view structure with corresponding .js and .html files. 8. Define views using MEPH.define() with alias, templates, and extend properties. 9. Use the dev-harness examples as reference for implementing buttons, forms, communication, and other UI components.
