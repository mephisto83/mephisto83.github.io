## Overview

PlayWatch is a comprehensive TypeScript-based application for managing social media accounts and content. The project implements a feature-rich platform that allows users to track, process, and manage various social media accounts through a structured data model. Built with a modular architecture, it provides both frontend React components and backend Firebase Cloud Functions for complete social media account lifecycle management, including creation, reading, updating, and deletion operations.

## Architecture

The application follows a layered architecture with distinct separation between frontend features and backend functions. The frontend is organized around feature modules (playwatch-code/features/) where each entity like socialMediaAccount and processedSocialMediaAccount has its own complete module containing models, services, slices, thunks, validation, and property definitions. The backend functions (playwatch-code/functions/) mirror this structure with corresponding service modules. Data persistence is handled through Firebase Firestore with user-scoped collections using the pattern 'socialMediaAccounts/{owner}/socialMediaAccount'. State management is implemented using Redux Toolkit with async thunks for API operations, and the entire system uses TypeScript interfaces for type safety across models and services.

## Key Features

- Social Media Account Management with CRUD operations supporting link tracking and social media type categorization
- Processed Social Media Account handling for advanced account processing workflows with owner-based access control
- Firebase Firestore integration with automatic timestamp management for created/updated fields and version control
- Redux state management with async thunks for LoadSocialMediaAccountsAsync, CreateSocialMediaAccountAsync, and DeleteSocialMediaAccountAsync operations
- User-scoped data access with Firebase security rules enforcing owner-based permissions (signedIn() && userId == request.auth.uid)
- Comprehensive validation system with enumeration rules for socialMediaType fields and property-based validation schemas
- Modular feature architecture with standardized patterns across models, services, slices, thunks, and property definitions
- Auto-generated controllers and property definitions supporting rapid development of new entity types

## Getting Started

1. Install dependencies using npm install in the project root. 2. Configure Firebase project credentials and initialize Firebase Admin SDK in playwatch-code/features/firebase-app.ts. 3. Deploy Firebase security rules from *.rules files to enable proper user authentication and data access controls. 4. Set up Firebase Firestore database with collections following the pattern 'socialMediaAccounts/{userId}/socialMediaAccount' and 'processedSocialMediaAccounts/{userId}/processedSocialMediaAccount'. 5. Configure Redux store to include socialMediaAccountSlice and processedSocialMediaAccountSlice reducers. 6. Build the TypeScript project using tsc or your preferred build tool. 7. Deploy Firebase Cloud Functions from the playwatch-code/functions directory. 8. Start the Express.js server and ensure all controller routes in playwatch-code/controllers/ are properly registered.
