## Overview

This repository appears to be a personal GitHub Pages site for mephisto83, serving as a comprehensive showcase and experimentation platform for web-based music and development tools. The primary component is 'Composer Companion SS', a sophisticated web-based music composition application that provides interactive musical instrument interfaces, MIDI playback capabilities, and music notation rendering. The repository also contains various web development experiments, testing frameworks, and utility applications, demonstrating expertise in JavaScript-based audio programming, music theory implementation, and modern web technologies.

## Architecture

The project follows a modular, client-side web application architecture centered around the composer-companion-ss directory. The core architecture includes: (1) Audio Engine - built on Web Audio API with MIDI.js for sound synthesis and playback, located in composer-companion-ss/Midi/trunk and related audio files, (2) Music Notation Rendering - using VexFlow library (composer-companion-ss/Vexflow/trunk) and a custom ScoreLibrary (composer-companion-ss/score-library) for MusicXML processing, (3) Instrument Interfaces - individual JavaScript modules for different instruments (piano.js, guitar.js, violin.js, etc.) providing interactive UI components, (4) Plugin System - extensive CodeMirror integration for code editing (composer-companion-ss/plugins/codemirror) and Summernote for rich text editing, (5) Progress Loading System - Pace.js themes for user experience enhancement. The application uses a traditional multi-page structure with index.html as the entry point, CSS for styling, and modular JavaScript for functionality.

## Key Features

- Interactive virtual instruments including piano, guitar, violin, drums, and exotic instruments like kalimba and xylophone with realistic audio synthesis
- MIDI file import/export and playback capabilities with Web Audio API integration for cross-browser audio support
- Music notation rendering using VexFlow library with support for complex musical symbols, accidentals, and staff notation
- MusicXML processing through custom ScoreLibrary implementation for importing and displaying sheet music
- Rich code editing environment with CodeMirror supporting multiple programming languages including JavaScript, Python, CSS, and specialized formats like Velocity templates
- Real-time audio visualization using SiriWave.js for waveform display during audio input/output
- Comprehensive metronome functionality with web worker implementation for precise timing
- Microphone input processing for audio recording and analysis capabilities
- Responsive web design with Bootstrap integration and custom SCSS styling
- Plugin architecture supporting various UI enhancements including progress indicators, social media integration, and specialized text editing tools

## Getting Started

1. Clone the repository: git clone https://github.com/mephisto83/mephisto83.github.io.git 2. Navigate to the repository directory: cd mephisto83.github.io 3. For local development, serve the files using a web server (required for MIDI and audio file loading): python -m http.server 8000 or use Live Server in VS Code 4. Open browser to http://localhost:8000/composer-companion-ss/ to access the main Composer Companion application 5. For Docker deployment, use the included Dockerfile: docker build -t mephisto83-site . && docker run -p 8080:80 mephisto83-site 6. To build documentation, run the provided build scripts: build_doc.bat (Windows) 7. For MIDI functionality, ensure browser supports Web Audio API (modern Chrome, Firefox, Safari) 8. Audio files and MIDI soundfonts will load automatically from the Midi/trunk directory when accessing instrument interfaces
