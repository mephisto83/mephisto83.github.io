# hudl-up

![Language](https://img.shields.io/badge/language-JavaScript-blue)

## Overview

hudl-up is a specialized Node.js automation tool designed to scrape and organize football defensive playbook data from Hudl.com. The application uses Puppeteer to programmatically log into Hudl, navigate through defensive plays, capture screenshots of play diagrams, and extract structured assignment data including player positions, alignments, run responsibilities, and pass responsibilities. The tool then processes this data to generate organized PowerPoint presentations grouped by defensive schemes, making it easier for coaches to analyze and present defensive strategies.

## Architecture

The application follows a simple pipeline architecture with two main components: index.js handles web scraping and data extraction using Puppeteer for browser automation, while pptx.js processes the collected data to generate PowerPoint presentations using the nodejs-pptx library. The workflow involves automated login to Hudl, systematic navigation through 520+ defensive plays, screenshot capture of play diagrams, JSON data extraction of player assignments, and finally PowerPoint generation grouped by defensive play types. All captured data is stored in a /diagram directory with paired JSON and PNG files for each play, following a naming convention of 'PLAY_NAME vs FORMATION.json/png'.

## Key Features

- Automated Hudl login and navigation through defensive playbook interface
- Screenshot capture of 520+ football play diagrams stored as PNG files in /diagram directory
- Structured data extraction of player assignments including position, alignment, run responsibility, and pass responsibility stored as JSON
- Automatic grouping of plays by defensive scheme type (DIME, ARROW, BEAR, BENCH, etc.) based on filename parsing
- PowerPoint presentation generation with play diagrams and formatted assignment tables organized by defensive concepts
- Comprehensive play coverage including multiple formations like DOUBLES, TWINS, TROOPS, EMPTY, TOM, TREY against various defensive schemes

## Getting Started

1. Clone the repository and navigate to the project directory. 2. Run 'npm install' to install dependencies (nodejs-pptx and puppeteer). 3. Create a /diagram directory in the project root for storing scraped data. 4. Create a /pp directory for generated PowerPoint files. 5. Update the credentials in index.js (currently hardcoded as mephisto8335@gmail.com) with valid Hudl login information. 6. Verify the starting URL in index.js points to the correct Hudl playbook (currently set to defense playbook ID 49428). 7. Run 'node index.js' to start the scraping process - this will take significant time as it processes 520+ plays with 1-second delays. 8. After scraping completes, run 'node pptx.js' to generate PowerPoint presentations grouped by defensive schemes. 9. Check the /pp directory for generated PPTX files organized by defensive play types.
