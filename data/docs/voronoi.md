# voronoi

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

This is a TypeScript implementation of Fortune's algorithm for computing Voronoi diagrams, based on Raymond Hill's JavaScript Voronoi library. The project provides a computational geometry solution that partitions a plane into regions based on distance to a specific set of points (sites). Each region corresponds to one site, and all points within a region are closer to that site than to any other site. The library is published as @mephistowa/voronoi on npm and provides a clean TypeScript API for generating Voronoi diagrams with customizable bounding boxes.

## Architecture

The codebase follows an object-oriented design with clear separation of geometric concepts. The main entry point is src/index.ts which exports the Voronoi class from src/voronoi.ts. The core algorithm is implemented in src/voronoi-core.ts and utilizes a red-black tree data structure (src/rbtree.ts) for efficient beachline management. Key geometric entities are modeled as separate classes: Cell represents Voronoi cells, Edge represents diagram edges, HalfEdge represents directed edge segments, Vertex represents intersection points, and Site represents input points. The Diagram class (src/diagram.ts) serves as the container for the computed result. BeachSection and CircleEvent classes extend RBTree to manage the sweepline algorithm's data structures. A pack-up.js script handles automatic version bumping during the build process.

## Key Features

- Computes Voronoi diagrams using Fortune's sweepline algorithm with O(n log n) time complexity
- Provides TypeScript type safety with comprehensive geometric entity modeling
- Supports bounded diagram generation with customizable bounding rectangles (bbox)
- Offers neighbor detection capabilities through Cell.getNeighborIds() method
- Includes point-in-polygon testing via Cell.pointIntersection() for spatial queries
- Provides bounding box calculation for individual cells through Cell.getBbox()
- Implements proper halfedge sorting and cleanup in Cell.prepareHalfedges()
- Maintains geometric relationships between sites, cells, edges, and vertices
- Supports clockwise and counterclockwise edge traversal through HalfEdge start/end point methods

## Getting Started

1. Clone the repository and run 'npm install' to install TypeScript devDependencies. 2. Build the project using 'npm run build' which compiles TypeScript files to dist/ directory and runs pack-up.js for version management. 3. For development, use 'npm run watch' to enable TypeScript compilation in watch mode. 4. Test the implementation by running 'npm run try' which executes test.js, or examine the usage example in index.js. 5. The main API is accessed by importing Voronoi from the compiled dist/voronoi.js file, creating a new Voronoi instance, and calling compute(sites, bbox) where sites is an array of {x, y} objects and bbox defines the clipping rectangle with {xl, xr, yt, yb} properties. 6. To publish updates, use 'npm run push' which builds and publishes to npm with automatic version incrementing.
