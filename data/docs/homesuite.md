## Overview

HomeSuite is a comprehensive multi-platform application suite designed for managing agencies, agents, and social media content creation workflows. The system appears to be built around Firebase backend services with a focus on automated content generation, particularly integrating with Midjourney for AI-generated imagery and social media posting capabilities. The project implements a sophisticated microservices architecture with shared models and business logic across web, mobile, and server environments.

## Architecture

The system follows a distributed microservices architecture with multiple deployment targets: Firebase Functions (funcs/), standalone Express server (server/), web applications (web/, web-local/), mobile app (localcode/mobile/), and various utility services. The architecture centers around shared TypeScript models and controllers that are replicated across different environments (funcs/, server/, localcode/maestros/). Key components include auto-generated API controllers for agencies and agents management, Firebase Firestore for data persistence, Redux Toolkit for state management in frontend applications, and a job processing system for handling Midjourney image generation and social media posting workflows. The codebase demonstrates heavy code generation patterns with identical controller implementations across different runtime environments.

## Getting Started

1. Install Node.js and npm dependencies in each subdirectory (funcs/functions/, server/, web/, web-local/, localcode/). 2. Set up Firebase project and configure Firebase Admin SDK credentials. 3. Deploy Firestore security rules from localcode/maestro-tests/tests/firestore.rules. 4. Configure local settings file for job processing (referenced in server/src/services/start_social_job.ts). 5. Install Python dependencies from py-write/requirements.txt for image processing capabilities. 6. Build TypeScript projects using 'npm run build' in each service directory. 7. Start Firebase Functions with 'firebase deploy --only functions' from funcs/ directory. 8. Start standalone server with 'npm run s' from server/ directory. 9. Launch web applications with 'npm start' from web/ or web-local/ directories. 10. Configure Midjourney integration and social media API credentials for content automation features.
