# educingo

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

Educingo is a React Native educational mobile application that provides an interactive quiz-based learning platform with gamification elements. The app features multiple academic courses including Chemistry, Economics, Criminal Justice, Sociology, and financial certification series (Series 6 & 7). Built using Expo framework, the application delivers personalized learning experiences with animated mascot characters, progress tracking, and celebratory feedback mechanisms to enhance student engagement and retention.

## Architecture

The application follows a modular React Native architecture with Redux state management and Firebase backend integration. The main entry point is App.tsx which implements a ViewProvider context for navigation between Home and Questions modules. Core components are organized into reusable UI elements (buttons, animations, characters) while course-specific logic is handled through dedicated modules like MultipleChoiceModule. The app uses a custom component library with animated elements built on react-native-reanimated, and implements LaTeX rendering for mathematical expressions. Asset management is centralized through assets.ts with categorized resources for different courses and character variations.

## Getting Started

1. Clone the repository and navigate to the educingo-rn directory. 2. Install dependencies using 'npm install' or 'yarn install'. 3. Configure Firebase by setting up a Firebase project and adding configuration files. 4. Install Expo CLI globally with 'npm install -g expo-cli'. 5. Set up development environment following Expo setup guide for iOS/Android. 6. Generate course content by running 'node courses/generate.js' to populate question databases. 7. Start the development server with 'expo start' and scan QR code with Expo Go app or run on simulator. 8. For production builds, configure app.json with appropriate bundle identifiers and build using 'expo build:android' or 'expo build:ios'. 9. Ensure all required assets are present in the assets directory including character images, logos, and Lottie animations.
