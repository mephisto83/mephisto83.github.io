# kikswitchbridge

> KikSwitch bridge to 3rd party providers

![Language](https://img.shields.io/badge/language-C%23-blue)

## Overview

KikSwitch Bridge is a comprehensive insurance integration platform designed to facilitate seamless data exchange between KikSwitch and third-party insurance providers. The system serves as a middleware layer that captures, maps, and transforms insurance-related data from external provider websites, enabling automated quote retrieval and policy management for both car and home insurance products. Built on .NET Core 3.1, the platform implements a sophisticated field mapping system that can dynamically capture form field values from third-party insurance sites, store them in a database, and provide bidirectional translation between KikSwitch's internal data structures and external provider formats.

## Architecture

The system follows a service-oriented architecture with clear separation between controllers, models, and interfaces. The netcore/kikswitch.Controllers directory contains numerous controller classes for handling different insurance entities (ApplicantPetition, CarInsurance, HomeInsurance, etc.) with corresponding mapping controllers for field translation. Each entity type has three main components: a Get controller for data retrieval, a MappingGet controller for field mappings, and a mapping bridge controller for translation logic. The architecture includes stream processing orchestration components (CustomerStreamProcessOrchestration.cs) for handling complex data workflows, validation layers (CustomerValidations.cs), and exception handling mechanisms (CustomerExceptions.cs). The build system utilizes both PowerShell and JavaScript-based build scripts with Cake build framework integration.

## Key Features

- Dynamic field capture system for extracting form field values from third-party insurance provider websites
- Bidirectional mapping engine that translates between KikSwitch internal data formats and external provider schemas
- Comprehensive insurance domain coverage including car insurance quotes, home insurance policies, applicant petitions, and coverage options
- Automated quote retrieval system that inputs customer data into third-party interfaces and extracts results
- Real-time field monitoring to detect changes in third-party provider forms and update mappings accordingly
- Stream processing orchestration for handling complex multi-step insurance workflows
- Validation framework for ensuring data integrity across different provider integrations
- Support for multiple insurance entity types including deductibles, coverage options, property details, and driver information

## Getting Started

1. Ensure .NET Core 3.1 SDK is installed on your system. 2. Clone the repository and navigate to the netcore directory. 3. Install Node.js dependencies by running 'npm install' in the netcore directory (requires cheerio package). 4. Configure the workspace.json file with appropriate solution settings. 5. Run the PowerShell build script 'build.ps1' or use the Cake build system via 'build.cake'. 6. Restore NuGet packages for the kikswitch.Controllers project using 'dotnet restore'. 7. Build the solution using 'dotnet build' from the solution root. 8. Configure database connections and RedQuick framework settings in the application configuration files. 9. Set up mapping configurations for target third-party insurance providers. 10. Deploy the controllers and ensure proper routing is configured for all Get and Mapping endpoints.
