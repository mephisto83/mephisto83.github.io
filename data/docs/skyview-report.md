# skyview-report

> SkyView Mind map report

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

SkyView Mind Map Report is a React-TypeScript web application that serves as a comprehensive documentation and visualization tool for a proposed financial services platform. The project generates detailed mind map reports that specify user permissions, validations, and execution flows for different agent types including wealth managers, banking clients, end-user customers, and retail investors. This is positioned as Phase 0 deliverable of a multi-phase project aimed at creating an MVP that connects wealth managers, banks, end users, and third-party banking platforms.

## Architecture

The application follows a standard Create React App architecture with TypeScript. The main entry point is src/index.tsx which renders the App component. The project includes a custom build post-processing script (fixreport.js) that modifies the generated index.html to fix relative path references for GitHub Pages deployment. The application generates extensive visual documentation through hundreds of JPEG images stored in the img/ directory, each representing specific agent access patterns and permissions (following the naming convention agent-access-description-access-{resource}-by-{agent-type}.jpeg). Service worker functionality is implemented for offline capabilities with multiple precache manifests for different build versions.

## Getting Started

1. Clone the repository from https://github.com/mephisto83/skyview-report
2. Install dependencies: npm install
3. Start development server: npm start (launches on http://localhost:3000)
4. Run tests: npm test
5. Build for production: npm run build
6. For GitHub Pages deployment, run the fixreport.js script after building: node fixreport.js (this fixes relative paths in index.html)
7. The built application will be in the build/ directory, ready for deployment
8. Access the live version at https://mephisto83.github.io/skyview-report/
Note: The project includes extensive image assets in the img/ directory that are automatically included in the build process.
