# VrProject

![Language](https://img.shields.io/badge/language-C%2B%2B-blue)

## Overview

VrProject is a comprehensive Virtual Reality application built with Unreal Engine 4 that integrates web content visualization and real-time networking capabilities. The project combines UE4's VR framework with a custom socket.io-based server system to enable dynamic content streaming and interaction within VR environments. The core innovation lies in its 'webcutter' system that captures and segments web pages into interactive VR panes, allowing users to interact with web content in 3D space through motion controllers and hand tracking.

## Architecture

The architecture consists of three main layers: the UE4 VR client with motion controller support (VirtualRealityPawn, BP_MotionController), a Node.js/TypeScript backend server for web content processing and socket communication, and a sophisticated plugin system bridging UE4 and socket.io. The webcutter module uses Puppeteer for web scraping and content segmentation, while the SocketIOClient plugin handles real-time communication between the VR environment and external services. The system uses a pane-based approach where web content is divided into interactive sections that can be manipulated as 3D objects within the VR space.

## Key Features

- Motion controller-based VR interaction with hand animations (MannequinHand_Right_* assets)
- Real-time web content capture and VR visualization through webcutter system
- Dynamic pane segmentation of web pages with customizable sections and positioning
- Socket.io integration for bidirectional communication between VR client and web services
- Teleportation locomotion system with visual feedback (M_TeleportPreviews, BeamMesh)
- Haptic feedback integration through MotionControllerHaptics blueprint
- Multi-platform VR support (Windows, Linux, iOS, Mac, Android)
- Object pickup and manipulation system (BP_PickupCube, PickupActorInterface)
- CSS property monitoring and mutation observation for dynamic content updates
- Automated screenshot and content extraction with configurable viewport sizing

## Getting Started

1. Install Unreal Engine 4 with VR development components. 2. Clone the repository and open VrProject.uproject in UE4 Editor. 3. Ensure VR hardware drivers are installed and headset is connected. 4. Navigate to socketserver directory and run 'npm install' to install Node.js dependencies including puppeteer, socket.io, and express. 5. Configure DefaultEngine.ini and DefaultInput.ini for your specific VR hardware setup. 6. Build the SocketIOClient plugin by compiling the C++ modules (SocketIOClient.Build.cs, SocketIOLib.Build.cs). 7. Start the backend server with 'npm run go' from socketserver directory (builds TypeScript and starts on port 4999). 8. Launch the VR application using HMDLocomotionMap.umap as the starting level. 9. Configure webcutter target URLs and pane sections in src/index.ts for desired web content integration. 10. Test motion controller functionality and socket connectivity through the VirtualRealityPawn blueprint.
