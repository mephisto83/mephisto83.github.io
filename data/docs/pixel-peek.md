# pixel-peek

> Peek at pixel positions while viewing images.

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

Pixel-Peek is a comprehensive image analysis toolkit designed for examining pixel-level details in images and video frames. The project consists of multiple interconnected components that work together to extract, store, serve, and visualize image data with precise pixel inspection capabilities. It's particularly useful for computer vision workflows, video analysis, and detailed image examination tasks where users need to analyze specific pixel coordinates and regions within images.

## Architecture

The system follows a modular architecture with four distinct components: 1) pixel-extract - a Python-based video frame extraction utility using OpenCV that converts videos into individual JPEG frames, 2) pixel-gcs-uploader - a Node.js service that uploads large image datasets to Google Cloud Storage with MD5 hash verification and progress tracking, 3) pixel-serve - an Express.js backend server that serves images from local filesystem and manages project data storage, and 4) pixel-peek - a React TypeScript frontend application using BaseUI components that provides an interactive interface for browsing images, examining pixel coordinates, and managing image collections through a sidebar navigation and bucket-based organization system.

## Key Features

- Video frame extraction with OpenCV converting MP4 files to sequential JPEG images
- Bulk image upload to Google Cloud Storage with MD5 hash verification and duplicate detection
- Real-time pixel coordinate inspection and analysis in browser
- Hierarchical folder-based image organization and navigation
- Virtual scrolling for efficient handling of large image datasets
- Project state persistence with JSON-based bucket storage
- Cross-origin image serving with Express.js static file handling
- Interactive sidebar with dropdown navigation for folder structures
- Frame number extraction and display utilities
- Progress tracking for batch upload operations

## Getting Started

1. Clone the repository and navigate to each component directory. 2. For pixel-extract: Install Python dependencies (opencv-python) and modify the hardcoded paths in extract.py to your video file and output directories. 3. For pixel-gcs-uploader: Run 'npm install' then set GOOGLE_APPLICATION_CREDENTIALS environment variable to your GCS service account key path, update bucketName and directoryPath variables in upload_folder_to_gcs.js, then run 'npm start'. 4. For pixel-serve: Run 'npm install', update baseDir and folderDir paths in server.js to point to your image directories, then start with 'node server.js' on port 3001. 5. For pixel-peek frontend: Run 'npm install' then 'npm start' to launch the React application on port 3000. 6. Ensure the backend server is running before starting the frontend to enable image loading and project data management.
