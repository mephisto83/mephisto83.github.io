## Overview

DigitalAssistant is a comprehensive bot framework built on Microsoft Bot Framework and Azure services that provides conversational AI capabilities for enterprise applications. The system implements a multi-agent architecture where different specialized agents handle specific intents like salutations, order management, and travel requests through LUIS natural language understanding. The platform serves as a digital assistant service that can route conversations to appropriate handlers based on user intent, maintaining conversation state and providing integration with Azure Table Storage for persistence.

## Architecture

The system follows a layered architecture with the main AssistantService web API at the core, built using OWIN and ASP.NET Web API. The data layer (AssistantService.Data) defines core models like Agent, Conversation, and BotConfiguration, while the Azure-specific implementation (AssistantService.Data.Azure) provides table storage handlers for persistence. The architecture supports multiple bot configurations through BotChannelIdentity and ChannelInformation, allowing different bots to operate with distinct Microsoft App IDs. Integration tests (AssistantService.Deploy.IntegrationTests) validate LUIS intent recognition and bot communication. The system uses dependency injection with Autofac and includes Service Fabric deployment scripts for cloud scalability.

## Key Features

- Multi-agent conversation routing system with specialized handlers for different business domains
- LUIS-powered natural language understanding for intent classification including Salutation, Order Management, and Travel Request intents
- Azure Table Storage persistence for conversation history, active conversations, and bot configurations
- Support for multiple bot instances with separate Microsoft App IDs (default bot and Wendy bot)
- Conversation state management with ActiveConversationInfo tracking flow status, priority, and expiration
- RESTful API endpoints for user messages with Bearer token authentication
- Integration test suite for validating LUIS intent recognition and bot communication
- Service Fabric deployment support with PowerShell scripts for cloud orchestration
- Application Insights telemetry integration for monitoring and diagnostics

## Getting Started

1. Configure Azure resources: Create Azure Table Storage account, LUIS service, and Application Insights instance. 2. Update configuration files: Set MicrosoftAppId, MicrosoftAppPassword, LUIS subscription keys, and Azure storage connection strings in app.config and Web.config files. 3. Register bot applications in Microsoft Bot Framework portal and obtain app credentials. 4. Configure LUIS models with the required intents (Salutation, U4BW.Order, U4.TravelRequest.Agent) and train the models. 5. Deploy to Azure: Use the provided Service Fabric deployment scripts in SalutationAgent/U4.Bot.Agent.Salutation.SF/Scripts/Deploy-FabricApplication.ps1 or deploy as a web application. 6. Update agents.json with your specific agent configurations including queue names and product IDs. 7. Run integration tests by configuring test settings in AssistantService.Deploy.IntegrationTests with your deployed bot URL and credentials. 8. Verify deployment by sending test messages to the /api/usermessage endpoint with proper Bot Framework authentication headers.
