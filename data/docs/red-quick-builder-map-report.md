## Overview

Red Quick Builder Map Report is a React-based visual diagram builder and mapping application that specializes in creating and managing social media account data models and their relationships. The application appears to be a domain-specific modeling tool that generates interactive node-based diagrams for representing entities like Social Media Accounts, Controllers, Maestros, and various business logic components. Based on the extensive JSON configuration files and CSS styling resources, this tool serves as a comprehensive visual interface for building and documenting complex data relationships in what appears to be a social media management or mediation system.

## Architecture

The application follows a React-based single-page application architecture with TypeScript support. The core structure includes a React frontend built with Create React App, utilizing @projectstorm/react-diagrams for interactive node-based diagramming capabilities. The system is organized around JSON configuration files that define various entity types including models (SocialMediaAccount, ProcessedSocialMediaAccount), controllers (SocialMediaAccountController), maestros (SocialMediaAccountMaestro), methods, and enumerations (SocialMediaType). The CSS architecture leverages AdminLTE theme with extensive social media button styling through bootstrap-social integration. The application includes a custom build process (fixreport.js) that processes the React build output, suggesting deployment-specific optimizations. TypeScript declaration files (.d.ts) are generated for CSS modules, indicating a type-safe styling approach.

## Getting Started

1. Clone the repository and navigate to the project directory. 2. Install dependencies using 'npm install' or 'yarn install' to install all packages including React, TypeScript, and diagram libraries. 3. Start the development server with 'npm start' or 'yarn start' which will launch the application at http://localhost:3000. 4. For production builds, run 'npm run build' or 'yarn build' which executes the standard React build process followed by the custom fixreport.js script for deployment optimization. 5. The application will be served from the docs/ directory containing the built assets, CSS themes, and JSON configuration files. 6. Configure social media account types and relationships by modifying the JSON files in the public/img/ directory. 7. Customize UI themes by editing the AdminLTE CSS files in public/css/ or docs/css/ directories. 8. Run tests using 'npm test' or 'yarn test' to execute the Jest test suite.
