# comfy-ui-apainter-file-watch

![Language](https://img.shields.io/badge/language-JavaScript-blue)

## Overview

This repository implements a file-watching service for ComfyUI that integrates with APainter, a digital art application. The system monitors Firebase Realtime Database for layer changes and automatically processes image generation requests by managing file I/O operations, downloading source images, and coordinating with CivitAI's API for AI model and LoRA data. The application serves as a bridge between a web-based painting interface and ComfyUI's stable diffusion workflows, enabling real-time collaborative AI-assisted image generation with layer-based composition and model management.

## Architecture

The system follows an event-driven architecture centered around Firebase Realtime Database listeners in index.js that respond to layer changes. When changes occur, the system queues processing tasks per user, downloads source images to designated folders, writes configuration files for ComfyUI workflows (area_settings.txt, layer_output.txt, prompt files), and manages LoRA model assignments. The CivitAI integration modules (civitai.js, cimages.js) handle API interactions for fetching model metadata and images, while c_model.js processes local model inventories against CivitAI data to build compatible model lists stored in JSON format.

## Key Features

- Real-time Firebase database monitoring for layer changes with user-specific queuing system
- Automatic image downloading and file management for ComfyUI workflow integration
- CivitAI API integration for fetching SDXL models, LoRAs, and training data with filtering capabilities
- Dynamic configuration file generation including area_settings.txt, prompt files, and layer-specific outputs
- Session-based processing to handle concurrent user requests with userSessionKeys tracking
- Local model inventory management comparing installed files against CivitAI database
- Multi-layer support with up to 11 configurable layers and bounding box coordination
- LoRA model assignment system supporting multiple LoRAs per layer with index-based file naming

## Getting Started

1. Install dependencies with 'npm install' (requires firebase-admin package). 2. Place Firebase service account credentials in service_accounts/apainter-945f5-firebase-adminsdk-7rp71-b0d1f8a392.json. 3. Create the required directory structure: D:\apainter\layers with subdirectories 'images' and 'source_images'. 4. Ensure ComfyUI model directory exists at D:\sd.webui\webui\models\XLStableDiffusion for model scanning. 5. Configure the config object in index.js to match your local paths if different from defaults. 6. Run the service with 'npm start' or 'node index.js'. 7. The system will automatically connect to Firebase and begin monitoring for layer changes. 8. Use civitai.js to populate local model databases and c_model.js to sync with installed models.
