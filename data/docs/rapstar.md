# rapstar

![Language](https://img.shields.io/badge/language-Roff-blue)

## Overview

Rapstar is a sophisticated AI-powered system for rap music generation and linguistic analysis, combining machine learning models for phoneme prediction, pronunciation analysis, and rhyme generation. The project implements a comprehensive pipeline that processes textual input through multiple neural network architectures to generate phonetically accurate rap lyrics with proper syllable counting, rhyme matching, and pronunciation guidance. It leverages transformer-based models, MLPs, and custom attention mechanisms to analyze grapheme-to-phoneme relationships, making it capable of understanding the phonetic structure of words for creative rap composition.

## Architecture

The system follows a multi-layered architecture with separate Python modules for different linguistic tasks. The core pronunciation module (python/pronunciation/) contains transformer-based models with custom attention mechanisms, deep residual networks, and MLPs for grapheme-to-phoneme conversion. The phoneme module handles phonetic analysis using datasets like CMU Pronouncing Dictionary (cmudict-0.7b). A distributed scraping system (distributed/scraping/) enables parallel data collection, while Express.js servers provide API endpoints. Firebase integration handles data persistence, and the frontend uses TypeScript/JavaScript for user interaction. The system employs a TwoWayTransformer architecture with positional encoding, custom transformer encoder layers, and deep residual blocks for enhanced phonetic prediction accuracy.

## Key Features

- Advanced phoneme-to-grapheme conversion using transformer models with custom attention mechanisms in pronunciation_mlp_model.py
- Distributed web scraping system with master-worker architecture for data collection from rap lyrics and dictionary sources
- Real-time syllable counting and pronunciation analysis with configurable YAML-based model parameters
- Integration with multiple AI APIs (OpenAI, Groq) for natural language processing and rap generation
- Firebase-based data persistence with automated upload scripts for large-scale phonetic datasets
- Socket.io-enabled real-time communication for distributed processing and live rap generation
- Comprehensive phonetic mapping using English phoneme dictionaries and n-gram analysis
- Deep residual learning with configurable transformer depths and MLP dimensions for improved accuracy

## Getting Started

1. Install Python dependencies for ML models and Node.js dependencies with 'npm install'. 2. Configure environment variables in .env file for OpenAI/Groq API keys and Firebase credentials. 3. Set up YAML configuration files (pronunciation_config.yaml, phoneme_config.yaml) with desired model hyperparameters. 4. Initialize CMU dictionary data by running 'npm run build_data_set' to process cmudict-0.7b. 5. Train phoneme models using 'npm run train_syllable' or 'python -m python.syllable.train_syllable'. 6. Build pronunciation models with 'npm run build_pronunciation' to generate phonetic mappings. 7. Start the main server with 'npm start' or run distributed scraping with 'npm run scrape-master' and 'npm run scrape-worker'. 8. Upload data to Firebase using the firebase-upload-script/uploadToFirebase.js utility. 9. For development, use 'npm run build' to compile TypeScript and run individual modules like urbandictionary.js or merge.js as needed.
