## Overview

FieldForce is a React Native mobile application designed for field service management and operations. The app targets both Android and iOS platforms, providing field technicians with tools to manage service orders, track time spent on assignments, record equipment usage, and collect feedback through an intuitive mobile interface. The application leverages Redux for state management and includes NFC (Near Field Communication) capabilities for enhanced field operations, making it suitable for technicians who need to interact with NFC-enabled equipment or tags in the field.

## Architecture

The application follows a Redux-based architecture with a clear separation between native platform code and shared business logic. The main entry points are index.android.js and index.ios.js, both importing the same app.js component for code reuse. The Redux store is configured in native/android/app.js with fieldForceReducer managing application state. Navigation is handled through React Native's Navigator component with scene-based routing defined in fieldForceAppAndroid.js. The app structure includes dedicated folders for native components (native/components/), Android-specific code (android/), iOS-specific code (ios/), and shared services (src/service/). Key components include Assignment.js for task management, DashBoard.js for overview screens, ServiceOrderList for work order management, and custom UI components like FieldForceListView.js and BottomImageBtn.js.

## Getting Started

1. Clone the repository and navigate to the project directory. 2. Install dependencies by running 'npm install' to fetch all required packages including React Native, Redux, and testing frameworks. 3. For Android development, ensure Android SDK is installed and configure the android/ directory with proper build.gradle settings. Use the included my-release-key.keystore for app signing. 4. For iOS development, open ios/FieldForce.xcodeproj in Xcode and configure signing certificates. 5. Set up the development server by running 'npm start' to launch the React Native packager. 6. Configure the DEFAULT_URL in native/globals.js for your backend service endpoints. 7. For Android, run 'react-native run-android' or use 'gradlew' commands from the android directory. 8. For iOS, run 'react-native run-ios' or build directly from Xcode. 9. Run tests using 'npm test' which executes Mocha with Babel compilation and watch mode. 10. Ensure NFC permissions are properly configured in AndroidManifest.xml for Android NFC functionality.
