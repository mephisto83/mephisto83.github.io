## Overview

The rq-local-storage-service is a Node.js-based local storage service that provides WebSocket communication capabilities along with file system operations. It appears to be designed as a lightweight backend service that can handle real-time data exchange through WebSocket connections while providing utilities for file and folder management. The service combines Express.js for HTTP server functionality with custom WebSocket implementation for real-time communication, making it suitable for applications that require both persistent storage operations and live data synchronization.

## Architecture

The application follows a modular architecture with separate concerns handled by distinct files. The main entry point (app.js) likely orchestrates an Express server with CORS support and WebSocket integration via express-ws. The websocket.js module provides a custom WebSocket server implementation that manages multiple client connections, maintaining a connections array and providing broadcast capabilities to all connected clients. The copy.js module serves as a standalone utility for file system operations, handling recursive folder copying with command-line argument parsing for source/target directories and optional target clearing. The architecture supports concurrent WebSocket connections with unique ID assignment and error handling for disconnections.

## Getting Started

1. Clone the repository and navigate to the project directory. 2. Install dependencies using 'npm install' or 'yarn install' (both package-lock.json and yarn.lock are present). 3. Start the main application by running 'node app.js' which will initialize the Express server with WebSocket support. 4. The WebSocket server will be available for connections (default configuration suggests port 6506 based on test.js). 5. For file copying operations, use 'node copy.js --source <source_path> --target <target_path>' with optional '--clear-target' flag. 6. Test WebSocket connectivity using the provided test.js file with 'node test.js'. 7. Ensure proper network configuration for WebSocket connections as the server binds to all interfaces ('::').
