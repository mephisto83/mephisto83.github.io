# woodbury

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

Woodbury is a comprehensive AI-powered development platform that combines an interactive CLI coding assistant with specialized applications for workflow automation and social media management. The core system features an embedded agentic-loop engine with 14 built-in tools for file operations, shell execution, git management, web scraping, and database queries. The platform includes two main applications: a remote dashboard for pipeline management and workflow execution, and a social scheduler for AI-powered content generation and multi-platform posting automation.

## Architecture

The system follows a monorepo architecture with three main components: the core CLI tool in src/ with an embedded agentic-loop engine, a remote dashboard app built with Preact/Vite for pipeline management, and a social scheduler app using Next.js for content creation and scheduling. The core uses an extension system where custom tools, slash commands, and web UIs can be added via ~/.woodbury/extensions/. The remote app communicates with Woodbury instances through WebSocket relay connections and Firebase for persistence, while the social scheduler uses a file-based storage system in ~/.woodbury/social-scheduler/ with JSON data files and platform connectors for automation.

## Key Features

- Interactive REPL with 14 built-in tools for file operations, shell execution, git, web scraping, and database queries
- Extension system for custom tools, slash commands, and web dashboards via activate(ctx) pattern
- Multi-turn conversation memory with project context loading from .woodbury.md files
- Remote pipeline management dashboard with real-time status monitoring and WebSocket communication
- AI-powered social media content generation using multiple LLM providers (Anthropic, OpenAI, Groq)
- Multi-platform social media scheduling with browser automation through connector JSON configurations
- Calendar view for scheduled posts with platform-specific posting flows
- File-based storage system with JSON data persistence and connector management
- Safety modes and model selection with support for Claude, GPT, and Llama models

## Getting Started

1. Clone the repository and ensure Node.js 22+ is installed. 2. Run setup.bat on Windows or ./setup.sh on Mac/Linux, or manually run npm run setup. 3. Create ~/.agentic-loop/.env with API keys: ANTHROPIC_API_KEY, OPENAI_API_KEY, and GROQ_API_KEY. 4. For the social scheduler, the system will automatically create ~/.woodbury/social-scheduler/ directory structure with posts/, media/, connectors/, and config.json. 5. For the remote dashboard, configure Firebase settings in apps/remote/firebase.json and database.rules.json. 6. Use woodbury command for CLI interaction, npm run dev in apps/social-scheduler for the Next.js app on port 3001, or npm run dev in apps/remote for the Preact dashboard. 7. Add platform connectors by creating JSON files in ~/.woodbury/social-scheduler/connectors/ with posting flow configurations.
