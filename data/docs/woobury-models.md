# woobury-models

![Language](https://img.shields.io/badge/language-Python-blue)

## Overview

woobury-models is a specialized computer vision library for UI element verification in web browser automation. It implements a Siamese neural network architecture to learn robust embeddings of UI element screenshots, enabling reliable element matching across visual variations like theme changes, hover states, and rendering differences. The system is designed as a core component of the Woodbury browser automation platform, providing site-specific models that can identify the same UI elements even when their visual appearance changes due to CSS state transitions, resolution scaling, or browser rendering artifacts.

## Architecture

The system uses a dual-encoder Siamese architecture built around MobileNetV3-Small backbone producing 64-dimensional L2-normalized embeddings. The core components include: SiameseEncoder in model.py for embedding generation, specialized loss functions (NT-Xent, ArcFace, Triplet) in losses.py for metric learning, PK sampling strategy in dataset.py for balanced batch construction, and comprehensive UI-specific augmentations. The training pipeline flows from capture.py (data storage) → prepare.py (element cropping) → train.py (model training) → export.py (ONNX conversion) → inference.py (production matching). The distributed training system in worker.py enables remote GPU utilization with live progress streaming.

## Key Features

- Site-specific Siamese encoder training with MobileNetV3-Small backbone (2.5M params, <2ms inference)
- 12 UI-specific augmentations including theme inversion, state color shifts, focus rings, JPEG compression simulation
- PK batch sampling (32 identities × 4 samples) with NT-Xent contrastive loss at 0.10 temperature
- Multi-crop inference strategies (tight, standard, padded, context) for robust element matching via generate_strategy_crops()
- Distributed training system with worker.py enabling remote GPU utilization and live progress streaming
- Comprehensive evaluation metrics (ROC-AUC, EER, TAR@FMR) implemented in evaluate.py
- ONNX export with optional quantization for fast CPU inference in production
- Element matching with embedding caching in ElementMatcher and MultiCropMatcher classes
- Support for both CropStore (pre-cropped elements) and SnapshotStore (full-page screenshots) data formats

## Getting Started

1. Install with dependencies: `pip install -e ".[dev]"` 2. Prepare training data from Woodbury snapshots: `woobury-prepare --crops-per-element 8` (uses prepare.py to crop elements from full screenshots) 3. Train model: `woobury-train --config configs/default.yaml --json-progress --export-onnx` (trains SiameseEncoder with NT-Xent loss) 4. For distributed training, start workers on GPU machines: `woobury-worker --port 8677` 5. Configure training via YAML files in configs/ (default.yaml, efficientnet_b0.yaml, resnet18.yaml) 6. Use trained models via ElementMatcher or MultiCropMatcher classes in inference.py 7. Benchmark performance: use benchmark.py for latency testing of ONNX models
