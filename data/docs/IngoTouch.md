# IngoTouch

> Ingo Touch, paint the web

![Language](https://img.shields.io/badge/language-CSS-blue)

## Overview

IngoTouch is a mobile web application framework that provides a touch-based painting interface for web browsers. The project implements a mobile-first drawing application built on top of the MEPH framework, allowing users to create digital artwork with customizable brush settings, colors, and drawing tools. The application features a modular architecture with separate components for drawing surfaces, menus, application containers, and user session management.

## Architecture

The application follows a modular MVC-style architecture built on the MEPH framework. The Touch.application.Application serves as the main entry point, extending MEPH.mobile.Application. Key components include ApplicationContainer for the main UI shell, ApplicationHeader for navigation, and a drawing surface implemented through MEPH.pad.DrawingSurface. The architecture uses a dependency injection system for services like userService, implements an event-driven communication pattern through MEPH.subscribe/publish, and organizes components into logical modules (application, main, menu, provider, session, constant). CSS theming is handled through Sass compilation with Bootstrap integration, and the drawing functionality supports configurable properties like strokeWidth, lineWidth, shadowBlur, and color settings.

## Key Features

- Touch-based drawing interface with HTML5 Canvas support through MEPH.pad.DrawingSurface
- Customizable brush settings including stroke width, line width, miter limit, shadow blur and color
- Draggable UI panels with MEPH.util.Draggable for repositionable drawing tools and menus
- Color picker integration via MEPH.input.Color for dynamic brush color selection
- Drawing surface management with clear functionality and image export capabilities
- Responsive mobile application container with TouchApplicationContainer and ApplicationHeader
- Event-driven architecture using MEPH publish/subscribe pattern for component communication
- User session management with login/logout state handling
- Modular menu system with TouchMenuProvider for navigation
- Theme compilation system using Sass with Bootstrap integration

## Getting Started

1. Ensure Ruby is installed for Sass compilation. 2. Navigate to resources/sass/ directory and run 'sass --watch ingo-touch-debug.scss:../css/ingo-touch-debug.css' or execute GenerateThemes.bat. 3. Set up a web server to serve the application files since it uses MEPH framework modules. 4. Include the compiled CSS from resources/css/ingo-touch-base.css and ingo-touch-debug.css. 5. Load the main application by instantiating Touch.application.Application with appropriate configuration. 6. Ensure all MEPH framework dependencies are available and properly loaded before initializing the application. 7. Configure the drawing canvas dimensions and brush properties through the Touch.main.view.Main component's properties (height, width, strokeWidth, etc.).
