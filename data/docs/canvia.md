## Overview

Canvia is a multi-platform collaborative art application that enables real-time shared canvas editing across web and mobile devices. The project implements a monorepo architecture with workspace management, featuring separate React web and React Native mobile applications that share common Firebase backend infrastructure. The application focuses on providing a comprehensive digital art creation experience with real-time collaboration capabilities, AI-powered image generation, and cross-platform synchronization through Firebase Firestore and Authentication services.

## Architecture

The project follows a monorepo pattern with three main workspaces: web (React/Vite frontend), mobile (React Native/Expo app), and shared (common utilities). The web application uses Fabric.js for canvas manipulation and is built with Vite for fast development. The mobile app leverages React Native Skia for high-performance graphics rendering. Both platforms integrate with a shared Firebase backend (firebase.ts) that provides authentication, Firestore database for real-time synchronization, and cloud storage. The Canvas.tsx components in both platforms handle the core drawing functionality, while Auth.tsx manages user authentication flows. The AIGenerator.tsx component in the web app handles AI-powered image generation features.

## Getting Started

1. Clone the repository and navigate to the workspace directory. 2. Run 'npm install' in the root directory to install workspace dependencies and set up the monorepo structure. 3. Configure Firebase by updating the firebaseConfig objects in both shared/firebase.ts and web/src/firebase.ts with your actual Firebase project credentials. 4. For web development: Navigate to the web directory and run 'npm run dev' to start the Vite development server on http://localhost:5173. 5. For mobile development: Install Expo CLI globally, navigate to the mobile directory, and run 'npm start' to launch the Expo development server. 6. The web app will be accessible at localhost:5173 with hot reload, and the mobile app can be tested via Expo Go app or iOS/Android simulators using 'npm run ios' or 'npm run android' commands.
