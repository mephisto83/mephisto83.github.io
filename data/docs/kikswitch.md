# kikswitch

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

Kikswitch is a comprehensive insurance quote and policy management platform built as a React web application with Firebase backend integration. The system specializes in auto insurance policy management, providing end-to-end functionality for collecting customer information, managing household data, vehicle details, driver information, and accident/violation history. The platform appears to be designed to integrate with multiple insurance providers (including Safeco and Progressive based on the provider directories) to generate competitive insurance quotes and manage policy workflows.

## Architecture

The application follows a modern React architecture with Redux Toolkit for state management and Firebase for backend services. The frontend is structured as a single-page application (SPA) in the k2/kikswitch directory, with a separate Firebase Functions backend in the firebase directory. The codebase includes extensive data models for insurance-related entities like AccidentViolations, PolicyInformation, and HouseHoldSelection, each with their own Redux slices, services, and thunks for async operations. A separate kikswitch-work directory contains TypeScript utilities for web scraping and data processing with insurance providers, using Puppeteer for automated form filling and data extraction from provider websites.

- Comprehensive accident and violation history management through AccidentViolationsSelectionForm
- Dynamic form generation system using BuildForm utility with conditional field visibility
- Integration with multiple insurance providers (Safeco, Progressive) for quote comparison
- Real-time data persistence with Firebase Firestore including proper security rules
- Automated web scraping capabilities for insurance provider data extraction
- Responsive form components with dropdown selections for driver licenses, incident categories, and violation types
- Household and vehicle management with cross-referenced data relationships
- Policy information tracking with version control and audit trails

## Getting Started

1. Install dependencies for the main React app: `cd k2/kikswitch && npm install`. 2. Configure Firebase by setting up a project and updating firebase configuration in the firebase directory. 3. Deploy Firebase Functions: `cd firebase && npm install && npm run deploy`. 4. Set up Firebase Firestore with the security rules from accidentViolations-rules.rules and similar rule files. 5. For the web scraping functionality: `cd kikswitch-work && npm install && npm start`. 6. Start the React development server: `cd k2/kikswitch && npm start`. 7. The application will be available at http://localhost:3000 with Firebase backend integration. 8. For production deployment, run `npm run build` in the React app directory and deploy to Firebase Hosting using `firebase deploy --only hosting`.
