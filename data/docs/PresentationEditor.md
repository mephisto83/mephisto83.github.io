# PresentationEditor

> Presentation Editor

![Language](https://img.shields.io/badge/language-JavaScript-blue)

## Overview

PresentationEditor is a React-based web application for creating and editing multimedia presentations with timeline-based slide management. Built using a Redux state management architecture, it provides a comprehensive editing interface that supports multiple media types including images, videos, and audio with real-time collaboration capabilities through WebSocket connections. The application features drag-and-drop functionality, material libraries, font selection, and advanced rendering capabilities with usage tracking for render time management.

## Architecture

The application follows a React-Redux architecture with components organized in src/components/, actions in src/actions.js, and reducers in src/reducers. The main entry point is main.js which imports src/app.js, establishing a Provider-wrapped Redux store with thunk middleware. The core editing interface is built around SlideEditor component with supporting components like Timeline, MaterialLibrary, and various resource management dialogs. Communication with backend services is handled through src/service/serviceCaller with WebSocket support for real-time updates. The build system uses Browserify with Babelify for ES2015 and React JSX transformation.

## Getting Started

1. Clone the repository and run 'npm install' to install dependencies including React, Redux, and build tools. 2. Use 'npm run browserify' to build the application or 'npm run watchify' for development with auto-rebuilding. 3. Start a local server using 'npm run live-server' to serve index.html. 4. The application will be accessible in the browser, with the main React app mounting to the 'root' element in index.html. 5. For development, the bundle.js file is generated in the root directory containing the compiled React application. 6. Configure WebSocket domain settings in src/app.js for backend communication if deploying to production.
