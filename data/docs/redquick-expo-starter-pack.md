# redquick-expo-starter-pack

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

The redquick-expo-starter-pack is a React Native/Expo boilerplate project that provides a foundational mobile application framework with authentication capabilities and QR code functionality. This starter pack appears to be designed for rapid prototyping of mobile applications that require user authentication, deep linking, and QR code scanning features. The project emphasizes a modular architecture with proper separation of concerns through dedicated directories for components, navigation, screens, and services.

## Architecture

The application follows a standard React Native/Expo architecture with TypeScript. The core authentication system is built around a Firebase context (FirebaseContext.tsx) that manages user state, login/logout operations, and user claims. The auth system includes an AuthGuard.tsx component for route protection and an AutoSignInComponent.tsx for persistent authentication. Navigation is handled through React Navigation with deep linking configuration in LinkingConfiguration.ts, supporting screen routing and URL-based navigation. The app includes specialized screens like AutoSignInScreen.tsx and ConnectScreen.tsx, with a QRCodeReaderComponent.tsx for barcode scanning functionality. Local data persistence is managed through the storageService.ts using AsyncStorage for mobile credentials storage.

- Connection screen for device or service pairing
- Utility functions including UUID generation and object merging capabilities

## Getting Started

1. Clone the repository and navigate to the project directory. 2. Install dependencies using npm install or yarn install. 3. Configure Firebase authentication by setting up your Firebase project and updating the FirebaseContext.tsx with your Firebase configuration. 4. Install Expo CLI globally if not already installed: npm install -g expo-cli. 5. Modify package_modify.json according to your project requirements and merge it with package.json. 6. Update the deep linking configuration in navigation/LinkingConfiguration.ts to match your app's URL scheme. 7. Configure the babel.config.js file if you need additional module aliases beyond the existing ones (features, service, components). 8. Run expo start to launch the development server and test the application on a physical device or emulator. 9. Test the QR code functionality on a physical device as camera features require actual hardware.
