# UberRailEnvironments

> Uber Rail Environments

![Language](https://img.shields.io/badge/language-JavaScript-blue)

## Overview

UberRailEnvironments is a 3D environment asset management and compilation system that processes and packages virtual rail environments for rendering applications. The project serves as a content pipeline tool that takes individual environment assets (textures, materials, scenes, billboards) and compiles them into unified environment packages. Each environment variant (BlueWall, BrownWall, PurpleWall) contains complete asset sets including EXR/HDR textures, normal maps, displacement maps, specular maps, and video sequences that can be dynamically loaded into 3D applications.

## Architecture

The system follows a modular environment-based architecture where each environment directory (BlueWall/, BrownWall/, PurpleWall/) contains a complete asset package with standardized file structures. The core compilation engine (_compile.js) orchestrates the build process by reading config.json files from each environment, loading referenced JSON asset definitions (materials.json, groups.json, worlds.json, scene.json), and merging them into unified environment-compiled.json files. The compile.js entry point processes all environments and generates a master railenvironments.json manifest. Each environment supports both static billboard representations and full 3D scene data, with texture assets organized in nested subdirectories containing multiple format variants (EXR for HDR, medium JPG for preview, and specialized maps for displacement, normals, and specularity).

## Key Features

- Environment asset compilation system that merges JSON configuration files into unified packages
- Multi-format texture support including EXR HDR textures, displacement maps, normal maps, and specular maps
- Billboard rendering system with separate material and world definitions for optimized display
- Video sequence integration with color and mask channels (shot6-mask0001-0250.avi, shot7-color0001-0250.avi)
- Automated environment discovery and processing through directory scanning
- Master manifest generation (railenvironments.json) for centralized environment registry
- Standardized asset structure with config-driven compilation (config.json defines environment.json and related assets)
- Support for both compiled and source asset formats with environment-compiled.json output

## Getting Started

1. Clone the repository and ensure Node.js is installed on your system. 2. Navigate to the project root directory containing package.json and run 'npm install' to install dependencies. 3. Update the hardcoded path in compile.js from 'D:\UberRailEnvironments\' to your actual project directory path. 4. Ensure all environment directories (BlueWall/, BrownWall/, PurpleWall/) contain valid config.json files referencing their respective asset files. 5. Verify that all referenced texture files, video sequences, and JSON assets exist in their expected locations within each environment subdirectory. 6. Run 'node compile.js' to execute the compilation process, which will generate environment-compiled.json files in each environment directory and a master railenvironments.json file in the project root. 7. The compiled environments can then be loaded by external 3D applications that consume the generated JSON manifests and asset references.
