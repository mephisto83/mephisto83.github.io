# ReddSend

![Language](https://img.shields.io/badge/language-Objective-C-blue)

## Overview

ReddSend is a React Native mobile application designed for cryptocurrency portfolio tracking and management. The app provides a cross-platform solution for iOS and Android that allows users to monitor their cryptocurrency holdings, view price data, and take portfolio snapshots. Built on React Native 0.53.0, the application leverages the redquicklib framework for rapid development and includes Twitter integration for social features.

## Architecture

The application follows a Redux-based state management architecture with React Native Router Flux for navigation. The core structure includes a main App.js component that wraps the application in a RedApplication container, with RouteScreen.js serving as the primary screen component. The app uses a modular approach with separate directories for Android (android/) and iOS (ios/) platform-specific code. The iOS implementation includes TwitterCore.framework integration for social authentication, while the Android implementation uses standard Gradle build configuration. State management is handled through Redux with redux-thunk middleware, and the UI is built using NativeBase components for consistent cross-platform styling.

## Getting Started

1. Clone the repository and navigate to the reddsend directory. 2. Install Node.js dependencies by running 'npm install' in the root directory. 3. For iOS development: Open ios/reddsend.xcodeproj in Xcode, ensure TwitterCore.framework is properly linked, and configure signing certificates. 4. For Android development: Ensure Android SDK is installed, configure android/keystores/debug.keystore.properties, and run './gradlew assembleRelease' in the android directory. 5. Install React Native CLI globally with 'npm install -g react-native-cli'. 6. For iOS: Run 'react-native run-ios' or use Xcode to build and run. 7. For Android: Run 'react-native run-android' or use 'npm run android'. 8. Configure API keys and authentication credentials in the keys.js file as needed. 9. Test the application using 'npm test' to run Jest test suites.
