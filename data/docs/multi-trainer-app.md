# multi-trainer-app

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

Multi-Trainer App is a comprehensive fitness booking and management platform built with React and Firebase. It enables fitness trainers to create professional booking systems where clients can schedule sessions, manage appointments, and process payments through Stripe integration. The platform supports multiple trainers, each with their own customizable booking types, availability schedules, pricing plans, and subscription-based services. The application provides both client-facing booking interfaces and trainer dashboards for managing services, availability, and bookings.

## Architecture

The application follows a modern React architecture with Firebase as the backend-as-a-service. The frontend uses React with TypeScript, organized around context providers for state management (AuthContext, BookingTypeContext, PricingContext, TrainerContext). Firebase handles authentication, Firestore provides the database layer, and Cloud Functions manage server-side operations including Stripe payment processing and booking confirmations. The component structure separates concerns with dedicated components for booking flows (BookingModal, BookingCalendar), management interfaces (TrainerSettings, PricingPlanForm), and public-facing elements (Pricing, Services, Testimonials). The Firebase Functions in functions/src/index.ts handle payment processing, booking event management, and availability calculations.

## Key Features

- Multi-trainer booking system with individual trainer profiles and settings
- Flexible booking types with customizable duration, pricing, and availability schedules managed in BookingTypeContext
- Real-time availability calendar using react-big-calendar in BookingCalendar component
- Stripe payment integration with platform fees and trainer-specific accounts via createCheckoutSession function
- Subscription-based services with session tracking managed through useSubscriptions hook
- Comprehensive pricing plan management via PricingContext and PricingPlanForm components
- Client booking management with cancellation policies handled in CancellationModal
- Blog and testimonials system with markdown editor support using MarkdownEditor component
- Email notifications triggered by Firebase Functions on booking confirmations
- Dark/light theme support via ThemeSwitcher component
- Mobile-responsive design with SCSS styling
- Admin dashboard for trainers to manage services, availability, and view bookings through DashboardMenu

## Getting Started

1. Clone the repository and install dependencies with `npm install` and `cd functions && npm install`. 2. Set up Firebase project and enable Authentication, Firestore, Functions, and Hosting. 3. Configure environment variables by creating a .env file with Firebase config (REACT_APP_FIREBASE_API_KEY, REACT_APP_FIREBASE_AUTH_DOMAIN, etc.) and Stripe keys (STRIPE_SECRET_KEY, REACT_APP_STRIPE_PUBLISHABLE_KEY). 4. Set up Firestore security rules using firestore.rules and indexes with firestore.indexes.json. 5. Configure Stripe by running `node set-firebase-config.js` to set Firebase Functions environment variables. 6. Deploy Firebase Functions with `npm run functions:deploy` to enable payment processing and booking management. 7. Start development server with `npm start` for frontend and `npm run functions:serve` for local Functions emulator. 8. For production deployment, build and deploy with `npm run deploy:all` which builds the React app and deploys to Firebase Hosting and Functions.
