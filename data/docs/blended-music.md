# blended-music

![Language](https://img.shields.io/badge/language-JavaScript-blue)

## Overview

Blended-music is a sophisticated music visualization and animation system that converts MIDI files into procedurally generated 3D animations using Blender. The project combines music analysis with computational geometry to create visual representations of musical data through dynamic textures, materials, and geometric patterns. It serves as a creative tool for musicians, visual artists, and content creators who want to transform musical compositions into immersive visual experiences with automated video generation capabilities.

## Architecture

The system follows a modular pipeline architecture with four main components: MIDI processing (midi-convert/ directory handles parsing and conversion), visual generation (src/movie/ contains materials, textures, and geometric generators), automation layer (src/puppeteer/ and various test scripts manage batch processing), and integration services (Python scripts bridge Node.js with Blender's API). The core workflow processes MIDI files through MidiConvert.mjs, generates visual elements via materials like hex-force-field.mjs and voronoi.mjs, creates Blender scenes through Python templates, and outputs final videos using the video editor pipeline in anim_video_editor.py.

## Key Features

- MIDI to JSON conversion with comprehensive note and track analysis
- Procedural material generation including hex force fields, scatter materials, and tree root systems
- Voronoi diagram-based geometric pattern generation
- Automated Blender scene creation with camera positioning and lighting
- Batch processing of multiple MIDI files through the 'farm' system
- Integration with Google APIs for cloud-based operations
- Composite material blending and texture generation
- Race track and root system visualization generators
- YouTube presentation defense system for content management
- Graph layout algorithms for spatial arrangement of musical elements

## Getting Started

1. Install Node.js and ensure experimental modules support is enabled. 2. Add VLC media player to your system PATH environment variable. 3. Install Blender 2.80 and add the blender executable to your system PATH. 4. Clone the repository and run 'npm install' to install dependencies including @std/esm, googleapis, puppeteer, and other required packages. 5. Configure Google API credentials if using cloud features by setting up google-auth-library. 6. Test the installation by running 'npm run test' which executes the MIDI conversion pipeline. 7. For specific workflows, use commands like 'npm run midi-to-json' for conversion, 'npm run farm' for batch processing, or 'npm run voronoi' for geometric generation. 8. Ensure adequate system memory as some scripts require --max_old_space_size=8192 flag for processing large MIDI files.
