## Overview

redhash is a JavaScript implementation of the Hashgraph consensus algorithm, a distributed ledger technology that provides Byzantine fault tolerance through a gossip protocol and virtual voting mechanism. The project consists of two main components: a core hashgraph library that implements the consensus algorithm, and a web-based explorer application that provides visualization and interaction capabilities for hashgraph networks. The implementation appears to be an experimental interpretation of the Hashgraph whitepaper, offering a platform for understanding and testing distributed consensus mechanisms in a browser environment.

## Architecture

The project follows a modular architecture with two distinct packages: the core 'redhashgraph' library located in the hashgraph/ directory implements the fundamental hashgraph consensus algorithms and data structures, while the explorer/ directory contains a React-based web application that consumes the hashgraph library. The explorer uses Redux for state management and D3.js for data visualization, with Webpack handling the build process. The hashgraph core is transpiled from ES6 to ES5 using Babel, making it compatible with both Node.js and browser environments. The explorer application includes extensive UI components built with Materialize CSS framework and integrates rich text editing capabilities through CKEditor and TinyMCE plugins.

## Getting Started

1. Clone the repository and navigate to the hashgraph/ directory. 2. Run 'npm install' to install core dependencies including Babel, Browserify, and Mocha. 3. Execute 'npm run build' to transpile ES6 source code to the distribution/ directory. 4. Navigate to the explorer/ directory and run 'npm install' to install React, Redux, D3.js and other frontend dependencies. 5. Run 'npm run watchify' in the explorer directory to build the bundle.js file with live reload capabilities. 6. Open explorer/index.html in a web browser or serve it through a local web server. 7. For development, use 'npm run test' in the hashgraph directory to run the Mocha test suite, and 'npm run nodeserver' to start a Node.js server environment.
