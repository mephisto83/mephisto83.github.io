# icanfeelorg

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

icanfeelorg is a comprehensive Firebase-hosted content management and collaboration platform built with TypeScript and Express.js. The application provides a sophisticated backend architecture for managing organizations, artists, concept boards, and content creation workflows through Firebase Cloud Functions. It serves as a multi-tenant platform where organizations can manage content creators, prompt groups, titles, and collaborative concept boards with fine-grained permission controls and agent-based access patterns.

## Architecture

The application follows a modular serverless architecture built on Firebase Cloud Functions with Express.js routing. The core structure is organized into three main layers: controllers (functions/src/controllers/) for HTTP endpoint handling, services (functions/src/functions/) for business logic and data operations, and models (functions/src/models/) for data structures. The system implements an agent-based authorization pattern where organization agents authenticate using custom Firebase claims, and all operations are validated through identity_check.ts. Auto-generated controllers in the controllers/auto/ directory handle CRUD operations for entities like organizations and concept boards, while the features/ directory contains cross-cutting concerns like Firebase function attachments and identity validation. Data persistence is managed through Firestore with custom service layers for each entity type (artist-service.ts, organization-service.ts, etc.).

## Getting Started

1. Install Node.js 16 and Firebase CLI globally. 2. Clone the repository and run 'npm install' in the root directory. 3. Navigate to functions/ directory and run 'npm install' to install function dependencies. 4. Configure Firebase project by setting up firebase.json, firestore.rules, and firestore.indexes.json. 5. Set up Google Cloud credentials by placing the service account key file (referenced as icanfeel-7b1f4-firebase-adminsdk-fusf8-53a48be3d0.json in serve-all script). 6. Build the TypeScript code using 'npm run build' or 'tsc'. 7. For local development, use 'npm run serve-all' to start Firebase emulators with proper environment variables. 8. Deploy functions using 'npm run deploy' for functions only, 'npm run publish' for hosting only, or 'npm run all' for complete deployment. 9. Configure Firestore security rules using firestore.rules and set up indexes from firestore.indexes.json. 10. Run tests using 'npm test' or 'npm run test:watch' for continuous testing.
