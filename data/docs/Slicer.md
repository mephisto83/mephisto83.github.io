## Overview

Slicer is an Unreal Engine 4 game project that appears to be a music/rhythm game with lightsaber-like mechanics, similar to Beat Saber. The project combines 3D gameplay elements with a web-based UI system, featuring custom materials, sound effects, and interactive gameplay mechanics. The repository contains a complete UE4 project with Android build support, custom blueprints, audio assets, and a JavaScript-based UI composition system for menu screens.

## Architecture

The project follows Unreal Engine 4's standard architecture with a main Slicer/ directory containing the core game project. The system uses UE4's Blueprint system for game logic, with custom UserDefinedStruct objects like Specificity.uasset for data management. A separate JavaScript build system (build-composite.js) handles UI composition by merging multiple JSON configuration files for different menu screens. The project includes native Android build configurations in Slicer/Build/Android/ with proper APK generation setup, and uses UE4's asset pipeline for 3D models (.fbx files), audio (.mp3), and materials (.uasset files).

## Getting Started

1. Install Unreal Engine 4.26 or compatible version. 2. Clone the repository and open Slicer/Slicer.uproject in UE4 Editor. 3. For UI development, install Node.js and run 'node build-composite.js' to generate composite.json from individual menu JSON files. 4. For Android builds, ensure Android SDK is configured in UE4 and use the existing build configuration in Slicer/Build/Android/. 5. Import 3D assets by opening .blend files in Blender and re-exporting .fbx files if modifications are needed. 6. Configure input settings through DefaultInput.ini and engine settings via DefaultEngine.ini in Slicer/Config/. 7. Build and package using UE4's standard build pipeline, with pre-configured Android packaging settings already available.
