# faiss-lib

![Language](https://img.shields.io/badge/language-Python-blue)

## Overview

faiss-lib is a Python-based document processing and vector search library that leverages Facebook AI Similarity Search (FAISS) for creating and querying document embeddings. The library provides a unified interface for processing various document formats (PDF, CSV, TXT, Jupyter notebooks, Git repositories) and converting them into searchable vector representations using HuggingFace embeddings. It serves as a wrapper around LangChain's document processing capabilities and FAISS's vector storage, making it easy to build semantic search systems over heterogeneous document collections.

## Architecture

The system follows a modular pipeline architecture with three main components: document processing (lib.py), indexing orchestration (index.py), and search functionality (search.py). The lib.py module contains format-specific processors that handle document splitting using RecursiveCharacterTextSplitter with configurable chunk sizes and overlap. Documents are converted to embeddings using HuggingFaceEmbeddings (default: all-MiniLM-L6-v2 model) and stored in FAISS vector indices that can be persisted and merged. The search component loads pre-built indices and performs similarity searches using cosine similarity on the embedded vectors.

## Key Features

- Multi-format document processing supporting PDF, CSV, TXT, Jupyter notebooks (.ipynb), and Git repositories
- Configurable text chunking with RecursiveCharacterTextSplitter supporting custom chunk sizes and overlap parameters
- Vector index persistence and merging capabilities allowing incremental document addition to existing FAISS indices
- Semantic similarity search using HuggingFace embedding models with default all-MiniLM-L6-v2 model
- Metadata preservation for PDF documents including page numbers and source file information
- Performance timing utilities for benchmarking search operations
- Batch processing of entire directories with automatic file type detection and routing

## Getting Started

1. Install dependencies using 'pip install -r requirements.txt' which includes faiss-cpu, langchain, sentence-transformers, and other required packages. 2. Prepare your document collection in a folder structure - the default example uses 'D:\pdfs\single' for PDF files. 3. Configure processing options in index.py by setting the folder_path variable and options dictionary with index_store name and chunk_size (default 500). 4. Run document indexing with 'python index.py' to process all supported files in the specified folder and create a FAISS vector store. 5. Perform searches using search.py by loading the created index with build_search() function and querying with search() function. 6. Customize embedding models by modifying the model_name parameter in HuggingFaceEmbeddings initialization within the do_it() function.
