# red-quick-extensions

> RedQuick Extensions

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

Red Quick Extensions is a TypeScript-based React diagramming library that extends the projectstorm/react-diagrams framework. It provides a flow and process-oriented diagramming solution inspired by visual programming environments like Blender, Labview, and Unreal Engine. The library is designed for creating interactive, dynamic diagrams with custom nodes, links, and behaviors, making it suitable for building visual programming interfaces, flowcharts, and process diagrams. The project emphasizes extensibility and modern software engineering practices, built entirely in TypeScript with powerful generics and modular architecture.

## Architecture

The project follows a monorepo structure managed by Lerna with multiple packages in the packages/ directory. The core architecture is built on top of react-diagrams-core, react-canvas-core, and geometry libraries. Key components include DiagramEngine for managing the diagram state, various Model classes (NodeModel, PortModel, LinkModel) for data representation, Factory classes for creating instances, and Widget/Component classes for rendering. The demo gallery (diagrams-demo-gallery/) contains extensive examples showing custom nodes (DiamondNodeModel), custom links, alternative linking states (CreateLinkState, DefaultState), and advanced features like drag-and-drop, serialization, and custom behaviors. The architecture supports factory pattern registration, state management through action-based systems, and event handling through ActionEventBus.

## Key Features

- Custom node creation with DiamondNodeModel and factory pattern implementation supporting various port alignments and custom behaviors
- Alternative linking system with CreateLinkState and DefaultState classes providing advanced link creation and interaction modes
- Comprehensive demo gallery with 20+ examples including drag-and-drop, custom actions, animations, grid layouts, and performance optimizations
- Editable label system with EditableLabelModel supporting in-place text editing and serialization
- Advanced state management using Action-based architecture with mouse, keyboard, and touch event handling
- Drag-and-drop functionality with TrayWidget and KIKNodeModel for creating nodes from external components
- Serialization and deserialization support for saving and loading diagram configurations
- Smart routing and right-angle link routing capabilities for professional diagram layouts
- Performance optimization features for handling large diagrams with many nodes and connections
- Extensible factory system allowing registration of custom node, port, and link factories with the DiagramEngine

## Getting Started

1. Clone the repository and install dependencies using 'yarn install' 2. Build all packages by running 'yarn build' in the root directory (required for monorepo dependencies) 3. Navigate to the demo gallery with 'cd diagrams-demo-gallery' and run 'yarn start' to launch the Storybook development server 4. For production builds, use 'yarn build:prod' which sets NODE_ENV=production 5. To run tests, execute 'yarn test' which runs Jest tests across all packages using Lerna 6. For publishing, use 'yarn publish:dev' for development releases or 'yarn publish:prod' for production releases 7. The project uses TypeScript compilation with 'tsc --build' and supports code formatting with Prettier using 'yarn pretty' 8. Individual demo examples can be found in diagrams-demo-gallery/demos/ directory, each with their own index.tsx entry point
