# ball-simulation

> Ball Simulation written in concert with ChatGPT.

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

Ball Simulation is a React-based physics simulation application that models realistic ball motion with gravity, collision detection, and MIDI-driven timing events. The simulation features dynamic line generation at impact points, boundary collision handling, and visual animation using React Spring. Developed collaboratively with ChatGPT, this project demonstrates interactive physics simulations in web browsers with precise timing controls and realistic motion calculations including gravitational acceleration and velocity-based collision responses.

## Architecture

The application follows a modular React component architecture with TypeScript for type safety. The core simulation engine resides in util.ts with functions for distance calculations, collision detection, and motion physics. The main App.tsx component orchestrates the simulation while Ball.tsx and Line.tsx handle individual entity rendering. The Simulation.tsx component manages the physics loop and state updates. MidiReader.tsx provides external timing input capabilities. The system uses a frame-based animation approach with configurable frame rates, maintaining separate tracking for ball positions, impact points, and dynamically generated collision lines.

## Key Features

- Physics-based ball motion simulation with gravity acceleration (9.8 m/s²)
- Real-time collision detection with boundary walls and dynamic line obstacles
- MIDI file integration for controlling simulation timing events
- Animated ball trajectories with position tracking and velocity calculations
- Dynamic line generation at impact points with configurable length and positioning
- React Spring animations for smooth visual transitions
- Comprehensive type definitions for Point, Line, Ball, and Rectangle entities
- Frame-rate configurable simulation with precise time-step calculations
- Impact point visualization and collision response mechanics

## Getting Started

1. Clone the repository and navigate to the ball-simulation directory. 2. Install dependencies using 'npm install' which will install React, TypeScript, React Spring, Tone.js MIDI, and testing libraries. 3. Start the development server with 'npm start' to launch the application at http://localhost:3000. 4. Run tests using 'npm test' to execute the util.test.tsx test suite. 5. For production deployment, build the optimized bundle with 'npm run build'. The main simulation interface will be accessible through the App.tsx component which integrates the Ball.tsx, Line.tsx, Simulation.tsx, and MidiReader.tsx components for the complete physics simulation experience.
