## Overview

This is a multimedia processing and visualization application that combines MIDI audio analysis with Blender-based 3D movie generation capabilities. The project appears to be a React-Redux web application that analyzes MIDI files to extract musical metadata, visualizes chord progressions using Google Charts, and generates automated 3D animations synchronized to musical data using Blender scripts. The application serves as a bridge between musical composition analysis and automated visual content creation, with cloud storage integration through AWS and Azure services.

## Architecture

The application follows a modular architecture with distinct layers: a React-Redux frontend for data visualization and user interaction, a Node.js backend server (src/server.js) for API endpoints, and specialized processing modules for MIDI analysis (lib/audio/) and Blender movie generation (lib/blender/movie/). The MIDI processing pipeline extracts chord progressions and musical metadata, while the Blender integration system generates batch rendering jobs for automated video production. The presentation layer (lib/presentation/) handles cloud resource synchronization and work processing distribution. Key data flow includes MIDI file ingestion through midiRepository.js, chord analysis via midiMeta.js and scales.js, and job creation through jobCreator.js for distributed rendering tasks.

## Key Features

- MIDI file analysis and chord progression extraction using custom algorithms in midiMeta.js
- Real-time chord frequency visualization with interactive Google Charts integration
- Automated Blender 3D movie generation with templates including virtualPiano.js and steamPunk.js
- Cloud-based job distribution system supporting AWS and Azure storage backends
- Interactive song selection interface with filtering capabilities for chord analysis
- Batch processing pipeline for large-scale MIDI file analysis and rendering
- Template-based 3D scene generation with armatureComposer.js for character animation
- RESTful API integration for distributed MIDI metadata processing

## Getting Started

1. Install Node.js and npm dependencies using 'npm install' 2. Ensure Blender is installed and accessible for 3D rendering functionality 3. Configure AWS and Azure credentials for cloud storage integration 4. Set up MongoDB instance for data persistence (required by mongoose dependency) 5. Create necessary directories: build folder for compiled assets, and configure MIDI input folder path in addJobs.js 6. Run 'npm run build' to compile ES6 source code from src/ to lib/ using Babel 7. Start development server with 'npm start' which runs HTML watching, JS compilation, and serves the application 8. For testing, use 'npm run test' for watch mode or 'npm run test_once' for single test execution 9. Configure API endpoints in the codebase (currently pointing to abridgetonowhere.ddns.net and blendedmusic.azurewebsites.net) 10. Ensure Google Charts API access for visualization components in index.js
