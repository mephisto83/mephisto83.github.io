## Overview

Flow-Frame is a comprehensive automation platform that orchestrates complex workflows involving AI, media processing, browser automation, and email management. It serves as a central nervous system connecting Large Language Models (OpenAI, Groq) with real-world interfaces like Gmail, Chrome browsers, and file systems to execute sophisticated automation tasks. The platform combines agentic UI planning with robotjs for human-like interactions, dynamic workflow execution from JSON configurations, and production-ready resilience with global error handling.

## Architecture

The system follows a modular Express.js server architecture with distinct service layers. The core server (src/server.js) provides global error handling and API endpoints, while specialized controllers (BrowserController, WorkflowController, PromptController) handle domain-specific operations. The platform integrates multiple automation engines: browser automation via robotjs and Chrome extensions (src/extension/), email processing through Gmail and Microsoft email pollers (src/gmail-poller/, src/filter-gmail-poller/), and AI services for image classification and prompt execution. Media processing pipelines handle audio generation (ElevenLabs), image processing (Sharp, Canvas), and PDF manipulation. The system supports both Docker containerization and GPU-accelerated deployments for ML workloads, with parallel processing capabilities using worker threads for data augmentation tasks.

## Getting Started

1. Clone the repository and install dependencies with `npm install`. 2. Create a `.env` file with required API keys: OPENAI_API_KEY, GROQ_API_KEY, ELEVENLABS_API_KEY, and Google Cloud credentials. 3. Set up Gmail authentication by placing Google OAuth2 credentials in `src/gmail-poller/credentials.json` and running `node src/gmail-poller/setup-auth.js`. 4. Configure the system by editing `src/config.json` with your specific settings for directories, API endpoints, and automation preferences. 5. Build the TypeScript code using `npm run build` which compiles to the `dist/` directory and copies non-TS files. 6. Start the server with `npm start` which runs the compiled `dist/server.js` on the configured port (default 8080). 7. For GPU-enabled deployments, use Docker with `docker build -f src/Dockerfile --build-arg TORCH_CHANNEL=cu126 -t flow-frame .` and ensure NVIDIA Container Toolkit is installed. 8. Run tests with `npm test` to verify the setup, and access the API endpoints for workflow management, browser automation, and media processing.
