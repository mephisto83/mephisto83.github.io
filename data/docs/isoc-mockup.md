## Overview

ISOC Mockup is a React Native mobile application built with Expo that serves as a construction site monitoring and floor plan visualization system. The app appears to be designed for tracking construction progress of high-rise buildings in New York City, featuring an extensive collection of floor plan blueprints and building imagery. Based on the package.json naming convention 'icsoverwatch' and the architectural focus, this is likely a proof-of-concept or demonstration application for construction oversight and project management capabilities.

## Architecture

The application follows a standard React Native/Expo architecture with TypeScript support. The main entry point is App.tsx, with source code organized under the src/ directory containing data management (src/data/index.ts), theming system (src/themes/), and component structure. The app utilizes a comprehensive asset management system with over 60 floor plan images stored in assets/floorlplans_400/ directory, each following a UUID-based naming convention with coordinate positioning (e.g., 'id-floor_x0_y0.png'). The floorPlanFiles() function in src/data/index.ts dynamically requires all floor plan assets, suggesting a gallery or carousel-based viewing system. Theme management is handled through separate light.ts and dark.ts configuration files providing consistent styling across the application.

## Getting Started

1. Clone the repository and navigate to the isoc/ directory. 2. Install dependencies with 'npm install' to set up React Native, Expo, and all required packages. 3. Ensure you have Expo CLI installed globally with 'npm install -g @expo/cli'. 4. For development, run 'npm start' to launch the Expo development server. 5. For platform-specific development, use 'npm run ios' or 'npm run android'. 6. To build for deployment, configure EAS (Expo Application Services) credentials and use 'npm run bid' for iOS builds or 'npm run bad' for Android builds. 7. The update_version.js script automatically handles version increments during builds. 8. For adding new floor plans, place images in assets/floorlplans_400/ and run the file_name.js utility to regenerate the require statements for src/data/index.ts.
