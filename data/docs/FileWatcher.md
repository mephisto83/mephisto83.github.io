# FileWatcher

> c#, One way mirror files across folders

![Language](https://img.shields.io/badge/language-C%23-blue)

## Overview

FileWatcher is a C# console application designed to provide one-way file synchronization and mirroring between source and target directories. The application monitors specified folders for file system changes (create, update, delete, rename operations) and automatically mirrors these changes to corresponding target locations. It maintains persistent state information about monitored files to detect changes between application restarts and implements a queue-based system to handle file operations reliably.

## Architecture

The application follows a manager-based architecture with FileWatcherManager as the central orchestrator. It uses .NET's FileSystemWatcher components to monitor source directories and maintains a queue-based processing system (Backlog queue) for handling file operations. The CommandReader loads JSON configuration files to define source-target folder mappings. State persistence is handled through StateWriter and State classes, storing file metadata in JSON format. The DirectoryInfoExtensions provides recursive directory copying functionality. A timer-based system processes queued operations and periodically writes state information to filewatch.json.

## Key Features

- Real-time file system monitoring using FileSystemWatcher for automated change detection
- One-way directory mirroring with support for create, update, delete, and rename operations
- JSON-based configuration system defining source and target folder relationships
- Persistent state management storing file timestamps and metadata in filewatch.json
- Queue-based operation processing ensuring reliable file synchronization
- Automatic change detection on startup comparing current file states with persisted data
- Recursive directory copying with subdirectory support
- Timer-based periodic state persistence and operation processing

## Getting Started

1. Clone the repository and open MEPH.util.FileWatcher.sln in Visual Studio. 2. Restore NuGet packages (Newtonsoft.Json is required). 3. Create a JSON configuration file with the following structure: {"root": "C:\\SourceRoot\\", "folders": [{"name": "subfolder1\\", "target": "targetfolder1\\"}]}. 4. Build the solution to generate the executable. 5. Run the application from command line: MEPH.util.FileWatcher.exe config.json where config.json is your configuration file path. 6. The application will start monitoring specified directories and create a filewatch.json state file in the application directory. 7. Press Enter in the console to stop the application.
