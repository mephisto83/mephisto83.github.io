# disonance

> javascript version of https://github.com/bzamecnik/dissonant

![Language](https://img.shields.io/badge/language-JavaScript-blue)

## Overview

Disonance is a JavaScript/TypeScript port of the Python library 'dissonant' that implements musical dissonance calculation algorithms. The library provides computational models for measuring the perceived harshness or dissonance between musical tones and chords based on psychoacoustic research. It implements five different dissonance models from academic literature (Sethares 1993, Vassilakis 2001, Cook 2002/2006/2009) and includes utilities for generating harmonic tones and converting between musical pitches and frequencies. The library is designed for music theory research, algorithmic composition, and applications requiring quantitative analysis of musical consonance and dissonance.

## Architecture

The project follows a modular TypeScript architecture with clear separation of concerns. The core functionality is divided into three main modules: dissonance.ts handles the primary dissonance calculation algorithms, models.ts implements the various psychoacoustic models as classes implementing DisonanceModel and TensionModel interfaces, and tuning.ts provides utility functions for harmonic tone generation and pitch-frequency conversions. The main entry point index.ts exports the public API. The architecture supports extensibility through interfaces, allowing new dissonance models to be easily added. Each model class encapsulates its specific parameters and calculation methods, with the dissonance() function orchestrating pairwise calculations across all frequency combinations in a chord.

## Key Features

- Five psychoacoustic dissonance models (Sethares1993, Vassilakis2001, Cook2002, Cook2006, Cook2009) with research-backed parameter corrections
- Chord dissonance calculation supporting multiple frequencies with different amplitudes and customizable aggregation functions
- Harmonic tone generation with configurable partial counts and amplitude profiles (exponential, inverse, constant)
- Pitch-to-frequency conversion utilities supporting different tuning systems and base frequencies
- Pairwise dissonance calculation between individual frequency pairs with amplitude weighting
- Triad tension calculation for three-note chord analysis (Cook2009 model)
- Extensible model system through DisonanceModel and TensionModel interfaces
- Comprehensive test suite covering all models and core functionality

## Getting Started

1. Clone the repository and install dependencies with 'npm install'. 2. Build the project using 'npm run build' which compiles TypeScript and runs grunt bump. 3. Run tests with 'npm test' to execute the mocha test suite in src/tests/. 4. For development, use 'npm start' to compile and run the built index.js file. 5. The main exports are available from src/index.ts after compilation to dist/. 6. Import dissonance calculation functions from the dissonance module, model classes from the models module, and tuning utilities from the tuning module. 7. Use HarmonicTone() to generate frequency/amplitude arrays and dissonance() to calculate chord dissonance scores.
