# playwatch-code

## Overview

PlayWatch Code is a comprehensive media content management platform built with TypeScript and React, designed to handle the approval, distribution, and monitoring of video content. The system appears to be focused on managing media libraries, series content, and user approval workflows for content distribution. Based on the extensive codebase with over 1 million lines of TypeScript, this is an enterprise-grade application that leverages Firebase for backend services and implements a sophisticated Redux-based state management architecture for handling complex media workflows.

## Architecture

The application follows a feature-based modular architecture with each domain entity (account, address, approvalRequest, approvedMedia, approvedSeries, metricDeployment, etc.) having its own complete module structure. Each feature module contains a consistent pattern: model definitions (*-model.ts), property configurations (*-properties.ts and *-property-definitions.ts), Redux slices (*-slice.ts), async thunks (*-thunk.ts), service layers (*-service.ts), validation schemas (*-validation.ts), and Firestore security rules (*-rules.rules). The frontend uses React components with TypeScript, while the backend leverages Firebase Functions with Express.js. State management is centralized through Redux Toolkit with async thunk patterns for handling API calls. The architecture supports metric deployment configurations and player-based content delivery systems.

## Getting Started

1. Install dependencies using npm/yarn (firebase-functions is the primary dependency). 2. Configure Firebase project settings and initialize Firebase CLI tools. 3. Set up Firestore database with security rules from the *-rules.rules files in each feature directory. 4. Configure Firebase Functions environment variables for Express.js backend. 5. Build the application using the provided build.cake and build.ps1 scripts. 6. Deploy Firebase Functions and Firestore rules using Firebase CLI. 7. Configure the localcode/features directory structure and ensure all feature modules are properly imported. 8. Set up the Redux store configuration to include all feature slices from directories like metricDeployment, approvedMedia, etc. 9. Configure Firebase Storage rules using the firestorage.rules file. 10. Test the application locally using Firebase emulators before production deployment.
