# space-ships

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

Space-ships is a web-based 3D spaceship builder and visualization application built with A-Frame (WebXR) and React. The application allows users to procedurally generate and customize 3D spaceships by combining various modular parts including hulls, engines, bridges, and weapons. The system uses a component-based architecture where ship parts are loaded from 3D models and can be textured, positioned, and assembled into complete vessels through an interactive web interface.

## Architecture

The application follows a modular A-Frame component architecture with React as the UI framework. Core components include: `model-loader.ts` for loading and managing 3D assets from OBJ/MTL files, `ship-part.ts` for individual component rendering with texture mapping, `ship.ts` for procedural ship assembly using seed-based randomization, and `container.ts` for dynamic UI layout management. The system uses a custom event system (`SpaceshipEvents`) for component communication and includes utilities in `shipparts.ts` for coordinate conversion, part positioning, and face-to-face attachment calculations. Firebase is integrated for deployment and potential data persistence.

## Key Features

- Procedural spaceship generation using seed-based randomization with modular parts system
- Interactive 3D visualization with A-Frame WebXR support for VR/AR experiences
- Dynamic texture application system supporting multiple color variations (red, blue, yellow) for ship parts
- Modular ship part attachment system with face-to-face positioning and normal-based rotation calculations
- Component-based UI system with dynamic container layout management and flexbox-style positioning
- Real-time 3D model loading and cloning system for efficient asset management
- Custom A-Frame primitives including a-ship, a-ship-part, a-model-loader, and a-container components
- Firebase deployment integration with automated build and deploy pipeline

## Getting Started

1. Clone the repository and navigate to the `my-aframe-project` directory. 2. Install dependencies with `npm install` to get React, A-Frame, TypeScript and other required packages. 3. Ensure 3D models are properly placed in `public/models/` directory (crystals.gltf, hull_tall.gltf, Shipyard.obj/mtl files). 4. Place texture assets in `public/assets/textures/` and `public/assets/images/` directories. 5. Configure Firebase by setting up `firebase.json` and `database.rules.json` for deployment. 6. Run `npm start` to start the development server on localhost:3000. 7. For production deployment, run `npm run deploy` which builds the app and deploys to Firebase. 8. Optional: Use Python scripts in the `python/` directory for batch processing texture images if adding new ship part textures.
