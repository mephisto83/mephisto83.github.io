# torch-graph

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

Torch Graph is a web-based visual neural network architecture designer that enables users to create PyTorch models through a drag-and-drop interface. Built as a React application, it provides an intuitive graph-based canvas where users can visually compose neural network layers, connect parameters, and generate executable PyTorch code. The application bridges the gap between visual model design and production code by automatically translating graph configurations into clean, maintainable PyTorch class definitions with proper configuration management.

## Architecture

The application follows a modular React architecture centered around a graph-based design canvas. The main components include Canvas.tsx for the ReactFlow-powered visual editor, Sidebar.tsx for layer selection and categorization, and GraphProvider.tsx for centralized state management. The core logic resides in utils/ with codeGenerator.ts handling Python code generation, layerParameters.ts defining comprehensive PyTorch layer configurations, and customModules.ts providing extensibility for custom layer types. The system supports complex neural network structures through specialized node types including CustomNode for standard layers, ConfigNode for parameter management, and container nodes like Sequential and ModuleList for hierarchical architectures. Parameter connections are managed through ReactFlow edges, enabling dynamic parameter binding between nodes.

## Key Features

- Visual drag-and-drop neural network design using ReactFlow canvas with custom node types for PyTorch layers
- Comprehensive PyTorch layer support including Conv2d, Linear, LSTM, BatchNorm, and 50+ other layer types defined in layerParameters.ts
- Dynamic parameter management through ConfigNode connections enabling runtime parameter binding and dependency resolution
- Automatic PyTorch code generation with proper class structure, configuration management, and topological sorting for parameter dependencies
- Local storage integration for saving and loading graph models with GraphProvider state management
- Support for complex architectures including nn.Sequential and nn.ModuleList containers with nested layer structures
- Real-time parameter validation and type checking through customModules.ts validation system
- Extensible architecture allowing custom layer definitions and parameter types through configuration files

## Getting Started

Clone the repository and navigate to the pytorch-visual-composer directory. Install dependencies using 'npm install' to set up React, TypeScript, ReactFlow, and Material-UI components. Start the development server with 'npm start' which launches the application at http://localhost:3000. The application requires Node.js and npm/yarn for dependency management. For production deployment, run 'npm run build' to generate optimized static assets. The main entry point is src/index.tsx which renders the App.tsx component containing the Canvas, Sidebar, and GraphProvider components. Configuration for layer types and parameters can be modified in src/utils/ files including layerParameters.ts for adding new PyTorch layers and layerCategories.ts for organizing layer types in the sidebar interface.
