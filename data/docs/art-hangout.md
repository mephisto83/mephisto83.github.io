# art-hangout

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

Art Hangout is a real-time collaborative art application designed for emotionally charged situations, built as a Google Meet-style platform where users can draw together in shared virtual rooms. The application enables multiple users to create art collaboratively with individual drawing layers, AI-powered art generation capabilities, and comprehensive presence management. It's specifically architected to support therapeutic or emotional expression scenarios with features like vibe presets (vent, grief, grounding, celebration, quiet) and pace modes for different interaction styles.

## Architecture

The application follows a monorepo architecture with three main packages: a React web app (apps/web), Firebase Cloud Functions backend (functions), and shared TypeScript types (shared). The system uses Firestore for persistent data storage with collections for canvases, members, profiles, layers, and operations, while Firebase Realtime Database handles real-time presence tracking. Each user draws on individual layers that are composited client-side, with all drawing operations logged as an append-only operations history. The DrawingStage.tsx component manages the HTML5 Canvas rendering, while presence is handled through usePresence.ts hook connecting to Firebase Realtime Database. Authentication and room membership is server-mediated through Cloud Functions, ensuring secure access control.

## Key Features

- Real-time collaborative drawing with individual user layers managed by DrawingStage.tsx
- Server-mediated room membership through redeemInvite Cloud Function with invite-based access control
- Live presence tracking showing online status, microphone/camera state, and active drawing locations via usePresence.ts
- Multiple user profiles per room for shared machine scenarios handled by ProfileManager.tsx
- AI-powered art generation with support for Imagen and custom models through createAiProposal functions
- Emotional context awareness with vibe presets (vent, grief, grounding, celebration, quiet) and pace modes
- Voice prompt integration through VoicePromptBar.tsx for hands-free interaction
- Comprehensive operation history with monotonic versioning through commitOp Cloud Function
- Multi-region AI proposals with color mapping and sketch-based generation
- Consent-based collaboration system with approval workflows for drawing permissions

## Getting Started

1. Install Node.js 20+ and npm 10+. 2. Run 'npm install' to install all workspace dependencies. 3. Set up Firebase project: run 'npx firebase login' and 'npx firebase use --add art-hangout-dev'. 4. Create 'apps/web/.env' with Firebase configuration variables (VITE_FIREBASE_API_KEY, VITE_FIREBASE_AUTH_DOMAIN, etc.) and set VITE_USE_EMULATORS=1 for local development. 5. Run 'npm run dev' to start the development environment which will build shared types, compile Cloud Functions, start Firebase emulators (Auth, Functions, Firestore, Database, Storage), and launch the Vite dev server at http://localhost:5173. 6. Access Firebase Emulator UI at http://localhost:4000 for debugging. 7. For production deployment, run 'npm run deploy:rules' for security rules and 'npm run deploy:functions' for Cloud Functions.
