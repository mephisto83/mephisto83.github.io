# red-duplicado

![Language](https://img.shields.io/badge/language-JavaScript-blue)

## Overview

Red-duplicado is a web scraping and HTML element duplication tool designed to extract and analyze web page components. The project's core functionality centers around duplicating HTML elements with their complete CSS styling applied inline, ensuring visual consistency even when elements are moved outside their original DOM context. The tool includes automated web crawling capabilities using Puppeteer to collect URLs, capture screenshots, and identify interactive elements (interest points) across web pages for data mining and component analysis purposes.

## Architecture

The architecture consists of three main components: a core duplication engine (src/duplicado.js) that handles HTML element cloning and CSS style extraction, a web automation layer (src/process.js) that manages Puppeteer-based crawling and data collection, and utility modules for array operations and I/O handling. The system operates through multiple entry points - collectUrls.js for URL discovery, work.js for parallel processing of web scraping tasks, and test.js for component analysis. JSON files store collected data including CSS selectors, interest points, and URLs for various Bootstrap components like alerts, buttons, cards, and forms.

## Key Features

- HTML element duplication with complete CSS style preservation through getComputedStyle API
- Automated web crawling with multi-threaded processing using Puppeteer for screenshot capture and URL collection
- Interactive element detection system that identifies clickable elements and event listeners across web pages
- CSS selector extraction and component categorization for Bootstrap elements (buttons, alerts, cards, forms, etc.)
- Visual overlay system with drawBox functionality for element highlighting and selection during analysis
- Batch processing capabilities with configurable memory allocation (--max-old-space-size=8192) for large-scale data collection

## Getting Started

1. Clone the repository and run 'npm install' to install dependencies including Puppeteer and Clipboardy. 2. Configure the SCREEN_SHOT_FOLDER path in work.js and collectUrls.js to specify where collected data will be stored. 3. Run 'npm run collect' to start URL collection process or 'npm run work' to begin the web scraping workflow. 4. For component analysis, execute 'node test.js' and modify the proc variable to select different operations (COLLECT_CSS_SELECTORS, CAPTURE_INTEREST_POINTS, etc.). 5. The system will generate JSON files for each component type (alerts.json, buttons.json, etc.) containing extracted CSS selectors and element data. 6. Use the duplicado.js functions in a browser environment to enable interactive element selection and duplication features.
