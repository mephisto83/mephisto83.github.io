# redquick

![Language](https://img.shields.io/badge/language-JavaScript-blue)

## Overview

RedQuick is a comprehensive .NET-based distributed work processing framework designed to accelerate application development and deployment. The project consists of multiple components including a core C# library, React Native mobile starter templates, and a distributed agent-based work coordination system. The framework provides rapid scaffolding capabilities, validation rules, background processing, and mobile development tools to help developers 'get started quicker' as indicated by its tagline.

## Architecture

The architecture follows a multi-layered distributed pattern with several key components: RedQuickCore serves as the foundational library containing validation rules, attributes, data models, and core utilities. The system implements a distributed agent-worker pattern through RedQuickAgentExample and RedQuickAgentWorker, where work coordinators manage task distribution across multiple worker nodes. The mobile layer includes RedQuickReactNativeStarter and RedQuickLib providing React Native components and utilities. The framework uses ASP.NET Core Web APIs for agent communication, Autofac for dependency injection, and implements background services for work processing. Key architectural files include WorkerController.cs for agent coordination, RedArbiter.cs for work arbitration, and various validation attributes in the Rules directory.

## Key Features

- Distributed agent-based work processing system with WorkerController and WorkerMinister components
- Comprehensive validation framework with rules like NumericIntAttribute, SocialSecurityAttribute, and RestrictedCharacterAttribute
- React Native starter templates with pre-configured components for mobile development
- Background task processing with RefreshResult, WorkReply, and DyingResult message types
- Auto-generated data models from JSON files in RedQuickCore/Automodel/auto directory
- Identity and authorization system with RedAttribute, Protected, and Allow attributes
- Configurable work distribution through IStreamTypeService, IWorkTaskService, and IWorkDistributionService interfaces
- Built-in accident/violation tracking system with AccidentViolations data structures

## Getting Started

1. Clone the repository and ensure .NET Core SDK is installed. 2. Navigate to RedQuick directory and restore NuGet packages using 'dotnet restore' on RedQuick.sln. 3. Build the solution with 'dotnet build'. 4. For the agent system, configure appsettings.json in both RedQuickAgentExample and RedQuickAgentWorker projects. 5. Start the coordinator service by running RedQuickAgentExample project, then start worker nodes with RedQuickAgentWorker. 6. For React Native development, navigate to RedQuickReactNativeStarter/RedQuick and run 'npm install' followed by 'react-native run-ios' or 'react-native run-android'. 7. To use RedQuickLib in other React Native projects, install via 'npm install redquicklib'. 8. Run tests using 'dotnet test' in the RedQuick.Tests project directory. 9. Use the PowerShell scripts start_processes.ps1 in agent directories to launch multiple worker instances.
