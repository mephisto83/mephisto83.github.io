# scrape-city

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

scrape-city is a distributed web scraping system specifically designed to extract dictionary data from Dictionary.com at scale. The project implements a master-worker architecture using the 'distributo' library to coordinate parallel scraping tasks across multiple worker processes. The system processes word lists and enriches each word with pronunciation data, synonyms, and antonyms by scraping Dictionary.com pages using Puppeteer for headless browser automation.

## Architecture

The system follows a distributed task processing pattern with three main components: a master process (src/master.ts) that distributes word scraping tasks, worker processes (src/worker.ts) that perform the actual web scraping using Puppeteer, and shared interfaces/utilities (src/interfaces.ts, src/utils.ts) for data structures and file operations. The master reads word lists from JSON files, creates tasks, and coordinates work distribution via the TaskDistributor on port 4012 with Bonjour service discovery. Workers connect to the master, receive batches of words to scrape, extract data from Dictionary.com pages, and return structured results that are saved to JSONL output files.

## Key Features

- Distributed task processing with automatic worker discovery via Bonjour/mDNS service discovery
- Headless web scraping of Dictionary.com using Puppeteer with request interception to block images/CSS for performance
- Batch processing of word lists with configurable batch sizes (default 5 words per batch)
- Structured data extraction including word pronunciation, synonyms, and antonyms
- Comprehensive logging system using winston with both console and file output
- JSONL output format for easy data processing and analysis
- Error handling and retry mechanisms for robust scraping operations
- Random user agent rotation and browser optimization for anti-detection

## Getting Started

1. Clone the repository and install dependencies with 'npm install'. 2. Prepare your input word dictionary JSON file (referenced as 'D:\words\words_dictionary.json' in src/master.ts). 3. Update file paths in src/master.ts to match your input dictionary file and desired output location. 4. Build the project using 'npm run build' to compile TypeScript to JavaScript in the dist/ directory. 5. Start the master process with 'npm run master' to begin task distribution on port 4012. 6. In separate terminals, start worker processes using 'npm run worker' - workers will automatically discover and connect to the master via Bonjour. 7. Monitor progress through console output and check the worker.log file for detailed logging. 8. Results will be written to the specified JSONL output file as workers complete scraping tasks.
