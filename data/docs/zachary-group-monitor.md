# zachary-group-monitor

![Language](https://img.shields.io/badge/language-Python-blue)

## Overview

Zachary Group Monitor is a Python-based multi-camera surveillance system that provides real-time person detection and automated recording capabilities. The system leverages YOLOv8 computer vision models to detect human presence across multiple camera streams (both local USB cameras and network RTSP streams) and automatically triggers video recording when persons are detected. The project offers both single-camera monitoring (stream.py) and multi-camera concurrent monitoring (multi_camera_recorder.py) with intelligent recording management that starts and stops based on detection events.

## Architecture

The system follows a multi-threaded architecture where each camera stream is handled by a separate CameraHandler thread in multi_camera_recorder.py. Each thread runs an independent detection loop that captures frames, processes them through a YOLO model instance, and manages recording/snapshot operations. The single-camera version in stream.py provides a simpler synchronous implementation for monitoring one stream. Both implementations use OpenCV for video capture and processing, Ultralytics YOLO for person detection with configurable confidence thresholds, and automatic file management with timestamped recordings and snapshots stored in organized directory structures.

## Key Features

- Multi-camera concurrent monitoring with threaded CameraHandler classes supporting both USB and RTSP camera streams
- Real-time person detection using YOLOv8 models with configurable confidence thresholds (default 0.5)
- Automatic video recording triggered by person detection with configurable timeout periods (2 seconds default)
- Periodic snapshot capture at configurable intervals (10 seconds default) regardless of detection status
- Organized file output with separate directories for recordings and snapshots per camera
- Support for both local USB cameras (device indices) and network IP cameras via RTSP URLs
- Graceful cleanup and thread management with keyboard interrupt handling
- Visual bounding box overlay on detected persons with confidence scores

## Getting Started

1. Install required dependencies using 'pip install -r requirements.txt' which includes opencv-python, ultralytics, and numpy. 2. Ensure camera hardware is connected (USB cameras) or network cameras are accessible via RTSP URLs. 3. For single camera monitoring, run 'python stream.py' and modify the cap = cv2.VideoCapture(1) line to use your desired camera index or RTSP URL. 4. For multi-camera monitoring, edit the camera_configs list in multi_camera_recorder.py main() function to specify your camera sources and output directories, then run 'python multi_camera_recorder.py'. 5. Create 'streams' and 'snapshots' directories for stream.py, or the system will auto-create camera-specific directories for multi_camera_recorder.py. 6. The system will automatically download the YOLOv8 nano model (yolov8n.pt) on first run. 7. Press 'q' to quit single camera mode, or use Ctrl+C to stop multi-camera monitoring.
