# bus-com

> Communicate between computers

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

bus-com is a TypeScript-based distributed communication system that enables real-time messaging between multiple computers on a local network. The project implements a peer-to-peer event bus architecture using Socket.IO for WebSocket communication and Bonjour (mDNS) for automatic service discovery. Each node in the network runs an Express server that can both serve as a message broker and connect to other discovered nodes, creating a mesh network topology where events can be published and subscribed to across all connected machines.

## Architecture

The system follows a distributed peer-to-peer architecture where each node runs both a server and client. The core application in src/index.ts consists of three main components: an Express HTTP server with Socket.IO for handling WebSocket connections and REST endpoints, a Bonjour mDNS client for network service discovery and advertisement, and a Socket.IO client for connecting to discovered peer nodes. Each node generates a unique identifier using the hostname and current date, stores it in nodeName.txt, and publishes itself as an 'eventbus' service type. When nodes discover each other through mDNS, they establish bidirectional Socket.IO connections and exchange hello/helloReply handshake messages. The system maintains a Set of connected peers to prevent duplicate connections and supports event publishing through both WebSocket events and HTTP POST endpoints.

## Key Features

- Automatic network service discovery using Bonjour mDNS protocol
- Real-time bidirectional communication via Socket.IO WebSockets
- Unique node identification system with hostname and date-based naming
- HTTP REST API endpoint for event publishing at /publish
- Peer-to-peer mesh network topology with automatic connection management
- Static file serving capability through /files route
- Connection deduplication to prevent multiple connections to same peer
- Hello/HelloReply handshake protocol for peer acknowledgment
- Topic-based subscription system for selective event handling
- CORS-enabled cross-origin communication support

## Getting Started

1. Clone the repository and install dependencies using 'npm install' 2. Build the TypeScript code using 'npm run build' which compiles src/index.ts to dist/index.js 3. Start the application using 'npm start' or use 'npm run go' for build and start in one command 4. For development, use 'npm run dev' to run directly with ts-node or 'npm run watch' for continuous compilation 5. The server will start on port 4000 and automatically generate a unique node name in nodeName.txt 6. Deploy on multiple machines on the same network - nodes will automatically discover each other via mDNS 7. Use the /publish POST endpoint to send events or connect Socket.IO clients to subscribe to real-time events 8. Monitor console output to see peer discovery and connection status
