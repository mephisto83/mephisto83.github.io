## Overview

redquickbuilderweb is a comprehensive TypeScript-based web application that appears to be a visual development platform combining mind mapping, diagramming, and code generation capabilities. The project contains two main applications: 'mind-builder' and 'redquickbuilder', both built with React and focused on creating interactive visual interfaces for building and managing complex data structures and workflows. Based on the extensive template system found in the codebase, this appears to be a rapid application development tool that can generate backend code patterns, particularly for .NET/C# applications with agent-based architectures and permission systems.

## Architecture

The application follows a modular React architecture with two distinct sub-applications. The mind-builder application uses Material-UI (@mui/material) and react-diagrams for creating interactive visual components, while redquickbuilder incorporates more advanced diagramming capabilities with @projectstorm/react-diagrams and Monaco Editor for code editing. Both applications utilize Redux for state management with react-redux and redux-thunk for async operations. The codebase includes an extensive template system located in src/templates/standard/ that generates C# controller methods, interfaces, and service layers following an agent-based permission model. The architecture supports real-time collaborative features through Firebase integration and includes comprehensive CSS styling with AdminLTE Bootstrap themes.

## Getting Started

Clone the repository and navigate to either the mind-builder or redquickbuilder directory. Install dependencies using 'npm install' to set up all required packages including React, TypeScript, and the extensive list of UI libraries. For the mind-builder application, run 'npm start' to launch the development server on localhost:3000. The redquickbuilder application can be started similarly but may require additional configuration for the Monaco Editor and diagram components. Ensure you have Node.js 16+ installed as indicated by the @types/node version. Configure Firebase credentials if using the real-time collaboration features. The build process uses 'set DISABLE_ESLINT_PLUGIN=true && react-scripts build' for production builds, indicating some ESLint configuration adjustments may be needed. Both applications support hot reloading and include comprehensive TypeScript definitions for all CSS modules.
