# sweatshirt

![Language](https://img.shields.io/badge/language-JavaScript-blue)

## Overview

Sweatshirt is a React-based 3D product customization and design application that enables users to create custom apparel designs on various garment types including hoodies, t-shirts, sweatshirts, and other clothing items. The application provides an interactive 3D preview system where users can apply designs to different parts of garments such as front panels, sleeves, and hood sections. Built with Three.js for 3D rendering and Fabric.js for 2D design editing, it serves as a comprehensive product design tool for custom apparel manufacturing.

## Architecture

The application follows a standard React component architecture with a main App component (src/index.tsx) bootstrapped via Create React App. The core functionality is split between 3D visualization components powered by Three.js and 2D design editing interfaces using Fabric.js. Product design data is managed through structured JSON configurations (productdesign.js, presets.js) that define editable fields, masks, overlays, and design objects for different garment areas. The application includes a comprehensive asset management system with 3D models (GetProduct3DModel.glb), texture maps, normal maps, and material definitions stored in the public/resources directory. The SweatshirtCore.js file appears to contain the main application logic for handling design interactions and 3D rendering.

## Key Features

- 3D garment visualization with realistic material rendering using PBR textures and normal maps
- Multi-area design editing supporting front panels, sleeves, hood sections with individual masks and overlays
- Real-time design preview with quality warnings and validation
- Support for multiple garment types including hoodies, t-shirts, sweatshirts, pajamas, and accessories
- Pattern-based design application with configurable pattern widths and treatments
- Image upload and positioning system with rotation, scaling, and mirroring capabilities
- Material-aware rendering with specular maps, ambient occlusion, and stitching details
- Design object management with locked, hidden, and read-only states
- Responsive design editing with aperture-based constraints
- Export functionality for production-ready design specifications

## Getting Started

1. Clone the repository and navigate to the project directory. 2. Install dependencies using 'npm install' which will install React, Three.js, Fabric.js, MUI, and other required packages. 3. Ensure the public/resources directory contains all necessary 3D models (GetProduct3DModel.glb), texture files, and art assets. 4. Start the development server with 'npm start' which launches the application on localhost:3000. 5. For production builds, use 'npm run build' which creates an optimized build and moves it to the docs directory for GitHub Pages deployment. 6. The application loads product design configurations from productdesign.js and presets.js files in the public directory. 7. Verify that all texture maps, normal maps, and 3D assets are properly loaded by checking the browser console for any 404 errors. 8. Test the 3D rendering functionality and design editing interfaces to ensure proper Three.js and Fabric.js integration.
