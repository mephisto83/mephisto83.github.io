# playwatch-media-process

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

Playwatch Media Process is a comprehensive Firebase-based backend system designed for media streaming and content management. The system serves as a sophisticated media processing platform that handles user profiles, content curation, metrics tracking, and social media integration for what appears to be a video streaming or media consumption application. Built with TypeScript and Firebase Functions, it provides a robust foundation for managing complex media workflows, user interactions, and content delivery systems.

## Architecture

The system follows a modular Firebase Functions architecture with auto-generated controllers and services. The codebase is organized into distinct layers: models (defining data structures like MetricDeployment, SocialMediaAccount, ProcessedSocialMediaAccount), services (handling CRUD operations with Firestore), and controllers (managing API endpoints). Key architectural components include the src/functions/ directory containing service layers for each entity, src/models/ containing TypeScript interfaces and property definitions, and src/controllers/auto/ providing RESTful endpoints. The system uses Firebase Firestore as the primary database with hierarchical document structures and implements versioning, soft deletes, and automated denormalization patterns. The job_index.js serves as the main entry point supporting multiple environments (dev, staging, prod).

- Comprehensive media processing pipeline with FFmpeg integration for video/audio manipulation
- Multi-tenant social media account management with platform-specific handling
- Advanced metrics deployment system supporting configurable analytics tracking
- Automated model synchronization with denormalization and meta-property management
- Content curation system with support for media lists, series, and house-based content organization
- Player role management and invitation systems for content creators

- Multi-environment deployment support (development, staging, production)

- Automated cleanup functions for mobile logins and periodic system maintenance

## Getting Started

1. Ensure Node.js 16 is installed and Firebase CLI is available globally. 2. Clone the repository and run 'npm install' to install dependencies including Firebase Functions, Express, FFmpeg, and Sharp. 3. Configure Firebase project by setting up firebase.json and ensuring Firestore security rules are properly configured via firestore.rules. 4. Set up environment-specific configurations for dev, staging, and prod environments. 5. Build the TypeScript code using 'npm run build' which compiles src/ files to lib/ directory. 6. For development, run 'npm run dev' to build and start the job processor, or 'npm run serve-all' to start Firebase emulators for local testing. 7. Deploy to Firebase using 'npm run deploy' for functions or 'npm run publish' for hosting. 8. Configure Firebase Authentication and Firestore database with appropriate indexes for the complex query patterns used throughout the application. 9. Ensure proper permissions are set for media processing operations and external service integrations.
