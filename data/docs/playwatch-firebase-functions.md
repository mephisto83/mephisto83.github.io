# playwatch-firebase-functions

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

Playwatch Firebase Functions is a comprehensive backend service for a media streaming and content management platform built on Google Firebase. The system manages user accounts, media content, social media integrations, player roles, and content distribution through a sophisticated permission-based architecture. It appears to be designed for a streaming service that supports content creators, viewers, and administrative players with different access levels and capabilities.

## Architecture

The application follows a layered architecture with auto-generated controllers in functions/src/controllers/auto/, service layers in functions/src/functions/, and data models in functions/src/models/. The system uses Firebase Firestore for data persistence with a complex permission system managed through PlayerRole, ProfileRole, and SystemRole entities. Key architectural components include: Firestore document collections for user management (players, profiles, systems), social media account management, content customization services, and automated CRUD operations. The permission-service.ts file implements role-based access control across different user types. The server_init.ts handles user creation workflows and initial role assignments.

## Key Features

- Role-based permission system with PlayerPermission, ProfilePermission, and PlayerAccountPermissions enums
- Social media account integration with ProcessedSocialMediaAccount and SocialMediaAccount models
- Auto-generated CRUD controllers for address, approvals, content management, and media customization
- User account lifecycle management with OnUserCreate function in server_init.ts
- Content creator and player management with invite systems
- Media quality assurance and branding approval workflows
- Release management with scheduling and zone-based distribution
- Comprehensive testing suite with Firebase Rules Unit Testing framework
- Version control and change tracking for all entities with created/updated/deleted timestamps

## Getting Started

1. Install dependencies: npm install in the functions/ directory. 2. Set up Firebase project and configure firebase.json with your project ID. 3. Configure Google Application Credentials by setting GOOGLE_APPLICATION_CREDENTIALS environment variable to your service account key file. 4. Build the TypeScript code: npm run build. 5. Start the Firebase emulators for local development: npm run serve-all. 6. Run tests: npm test. 7. Deploy to Firebase: npm run deploy for functions or npm run publish for hosting. 8. Ensure firestore.rules and firestore.indexes.json are properly configured for your security requirements.
