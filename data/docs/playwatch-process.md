# playwatch-process

![Language](https://img.shields.io/badge/language-JavaScript-blue)

## Overview

playwatch-process is a Node.js-based media processing pipeline specifically designed for the PlayWatch video platform. The system handles video transcoding, image processing, and Firebase integration for a video streaming service focused on educational and children's content. The application processes video files by downloading them from URLs, generating multiple format/resolution variants, creating thumbnails, and uploading processed media to Firebase Storage while maintaining metadata in Firestore.

## Architecture

The system follows a batch processing architecture with Firebase as the backend storage and database. The core processing workflow is orchestrated through process.js which queries Firestore for media requiring processing, downloads source files, processes them using FFmpeg and Sharp, then uploads results back to Firebase. The fire.js module handles all Firebase operations including authentication, Firestore queries, and Storage bucket management. Processing jobs are organized in temporary directories with each media item getting its own folder containing source files, processed variants, and metadata.

## Key Features

- Video transcoding to multiple formats (MP4, WebM) and resolutions (240p to 1080p)
- Automatic thumbnail generation from video screenshots at mid-point
- Image processing and resizing to multiple dimensions (30px to 1920px)
- Firebase Storage integration for processed media hosting
- Firestore database management for media metadata and processing state
- Batch processing system that continuously monitors for new media
- Profile image processing with multiple size variants
- Temporary job directory management with automatic cleanup
- Support for educational video content with creator-based organization

## Getting Started

1. Install Node.js and ensure FFmpeg is available in system PATH. 2. Clone the repository and run 'npm install' to install dependencies (firebase, firebase-admin, fluent-ffmpeg, sharp, uuid). 3. Create a Firebase project and download the service account key as 'serviceAccountKey.json' in the project root. 4. Configure Firebase Storage bucket and Firestore database in the Firebase console. 5. Update the Firebase configuration in fire.js with your project's storage bucket URL and database URL. 6. Create necessary Firestore collections: 'medias/{creatorId}/media', 'processed_medias', 'profile/media/images', and 'user_profile'. 7. Run 'node process.js' to start the main processing loop, or use individual scripts like 'node gen-images.js' for specific tasks. 8. Ensure proper file permissions for temporary directory creation and file operations.
