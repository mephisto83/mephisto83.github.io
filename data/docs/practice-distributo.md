# practice-distributo

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

practice-distributo is a TypeScript-based distributed task processing system built on top of the 'distributo' framework. The project demonstrates a practical implementation of distributed word scraping using a coordinator-worker architecture. It processes large JSON dictionaries by distributing word lookup tasks across multiple worker nodes, collecting pronunciation data, synonyms, and antonyms for each word. The system implements fault tolerance through result persistence and deduplication to handle interruptions and resume processing efficiently.

## Architecture

The system follows a distributed coordinator-worker pattern with two main components: src/index.ts implements the CoordinatorTaskDistributor that reads from a words dictionary JSON file (words_dictionary.json), generates DynamicTask objects for each word key, and coordinates task distribution across worker nodes via TCP (port 4193) and HTTP (port 4300) with Bonjour service discovery. The src/worker.ts implements an EnhancedWorkerClient that connects to available coordinators and executes word scraping tasks. Results are persisted to words_dictionary_practice_word_scape.json with built-in deduplication logic. The coordinator uses an async generator pattern (jsonKeyTaskGenerator) for memory-efficient streaming of tasks from large dictionary files.

## Key Features

- Distributed word scraping across multiple worker nodes
- Async generator-based task streaming for memory efficiency
- Automatic result persistence with deduplication to prevent duplicate processing
- Fault tolerance through incremental result saving and resume capability
- Service discovery using Bonjour protocol for automatic worker-coordinator pairing
- Configurable task timeouts (100 seconds) and randomized processing order
- Environment-based worker configuration supporting multiple task types
- Real-time result processing with structured output containing pronunciation, synonyms, and antonyms

## Getting Started

1. Install dependencies with 'npm install' (requires local distributo package at ../distributo/distributo-0.7.3.tgz). 2. Create a .env file in the root directory with SERVICE_TYPE, TASK_TYPES (comma-separated), and WORKING_DIRECTORY variables for worker configuration. 3. Ensure the words dictionary JSON file exists at D:\words\words_dictionary.json or modify the filePath variable in src/index.ts. 4. Update projectPath in src/index.ts to point to your word scraping implementation directory. 5. Run the coordinator with 'npm run go' to start task distribution on ports 4193 (TCP) and 4300 (HTTP). 6. Start worker nodes with 'npm run worker' on separate machines or processes. 7. Results will be automatically saved to D:\words\words_dictionary_practice_word_scape.json as tasks complete.
