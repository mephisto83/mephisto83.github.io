## Overview

Story-Gen is a comprehensive AI-powered content generation platform that orchestrates multiple AI services to create multimedia content including images, audio, video, and text. The system integrates with services like Midjourney, OpenAI, ElevenLabs, and Suno AI to automate the creation of educational content, music videos, historical narratives, and interactive stories. Built as a distributed system with microservices architecture, it supports both cloud-based processing via Google Cloud Run and local GPU-accelerated workers for intensive AI model inference.

## Architecture

The system follows a distributed microservices architecture with Firebase as the central data store and job queue. The main components include: flow-frame-app (React frontend with visual workflow builder), midjourney workers (containerized GPU-enabled processors), various specialized services for audio/video processing, text extraction, and content automation. The architecture supports horizontal scaling through Google Cloud Run for stateless services and dedicated GPU VMs for compute-intensive tasks. Communication between services happens through Firebase Firestore collections and Google Cloud Tasks for job orchestration.

## Key Features

- Visual workflow builder with React Flow for creating AI content pipelines
- GPU-accelerated image generation using Midjourney API with automated job queuing
- Multi-modal content creation combining text, images, audio, and video
- Automated YouTube content pipeline from script generation to video upload
- Real-time audio processing with beat detection and music video generation
- Character extraction and face detection for personalized content
- Blender integration for 3D visualization and animation
- Distributed worker system with automatic GPU fallback (H100 -> A100 -> L4 -> T4 -> CPU)
- Educational content automation for historical figures and biblical narratives
- Microsoft Graph integration for email automation and Office 365 connectivity

## Getting Started

1. Set up Firebase project and configure authentication in flow-frame-app/firebase.json. 2. Install dependencies: run 'npm install' in root, flow-frame-app/, and image_selection/image-selector/ directories. 3. Configure API keys: create api_key.txt files for Groq, OpenAI, and other services. 4. Set up Google Cloud: enable required APIs (Cloud Run, Artifact Registry, Compute Engine) and configure service accounts. 5. Build and deploy workers: use deploy-cloud-run.sh or deploy-gpu-vm.sh scripts in midjourney/ directory. 6. Configure environment variables: PROJECT_ID, SERVICE_ACCOUNT_JSON_BASE64, and API endpoints. 7. Deploy frontend: run 'npm run deploy' in flow-frame-app/ to deploy to Firebase Hosting. 8. Start local development: use 'npm run llmworkflow' for the main app or 'npm run midjourney' for the worker server. 9. For GPU acceleration: follow gpu-startup.sh script to install NVIDIA drivers and container toolkit. 10. Configure Firestore security rules and create initial collections for jobs, workflows, and user data.
