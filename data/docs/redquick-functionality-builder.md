## Overview

RedQuick Functionality Builder is a comprehensive visual development platform that enables developers to create and manage software functionality through an interactive graph-based interface. The project combines a Monaco Editor-powered code editing environment with a React Diagrams-based visual modeling system, allowing users to define TypeScript interfaces and functions visually and generate corresponding code. It appears to be designed as a low-code/no-code solution that bridges visual modeling with actual code generation, particularly focused on Firebase backend services and React frontend applications.

## Architecture

The application follows a modular architecture with three main components: a web-based visual editor (built with React and @projectstorm/react-diagrams), a Firebase Functions backend (red-quick-functions), and a React template system (red-quick-template). The core application (src/) contains the main visual interface with components like BodyWidget for the diagram canvas, CodeEditor components for Monaco Editor integration, and Application.ts for orchestrating the visual-to-code transformation. The red-quick-functions directory contains auto-generated Firebase Cloud Functions with standardized CRUD operations for entities like SocialMediaAccount and ProcessedSocialMediaAccount, following a consistent service pattern with Firestore integration. The system uses TypeScript AST parsing to extract function and interface definitions from code and convert them into visual graph nodes.

## Key Features

- Visual graph-based interface for defining software functionality using @projectstorm/react-diagrams
- Integrated Monaco Editor with TypeScript support and syntax highlighting for code editing
- Automatic code generation from visual models with TypeScript AST parsing via captureGraphNodeDefinitions
- Firebase integration with auto-generated CRUD services in red-quick-functions/src/functions/
- Multi-version TypeScript support (3.6.4 through 4.5.4) for compatibility across different projects
- Material-UI based responsive interface with dark theme support
- Real-time code synchronization between visual models and generated TypeScript code
- Firestore integration with standardized data models and relationship management
- Export/import functionality using JSZip and file-saver for project persistence
- Split-pane interface allowing simultaneous visual and code editing

## Getting Started

1. Clone the repository and install dependencies with 'npm install'. 2. For the main application, run 'npm start' to launch the webpack dev server with the visual editor interface. 3. For Firebase functions, navigate to red-quick-functions/ and run 'npm run build' followed by 'npm run serve-all' to start the Firebase emulators. 4. For the React template, navigate to red-quick-template/ and run 'npm start' to launch the Material-UI based frontend. 5. Configure Firebase by setting up your Firebase project credentials in the appropriate configuration files. 6. The application will be available at the webpack dev server URL (typically localhost:3000) with the visual graph interface ready for creating functionality definitions.
