## Overview

This is a React-Redux test web application that demonstrates job management functionality using Redux for state management and Material-UI for the user interface. The application serves as a proof-of-concept for a job processing system where users can create, view, delete, and monitor jobs with detailed logging capabilities. Built in 2015, it showcases the integration of React 0.14 with Redux 3.0, using redux-router for client-side routing and isomorphic-fetch for API communication with a backend service.

## Architecture

The application follows a typical Redux architecture pattern with a clear separation of concerns. The main entry point is src/index.js which sets up the Redux store and routing configuration. The architecture consists of: Actions (src/actions/jobAction.js) that define job-related operations and async API calls, Reducers (src/reducers/jobReducer.js) that manage job state, Components (src/components/) for UI presentation including BlendedBatch, Details, EditJob, Job, and JobList, and Containers (src/containers/App.js) that connect Redux state to React components. The store configuration is handled in src/store/configureStore.js, and the application communicates with a backend API for job CRUD operations and logging.

## Key Features

- Job management system with create, read, update, delete operations through REST API integration
- Real-time job status monitoring with busy/available indicators using Material-UI avatars
- Detailed job logging and history tracking accessible via Details component
- Responsive Material-UI interface with left navigation drawer, app bar, and dialog modals
- Client-side routing with redux-router supporting /details/:id and /newjob routes
- Form validation and submission for new job creation with title, description, link, and keywords fields
- Asynchronous state management using redux-thunk for API calls to job endpoint
- Confirmation dialogs for destructive operations like job deletion

## Getting Started

1. Clone the repository and navigate to the project directory. 2. Install dependencies using 'npm install' which will install React 0.14, Redux 3.0, Material-UI, and all dev dependencies. 3. Ensure you have a backend API server running (the code expects it at localhost:52989 or configure window.jobEndpoint). 4. Start the development server using 'npm start' which runs the watch-html, watch-js, and serve scripts concurrently. 5. The serve script starts the Express server defined in src/server.js. 6. Open your browser to the development server URL to access the application. 7. For testing, run 'npm test' to execute Mocha tests with Babel compilation. 8. For production build, use 'npm run build' to compile the source with Babel to the lib directory.
