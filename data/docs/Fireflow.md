# Fireflow

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

FireRTC is a TypeScript library that enables real-time peer-to-peer data communication by combining Firebase Realtime Database for signaling coordination with WebRTC data channels for direct browser-to-browser communication. The library abstracts the complexity of WebRTC connection establishment, ICE candidate exchange, and offer/answer negotiation by using Firebase as a signaling server, allowing developers to easily implement P2P messaging and broadcasting functionality in web applications. Despite the package.json name 'firertc', the repository is named 'Fireflow' and provides a clean event-driven API for establishing secure, low-latency data channels between clients without requiring dedicated signaling infrastructure.

## Architecture

The architecture follows a client-side P2P pattern with Firebase-mediated signaling. The main entry point (src/index.ts) provides a useDataLayer factory function that handles Firebase app initialization, anonymous authentication, and returns a Promise-resolved DataLayer instance. The core DataLayer class (src/DataLayer.ts) manages RTCPeerConnection objects and RTCDataChannel instances in Maps, using Firebase Realtime Database paths structured as 'signaling/{prefix}/{channel}/{source}' for coordinating WebRTC offer/answer exchanges and ICE candidate sharing. The library implements an event-driven pattern where clients listen for signaling messages through Firebase onChildAdded listeners, establish direct WebRTC connections, and then communicate through data channels while maintaining connection state internally.

## Key Features

- Automatic Firebase Authentication with anonymous sign-in fallback for seamless user onboarding
- WebRTC peer-to-peer data channel establishment with automated ICE candidate exchange and offer/answer negotiation
- Multi-target messaging through send() method for direct peer communication and publish() method for channel-wide broadcasting
- Namespace isolation using configurable prefix option to separate signaling data across different applications or environments
- Event-driven message handling with on('message', callback) pattern for real-time data reception
- Connection state management tracking active RTCPeerConnection and RTCDataChannel instances with getActiveConnections() monitoring
- Promise-based initialization with useDataLayer factory function that handles asynchronous authentication flows
- Automatic signaling path construction with getSignalingPath() method supporting hierarchical Firebase database organization

## Getting Started

1. Install dependencies by running 'npm install' to get Firebase SDK and development tools including TypeScript, Jest, and type definitions. 2. Create a new Firebase project at Firebase Console and enable Realtime Database with appropriate security rules for signaling data. 3. Enable Firebase Authentication and configure Anonymous sign-in provider to allow automatic user authentication. 4. Copy your Firebase configuration object from Project Settings including apiKey, authDomain, databaseURL, projectId, storageBucket, messagingSenderId, and appId. 5. Build the TypeScript source by running 'npm run build' which compiles src/ files to dist/ directory using tsc compiler. 6. Import useDataLayer from the built library and initialize with your Firebase config and DataLayerOptions containing channel name, optional source identifier, and optional prefix for namespace isolation. 7. Use the returned DataLayer instance to set up message listeners with on('message', callback) and send data using send(data, target) or publish(channel, data) methods. 8. Run tests with 'npm test' to verify Firebase mocking and DataLayer functionality using Jest test suite in src/__tests__/DataLayer.test.ts.
