# fire-router

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

Fire-router is a TypeScript-based networking application that implements a WebRTC-based peer-to-peer communication system with Firebase integration for real-time data routing and connection management. The system acts as a communication bridge between peers, enabling them to establish connections through WebSocket signaling and exchange messages via data channels. It provides a distributed networking solution where nodes can communicate directly with each other while leveraging Firebase as a coordination layer for connection establishment and message routing.

## Architecture

The architecture follows a modular design with distinct components for connection management, WebRTC communication, and Firebase integration. The core components include ConnectionManager and ConnectionClient for managing peer connections, WebRtcConnectionManager for handling WebRTC-specific functionality, and FirebaseAdminApp for Firebase integration. The system uses a dual-communication approach: WebSockets (via websocket.ts) for initial signaling and control messages, and WebRTC data channels for direct peer-to-peer communication. Message routing is handled through the FireRouterMessage system defined in message.ts, with chunking support for large payloads. The client_emitter.ts and data_emitter.ts modules handle bidirectional message flow between peers, while peerconnectioncontext.ts maintains connection state and message queuing.

## Key Features

- WebRTC peer-to-peer connection establishment and management via ConnectionManager and WebRtcConnection classes
- Message chunking and reassembly system supporting large payload transmission through data channels with 8KB chunk size
- Firebase Realtime Database integration for connection coordination and logging with FirebaseAdminApp wrapper
- Dual-transport communication supporting both WebSocket signaling and WebRTC data channels
- HTTP proxy functionality allowing peers to make HTTP requests through remote connections via sendData function
- Real-time connection state monitoring with automatic cleanup and connection lifecycle management
- Message queuing and buffering system for handling connection state transitions and ensuring reliable delivery
- Comprehensive logging system with Winston integration supporting multiple log levels and file outputs

## Getting Started

1. Install dependencies using 'npm install' to get WebRTC, Firebase, and WebSocket libraries. 2. Configure Firebase credentials by placing your service account key in token.json file. 3. Set up Firebase project with Realtime Database enabled for connection coordination. 4. Build the TypeScript code using 'npm run build' which compiles src/ files to lib/ directory. 5. Start the first node with 'npm start' (runs with --f1 --computer --ws 6526 flags on port 6526). 6. Start the second node with 'npm run start2' (runs with --g1 --ws 6527 flags on port 6527). 7. Alternatively, run WebRTC-only mode with 'npm run w' which executes index_rtc.js. 8. Configure WebSocket ports and connection parameters in job_index.js based on your network setup. 9. Monitor connection status through Winston logs in logs.log and errors.log files.
