# petty-story

![Language](https://img.shields.io/badge/language-TypeScript-blue)

## Overview

Petty Story is a web-based comic book creation and editing platform that enables users to design comic covers, arrange comic panels, and generate complete comic books with PDF export functionality. The application consists of a React TypeScript frontend (petty-story) for the user interface and comic editing tools, paired with an Express.js backend server (petty-serve) that handles file management, image processing, and PDF generation. The platform provides a comprehensive comic creation workflow from character placement and speech bubble creation to final publication-ready output.

## Architecture

The system follows a client-server architecture with clear separation of concerns. The frontend is built as a React SPA with components organized into UI elements (AddButton, ColorPicker, Modal), canvas-based editors (ComicCoverCanvas, LayoutCanvas), and service layers (PettyService, BubbleDraw, drawing utilities). The backend Express server manages file operations across structured directories (comics, fonts, output) and provides RESTful endpoints for project management, character resources, and PDF compilation. State management is handled through React Context (ProjectsContext) while canvas rendering utilizes HTML5 Canvas API with custom drawing utilities for speech bubbles, masked images, and text rendering. The image caching system (useImageCache) optimizes performance for comic asset management.

## Key Features

- Interactive comic cover design with character positioning and scaling through ComicCoverCanvas component
- Multi-panel comic page layout system with drag-and-drop functionality via DraggableItem and DroppableArea components
- Advanced speech bubble creation with customizable pointers, borders, and text wrapping implemented in BubbleDraw.ts
- Character mask and image compositing system for visual effects using drawMaskedImage utility
- Dynamic font loading and text rendering with outline support through loadFonts.ts and custom text drawing functions
- Project-based file organization with hierarchical structure (projects/models/characters/images)
- PDF export functionality that compiles comic covers and pages into publication-ready documents
- Real-time image caching system for optimized asset loading and rendering performance
- Barcode generation for comic cover authenticity using drawBarcode utility
- RESTful API endpoints for project management, character resources, and file upload operations

## Getting Started

1. Clone the repository and navigate to the project directory. 2. Install backend dependencies by running 'npm install' in the petty-serve directory. 3. Install frontend dependencies by running 'npm install' in the petty-story directory. 4. Configure the base paths in server.js - update basePath (comics directory), fontsPath (fonts directory), and uploadPath (output directory) to match your local file structure. 5. Start the backend server by running 'node server.js' from petty-serve directory (runs on port 5315). 6. Start the React development server by running 'npm start' from petty-story directory (runs on port 3000). 7. Ensure the directory structure exists: create the comics, fonts, and output directories as specified in server.js configuration. 8. Access the application at http://localhost:3000 and the API at http://localhost:5315. The application expects a specific folder structure with projects containing models, which contain characters and their associated image assets.
