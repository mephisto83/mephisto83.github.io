## Overview

BridgeNative is a React Native mobile application that appears to be a communication and contact management platform built in 2015-2016. The app provides cross-platform functionality for iOS and Android, featuring real-time messaging, contact management, and various communication tools including SMS, calling, and email capabilities. The application implements a Redux-based state management architecture and includes authentication flows with external login providers, suggesting it's designed as a comprehensive business or personal communication hub.

## Architecture

The application follows a Redux-based architecture with separate platform-specific entry points (index.ios.js and index.android.js) that bootstrap platform-specific app containers. The core architecture includes: Redux store configuration with thunk middleware for async actions in native/android/containers/app.js, Navigator-based routing with conditional screen rendering in bridgeAppAndroid.js, modular screen components including LoadingScreen, LoginScreen, and CompositeScreen as primary views, and a tab-based interface using ScrollableTabView with custom TabBar components. The app integrates SignalR for real-time communication via a custom react-native-signalr module, implements WebView-based OAuth authentication flow in loginScreen.js, and uses a service layer (serviceCaller) for API communication with configurable domain endpoints.

## Getting Started

1. Clone the repository and run 'npm install' to install all dependencies including React Native 0.19.0 and required native modules. 2. Configure the domain endpoints by updating DEFAULT_URL and WS_DEFAULT_URL constants in native/globals.js for your backend services. 3. For Android: Navigate to android/ directory and run 'gradle build' or use Android Studio to build the project, ensure Android SDK and build tools are properly configured. 4. For iOS: Open ios/BridgeNative.xcodeproj in Xcode, configure provisioning profiles and certificates, then build for device or simulator. 5. Link native dependencies by running react-native commands for modules like react-native-camera, react-native-contacts, react-native-communications, and react-native-vector-icons. 6. Configure SignalR backend service to match the WebSocket endpoints defined in the app. 7. Set up OAuth providers for the authentication flow that matches the WebView implementation in loginScreen.js. 8. Run 'npm run start' to start the React Native packager, then use 'react-native run-android' or 'react-native run-ios' to launch the application.
