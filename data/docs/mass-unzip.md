# mass-unzip

![Language](https://img.shields.io/badge/language-JavaScript-blue)

## Overview

Mass-unzip is a specialized Node.js utility designed to automate the extraction and organization of font files from multiple ZIP archives. The tool recursively processes all ZIP files in a source directory, extracts font files (TTF, OTF, WOFF, WOFF2) from each archive, and consolidates them into a single target directory. This utility is particularly useful for developers, designers, or content creators who need to process large collections of compressed font packages and organize them into a unified font library structure.

## Architecture

The application follows a simple procedural architecture implemented in a single index.js file. The core workflow consists of three main functions: processZipFiles() which serves as the entry point and iterates through all ZIP files in the source folder, unzipAndMoveFonts() which handles the extraction logic for individual archives using the unzipper library, and isFontFile() which provides file type validation based on file extensions. The application uses fs-extra for enhanced file system operations and unzipper for ZIP archive manipulation, with synchronous directory creation and asynchronous file processing patterns.

## Key Features

- Batch processing of multiple ZIP archives from a specified source directory
- Intelligent font file detection supporting TTF, OTF, WOFF, and WOFF2 formats
- Automatic target directory creation using fs.ensureDirSync()
- Stream-based file extraction for memory-efficient processing of large font files
- Error handling with catch blocks for processing failures
- Configurable source and target directory paths via hardcoded constants

## Getting Started

1. Clone the repository and navigate to the project directory. 2. Install dependencies by running 'npm install' to install fs-extra and unzipper packages. 3. Edit the index.js file to modify the sourceFolder and targetFolder variables to match your desired source ZIP directory and target font output directory. 4. Ensure the source directory contains ZIP files with font assets. 5. Run the application using 'node index.js' to begin processing all ZIP files in the source folder. 6. Monitor the console output for the 'Processing complete.' message indicating successful execution.
