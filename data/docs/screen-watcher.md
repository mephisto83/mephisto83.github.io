## Overview

Screen-Watcher is an automated browser interaction and workflow orchestration system that enables scripted control of web applications through programmatic recipes. The system specializes in automating complex multi-step workflows on platforms like Instagram and Contrado (print-on-demand service), providing a framework for recording, storing, and executing browser automation sequences. Built with TypeScript/Node.js backend and browser-based element detection, it bridges the gap between manual user interactions and fully automated business processes.

## Architecture

The system follows a modular recipe-based architecture with several key components: a Node.js server (node/src/index.ts) that orchestrates automation workflows, WebSocket communication layer (node/src/websocket.ts) for real-time browser control, Chrome browser integration (node/src/chrome.ts) for programmatic browser control, and a recipe engine that processes workflow definitions stored as .recipe and .section files. The core workflow engine (node/src/workflows/workflows_support.ts) provides primitive operations like clicking, typing, and waiting, while recipe processors (node/src/recipeIndex.ts) coordinate complex multi-step sequences. Element detection and interaction is handled through CSS selectors and coordinate-based targeting, with screen capture capabilities for monitoring and debugging.

## Key Features

- Recipe-based workflow automation with sectioned command sequences stored in .recipe and .section files
- Multi-platform support for Instagram posting workflows (INSTAGRAM_post.recipe) and Contrado print-on-demand automation (Contrado.recipe)
- Real-time browser element detection and interaction through CSS selectors and coordinate targeting
- WebSocket-based communication for live workflow monitoring and control
- Screenshot capture and visual feedback system for workflow debugging
- Modular watch modes (contrado_wm, discord_wm, watcher_wm) for different automation contexts
- Custom function support for complex business logic like image selection and processing
- Cross-platform desktop automation using RobotJS for system-level interactions
- Docker containerization with Google App Engine deployment support

## Getting Started

1. Clone the repository and navigate to the node/ directory. 2. Install dependencies with 'npm install' which will install TypeScript, RobotJS, Express, WebSocket libraries and other required packages. 3. Build the TypeScript source with 'npm run build' which compiles src/ files to lib/ directory. 4. Configure workflow recipes by modifying files in node/src/recipes/ directory - customize Contrado.recipe for print-on-demand automation or INSTAGRAM_post.recipe for social media posting. 5. Set up instruction files in instructions/ directory (instagram.json, twitter.json) with platform-specific credentials and settings. 6. Run the main application with 'npm run app' or specific workflows like 'npm run instagram'. 7. For production deployment, use the included Dockerfile to containerize the application and deploy to Google App Engine. 8. Access the WebSocket server on the configured port (defined in node/src/PORTS.ts) for real-time workflow control and monitoring.
