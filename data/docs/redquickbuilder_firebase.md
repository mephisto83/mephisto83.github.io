## Overview

RedQuickBuilder Firebase is a comprehensive web application development platform that combines Firebase backend services with a sophisticated visual builder interface. The project appears to be a low-code/no-code solution for rapid application development, featuring a mind-mapping style interface for building functionality and UI components. Based on the extensive AdminLTE CSS framework integration and multiple bundle files, this is a full-featured development environment that allows users to visually construct applications with drag-and-drop components, data visualization tools, and real-time collaboration features.

## Architecture

The application follows a Firebase-hosted architecture with separate frontend and backend concerns. The frontend is built as a single-page application using webpack bundling (evidenced by bundle.js files) and includes a comprehensive mind-builder interface located in public/mind-builder/ with modular CSS components. Firebase Functions handle backend logic while Firebase Hosting serves the static assets from the public/ directory. The architecture includes TypeScript definition files for CSS modules, suggesting a modern component-based approach with type safety. The project includes Docker support and uses AdminLTE as the primary UI framework with extensive customization through multiple skin variants and plugin integrations.

## Getting Started

1. Install Node.js 16 (as specified in package.json engines). 2. Clone the repository and run 'npm install' to install firebase-admin and firebase-functions dependencies. 3. Install Firebase CLI globally with 'npm install -g firebase-tools'. 4. Initialize Firebase project and configure firebase.json, firestore.rules, and database.rules.json files. 5. Set up Firebase project credentials and ensure proper IAM permissions. 6. Run 'npm run build' to compile TypeScript files. 7. For development, use 'npm run test-watch' for continuous TypeScript compilation. 8. Deploy functions only with 'npm run deploy', hosting only with 'npm run publish', or everything with 'npm run all'. 9. Configure CORS settings using cors.json file. 10. Set up Firestore indexes using firestore.indexes.json. 11. If using Docker, build the container using the Dockerfile in the project root.
