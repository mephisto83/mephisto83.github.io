# htmlelementtest

![Language](https://img.shields.io/badge/language-HTML-blue)

## Overview

htmlelementtest is an experimental Unreal Engine 4 project that implements HTML/CSS element rendering and visualization within a 3D environment. The project combines web scraping capabilities with real-time 3D rendering to capture and display HTML elements as interactive visual components. It features a sophisticated web crawling system using Puppeteer that extracts visual elements from web pages and translates them into Unreal Engine assets, creating a bridge between web content and 3D visualization. The system includes a socket.io server for real-time communication and a comprehensive set of UE4 blueprints for rendering HTML elements as 3D objects with proper CSS styling and positioning.

## Architecture

The project follows a multi-tier architecture with three main components: a Node.js web scraping server (socketserver/), an Unreal Engine 4 game project, and a real-time communication layer. The web scraping tier uses Puppeteer to navigate websites, extract visual elements, and capture screenshots, implemented in webcutter.ts. The extracted data is structured using TypeScript interfaces (visual-element.ts) defining VisualElement, HTMLPaneUpdate, and related data structures. The UE4 tier contains blueprint assets for HTML rendering including HTMLElement.uasset, HTMLPane.uasset, HTMLVisual.uasset, and specialized components like DivElement.uasset and TextElement.uasset. Communication between tiers is handled via Socket.IO, enabling real-time updates of HTML content within the 3D environment. The Content/HTML/ directory contains the core rendering blueprints, while Content/Mannequin/ provides standard UE4 character assets for interaction testing.

## Key Features

- Web page scraping and visual element extraction using Puppeteer with configurable viewport sizing
- Real-time HTML element to 3D object conversion with CSS property preservation
- Socket.IO-based live communication between web scraper and UE4 engine
- Comprehensive Blueprint system for rendering HTML elements including HTMLElement, HTMLPane, DivElement, and TextElement
- Screenshot capture system for individual HTML elements stored in output directory
- CSS property parsing and translation to UE4 material properties including background-color, border properties, and positioning
- 3D spatial positioning system with PaneProperties supporting location, rotation, and scale transformations
- Dynamic image updating system with timer-based content refresh
- Visual depth calculation based on CSS z-index and DOM hierarchy
- Modular architecture supporting multiple HTML panes with independent update cycles

## Getting Started

1. Install Node.js dependencies by navigating to socketserver/ and running 'npm install' to install TypeScript, Puppeteer, Socket.IO and Express dependencies. 2. Create an 'output' directory in the socketserver folder for screenshot storage. 3. Build the TypeScript server using 'npm run build' which compiles src/index.ts and related TypeScript files to dist/. 4. Open the Unreal Engine 4 project by loading HtmlElementTest.uproject, ensuring UE4 version compatibility with the .uasset files. 5. Verify the C# build configuration by checking Source/HtmlElementTest/HtmlElementTest.Build.cs for proper module dependencies. 6. Start the Node.js server using 'npm run go' which will launch the web scraping service on port 3000. 7. Configure the target website URL in src/index.ts (currently set to Epic Games Store) and adjust viewport dimensions as needed. 8. Launch the UE4 project in editor mode and ensure the SocketManager blueprint is properly configured for localhost:3000 connection. 9. Test the HTML element rendering by running the game and verifying that HTMLPane blueprints receive and display web content updates via the Socket.IO connection.
