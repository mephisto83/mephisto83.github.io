# flow-journies

![Language](https://img.shields.io/badge/language-HCL-blue)

## Overview

Flow-journies is a comprehensive workflow automation platform that orchestrates complex multi-step processes for content creation, particularly focused on social media and creative workflows. The system uses a node-based visual workflow engine that can automate tasks like AI-generated content creation, social media posting, video rendering, and browser automation. The platform integrates with multiple services including Midjourney for image generation, Blender for 3D rendering, Chrome browser automation, and various social media platforms like Instagram, TikTok, and YouTube.

## Architecture

The system follows a modular workflow architecture with JSON-based workflow definitions containing nodes and edges that define execution flow. Each workflow file (*.workflow) contains a graph of interconnected nodes including inputNode, execJSNode, fetchNode, workflowNode, pauseNode, and resultNode types. The architecture supports nested workflows through graphNode and workflowNode references, allowing complex process composition. Browser automation is handled through REST API endpoints for Chrome control, file operations, and system interactions. Global state management is provided through globalNode and globalGetNode for sharing data across workflows.

## Getting Started

1. Set up the workflow execution environment with Node.js runtime for execJSNode processing. 2. Configure REST API endpoints for browser automation (routes like /openChrome, /closeChrome, /bringAppToFront). 3. Install and configure Blender with Python API access for 3D rendering workflows. 4. Set up Chrome browser with automation capabilities and required extensions. 5. Configure API keys and authentication for external services (Midjourney, OpenAI, social media platforms). 6. Set global variables using globalNode configurations found in projects/configs/social/poller.workflow including paths like music_video_ouput_path, settings_path, and service-specific prefixes. 7. Configure file system permissions for automated file operations in specified directories. 8. Set up email integration for workflow triggering via the pollerNode system. 9. Configure output directories for generated content as specified in workflow outputDirectory fields.
